Registry Printer Margin Override

Margin Issue On Specific Printers

This registry option has been available for some time but was not released by programming for documentation until now. This registry should only be used if a certain printer consistently ‘chops off’ the left-hand (or top line) of SAM Pro reports and no other remedy is available. Note that this entry must be used with caution as bitmaps associated with the report (e.g., invoices and purphase orders) are NOT repositioned, when may lead to lose of proper alignment; adding whitespace to the graphic may correct this issue. It is up to the support person to ensure that all reports print in a satisfactory manner with this entry in place.

Id:

sys-printer-printername--userid

sys-printer-printername-branchid-userid

sys-printer-printername-branchid

sys-printer-printername-securityid

sys-printer-printername

where printername is the name of the printer exactly as it appears in the Name field of the Windows Printer dialog box. Please note that this field is upper/lower case sensitive. If this entry is specific to a user, the branch must be specified if a user belongs to a branch. If branches are not implemented, two dashes (not separated by a space) must precede the user id.

Setting Key: marginoffset Setting Value: x@y

Where x is the character offset (horizontal) and y is the line offset (vertical) from the upper left-hand corner of the page.

For example, if three characters were consistently chopped off of the left-hand side of reports, the following option would be enabled:

Setting Key: marginoffset Setting Value: 3@0
