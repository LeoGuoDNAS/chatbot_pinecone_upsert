AR Terms Viewer

Purpose:

Maintain standard accounts receivable terms information.

Example:

Net30, COD, etc.

Usage:

Referenced By: Client Ship To, Client Order, Client Site, and Job.

Client Site: The AR Terms referenced in the Client Site record is used for Client Order Processing to provide the default AR Terms Id for COP Invoices. It is not used in conjunction with Job or Service Invoicing.

Job: The AR Terms used for Construction and Service Billed defaults from the Job record.

Important Fields:

Note: this Help Topic lists the standard configuration of all fields which may be included in this record. If you are using a customized system, the fields displayed on your screen may not exactly match the order or status (required, optional, default, display-only, hidden, etc.) of the fields described in this Topic.

AR Terms ID: (10 character, only . and - special characters)
A unique Id. This field is required.

AR Terms Description: (40 character)
A brief description of these terms or how they are applied. This field is optional.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

Due Days:
The number of days after billing until this account is considered overdue.

Discount Days
This field should only be filled in if you allow your customer a discount for early payment. This field specifies the number of days from your invoice date that the discount is available (e.g., 10).

Discount Percent:
If a discount is offered for early payment, enter the value as a percent. Note that the available discount will be calculated automatically for COP Invoices and manual Sales Journal entries.

In Version 9.1.611.058 and above, the AR Terms viewer displays the Discount Percent as 000.00. However, validation prevents setting the decimal places unless the Global Registry for AR Terms Discount Percent  is set. Validation also controls the hundreds / tens settings (XX0.00) in AR Terms when the registry is set.

COD: Check the COD box (by double-clicking or pressing the spacebar) if the terms are Cash on Delivery.

Annual Finance Charge Rate:
If these AR terms include a finance charge, enter the rate as an annual percent.

Obsolete?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be cleared when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record.
