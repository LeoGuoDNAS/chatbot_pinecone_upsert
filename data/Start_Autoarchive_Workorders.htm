Start Autoarchive Workorders

Purpose:

The function Start Autoarchive Work Orders is designed to allow high-volume customers to set older Work Orders to an Archived status.

Related Functions:

See Start Archive Work Orders for details on this manual archiving function.

Usage:

This function is designed to be run as a scheduled event. The following registry options must be in effect. Contact your support representative for assistance in setting this up.

Registry setting parameters:

Id: sys-start-Sautoarchivewo

Setting Key: DaysOld Setting Value: #days

If this was set to 180 days, the work order would have to have a date closed on it 180 days or earlier than today.

Setting Key: EscalationStatuses Setting Value: billed,cancelled

This is the recommended setting. Archiving ‘completed’ work orders with this function is NOT recommended as they may have unbilled costs.
