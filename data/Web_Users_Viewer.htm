Web Users Viewer

Purpose:

Create accounts that will enable to enable web access logons to FMAnywhere and the Client Web interface.

Usage:

This viewer supports FMAnywhere – the Facility Manager interface.

Registry:

Registry FMAnywhere

Registry CRMAnywhere (web-settings)

Reporting on Customer Logons

When a Web User logs on an entry will be added to the User Logon Table. This table may then be queried for reporting purposes such as marketing analyses of which customers are using web access and so on. Information in this table includes: user id, date and time of logon, date and time of logoff, user access type, virtual machine and user name, logon status, and sampro version number.

Main Tab

Web User Name:
This will be the name the user logs in as, and must uniquely identify this record.

Web User Email:
A valid e-mail address for this user.

New Password:  Confirm Password: (10 character)
Use these fields to enter and confirm a new/changed password for this Web User. The password does not display in either of these fields for security purposes. The password may contain letters and numbers, but should NOT include special characters such as the ampersand &, pound sign #, and blank spaces.

Expiration Date:
The expiration date for this web user account.

Disabled?
If this field is flagged with an X, the user will be unable to access the Data-Basics' software. Double-click on this field (or press the Spacebar) to flag/clear it.

Web Profile Tab

This tab should be completed if this is Web User record is for a customer who will be accessing FMAnywhere or the Client Web Interface.

Web Profile Id:
Enter a valid Web Profile Id or double-click to select from a list. This field is required for FMAnywhere or /Client Web Interface  users.

Service Category:
This range enables you to filter the Service Categories that will be available to this Web User on FMAnywhere or the Client Web Interface. If this range is empty, all Service Categories will be listed. The Client Web Interface applies the Web User's Service Category filtering (if set) to the Equipment and Open and Completed WO lists.

Store #
If this range is empty, the web user will be able to ‘see’ all sites linked to the Web Profile subject to the restrictions in the custom ranges below.

Custom Range 1, 2, 3, 4, 5
The ranges below the Store # range will be labeled according to the scheme defined in the Web Profile (if implemented). If this range is empty, the web user will be able to ‘see’ all sites linked to the Web Profile subject to any restrictions in the Store # range above.

To restrict the sites available, enter an appropriate custom value (or list of comma-separated values) in one or more of the custom ranges.

Registry Tab

Registry Value:
The registry options entered on this tab will override those specified in the Web Profile record (for CRMAnywhere users) and web-settings registry entry, and are typically used for custom branding (e.g., displaying Big Bank graphics when your Big Bank customers log in). You may also enter registry options on this tab to customize work order and invoice formatting, reporting, and e-mail settings for CRMAnywhere. Click Registry Entry Client Web Interface ; the Contents/Additional Contents options described for the web-settings registry entry can also be entered on the lines in this tab.
