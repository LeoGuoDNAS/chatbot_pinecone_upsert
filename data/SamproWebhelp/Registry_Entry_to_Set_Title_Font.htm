Registry to Set the Title Font (sys-font-title)

Purpose:

This registry entry enables you to control the font used to title tables that include at least one data-entry field.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id:  sys-font-title

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Set Font Name

Setting Key: Facename Setting Value: aFaceName

where aFaceName is replaced by a valid font name (e.g., Arial, Tahoma). This font must be installed on the users’ computer or it will not be used.

Setting Key: Bold  Setting Value: Yes or No

Specify yes to use the bold version of the font, or no to use the normal version.

Setting Key: Points:  Setting Value: aPointSize

Where aPointSize: specifies the size in points. Point sizes ranging from 8-10 are recommended.

In previous releases, the registry entry (sys-font-title) used to specify the font for labeling titles and lists was restricted to a 10 point fixed width font. Changes to the way in which the titles are constructed means the registry is no longer limited to fixed-width fonts or a 10 point font size.

Example:

Id: sys-font-title

Setting Key: Facename  Setting Value: Arial

Setting Key: Bold  Setting Value: No

Setting Key: Points  Setting Value: 9
