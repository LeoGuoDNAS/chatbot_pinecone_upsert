Internal Registry User Status Tracking (UserStatusTracking)

Purpose:

This registry entry is required to implement User Status Tracking.

Id:

UserStatusTracking

Contents / Additional Contents:

DoTracking=value
Any value beginning with ‘y’ or ‘Y’ turns tracking on; any other value (or no entry at all) turns tracking off. This value is checked each time a tracking event is registered. The user does not have to log out and back in for this change to be recognized.

RetainDays=value
’value’ represents the maximum number of days of history to be retained in the UsrSttsTrckr table. If this line is absent or if ‘value’ is zero, all previous entries for the user/function combination are deleted from UsrSttsTrckr. If ‘value’ is non-zero, only entries older than a computed cutoff date are removed. This cutoff date is computed by subtracting the retain days indicated by ‘value’ from the current date.

The purging action is initiated each time a function is launched with the User Status Tracking mechanism turned on.  Purging is done by User for the current function being launched.  Consider the following example, which for simplicity sake is based on RetainDays=0.

On the first Monday of January, user Joe starts his first day by logging in and opening the Client viewer.  During the course of the day, Joe opens and closes the Client viewer five times.  At the end of the day, Joe has five entries in the status tracking table.

Joe logs in on Tuesday and opens the Client viewer.  At that point, the five entries from Monday are deleted (RetainDays=0) from the table and a new one is added for Tuesday.

This scenario repeats daily until Friday at which point Joe quits and never returns.  Because Joe's account is never used to open the Client viewer again, his status tracking entries from Friday will never be deleted.
