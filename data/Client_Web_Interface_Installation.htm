Client Web Interface Installation

Network Considerations

If you want to be able to access the CRMAnywhere website from outside of the local network, the appropriate network policies for the TCP port will need to be changed in the network firewall.

Installing the Client Web Interface

Load the appropriate graphics and default.css files into a subdirectory of SAMPRO (see the graphics files specifications below). If you are implementing client branding, you will need to create additional directories to contain the appropriate graphics files for your customers.

- Create a temp files subdirectory

- Make the appropriate Registry Entry Client Web Interface record to specify the location of the graphics files and temp files. Note the some of these graphics will be standard ones obtained from your support representative and others will be your own custom ones.

- Restart the web server

- Import the Web reports

- Make user-defined function entries for those reports

- Java Runtime Environment installed

- Latest jar file uploaded

Notes on SAM Pro Records

Creating accounts for your Web Clients: Client Web users are set up in the Web Users Viewer. Create a new Web User Record for each Client you wish to give web access to.

The Client Web Interface now applies the Web User's Service Category filtering (if set) to the Equipment and Open and Completed WO lists. Only Work Orders with blank Service Categories (in the Work Order header) or those that match those specified in the Web User’s record will appear in the Equipment and WO lists.

Web Profiles: these records filter. client site equipment (and client sites) for users, as well as Service Categories, and through them, Work Codes. Web Profiles may also be used to implement corporate branding by customer – see Web Profile Viewer for details.

Update the Client Site record’s Web Access tab to point to appropriate Web Profile and custom values; see Client Site - Web Profile for detail.

Purchase Order numbers don’t have a pre-defined field in the New Call page in this release. You can set up a P.O.# call script field to handle it (to be entered manually into the work order on work order creation).

CRMAnywhere supports user-defined remapping of work order status codes (see Registry Entry to Remap WO Status on Web Client).

Service Categories are required in order to generate new service requests from the New Call page. If you do not have service categories set up, create one (e.g., All), and point all applicable Work Codes at this Service Category.

In the Work Code record:

Check the Web Access: Allowed? box if you want it to appear on the CRMAnywhere New Call page. This box should remain blank if you do not want your client to be able to select this work code.

In addition, the Service Category field must be set in order for the CRMAnywhere user to be able to select the work code successfully when requesting a New Call.

To display Call Script questions in CRMAnywhere, a Call Script Id must be referenced Work Codes that are flagged for web access.

See Auto-Generate Work Orders from Web Logs for details on implementing this functionality.

Service History Report

The web Client Web Service History Report has been enhanced so that the equipment that appears on the report may be selected, and a custom date range may be specified.

Setup Information:
To implement this functionality, the zipped file WebCalendarCode.zip needs to be unzipped to the SAMPro folder specified by the web-settings registry’s web-dir= option.

An up-to-date version of RSRVCEHSTRYDTL.rpt should be installed in the datatabase.

Service History Report Page

New Field: Equipment Id:
An Equipment Id drop down box has been added to the Service History Report range screen; it will initially be set to "ALL" so that all active equipment will be included on the report. To limit the report to a specific item, you may select it from the list of Id’s.

Date Range:
A new option, "Custom Date Range" has been added to the date range drop down for the Service History Report.  If this is selected, two fields for start and end dates will be added to the right of the drop down.  The way to set their values is through a pop up calendar. 

Manage Quotes Page

The print quote code uses the Report Id in the Quote viewer to select which format quote form to use. If the Quote doesn't include a Report Id (or when the Report Id links to a report produced outside of SAMPRO) the user-defined function 'RQTEFRM' is executed to print the standard quote.  If RQTEFRM doesn't exist as a user-defined function, then the user sees an error message ... "The function entry was not found for this report.".

Notes on SAM Pro Records – Version 6.1

Note: Before version 6.2, the Technician and Web Security records controlled web access and structure. Although these records are still supported, the Web User and Web Profile records are preferred.

Reporting on Customer Logons

When a user logs on, an entry will be added to the User Logon Table. This table may then be queried for reporting purposes such as marketing analyses of which customers are using web access and so on. Information in this table includes: user id, date and time of logon, date and time of logoff, user access type, virtual machine and user name, logon status, and sampro version number.

Graphics File Specifications – New Header Styles

This enhancement may be enabled to improve the way the page headers are handled. If CRM was implemented in an earlier release, modifications to your existing graphics will be needed to support the change.

When these functions were first implemented, lower resolution screens were the norm, and the headers were designed be left justified on 600X800 pixel screens, with the HTML generated by SAM Pro centered.

To be more suitable for larger screen, the following registry entry may be used to enable the centering of headers with an appropriated background graphic, and a right-justified graphic to complete the screen:

Id: web-settings
Setting Key: NewHeaders Setting Value: true

  1. The right-hand side of the header should be named right.gif. This graphic will appear at the right-hand side of the header for all pages. It should be 68 pixels high, and wide enough to display the desired graphic.

  2. The portion of the header gif’s containing the title (e.g., Main Menu, View Quotes, etc.) should be wide enough to display the appropriate title, and be 68 pixels high. These graphics will be centered in header.

Graphics Files Specifications

The Client Web Interface allows for your customer’s corporate branding on individual page headers, a standard sidebar, and buttons. Your graphics must be named as follows (and use the following specifications) in order to work properly. Branding by customer is accomplished by placing the customized files in subdirectories, and referencing the appropriate path in the Web Profile or Web User record.

1. All graphics must use the gif format.

2. Filename: sidebar.gif

A sidebar area - 112 pixels wide by at least 440 pixels high. This will stay the same for all screens. The top 80 pixels may include your logo. The software will overlay menu options on the middle 300 pixels. A message or additional graphic may be positioned on the bottom 80 pixels of the sidebar.

5. Setting up the Calendar for the Service History Report Enhancement

The web Service History Report has been enhanced so that the equipment that appears on the report may be selected, and a custom date range may be specified.

To implement the new functionality, the zipped file WebCalendarCode.zip needs to be unzipped to the SAMPro folder specified by the web-settings registry’s web-dir= option.

RSRVCEHSTRYDTL.rpt should be installed in the database.

A "Custom Date Range" appears in the date range drop down for the Service History Report.  If this is selected, two fields for start and end dates will be added to the right of the drop down.  The way to set their values is through a pop up calendar.  If both values are present in the fields, they will be used for the date range of the report.  A message to fill both fields will be alerted to the user if only one field has a value.  If the "Custom Date Range" entry is deselected, the two fields will disappear and the selected date range will be used as per normal.

Stylesheet Options

The Client Web Interface uses standard stylesheet conventions to customize various elements such as the font type and color, background color, and so on. If you are not familiar with standard .css conventions, please contact your support representative.
