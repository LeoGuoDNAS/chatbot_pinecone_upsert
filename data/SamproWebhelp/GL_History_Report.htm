GL History Report

Purpose:

This viewer runs the GL History Report, and controls the range of data and dates on the GL History report.

This report is ordered by GL Account Id, and displays the following information for each transaction: Effective Date, Journal, Transaction Id, GL Entity Id, Debit or Credit amount, Balance and Description.

Usage:

After completing the fields and clicking the go button, the system will ask you to select a printer. After you do so, the Report viewer will appear so that you can select the pages and number of copies to print.

If multinational currency is implemented on your system, be aware of the following when reading this report:

A. A/P and A/R Control and Retainage Accounts for Foreign Currency Vendors/Clients:

 These will be maintained in the ‘original amount’ of the foreign currency

B. Inventory Expense Accounts

These accounts will be maintained in your firm’s local currency. Foreign currency amounts will have been converted to your local currency using the Currency Factor in the AP Journal.

C. Foreign Exchange / Currency Adjustment Accounts

 These will balance the difference between A & B (see above).

Important Fields:

Report Options:
The Report Options selector pane controls how the report will be output. Click Report Options to learn more.

Report Ranges:
The Report Ranges selector pane enables you to select a customized range for the report. When you click on a range, the criteria defining it will appear in the ID field(s). Creating a customized report range is discussed below. Notice that you must name a custom range in order to save it. Enter a unique, alphanumeric name to save and identify your customized report Range.

Hint: You can create user-defined report ranges that are available to all users with access to that report. To do so, the first character of the named range should be an asterisk. Named ranges beginning with an asterisk will appear on all users' report ranges options so that they can be selected (but not altered) by any user with access to that report. Only the user that created the report range is allowed to alter it.

Start Date End Date
These required fields define the range of effective dates on the report. Enter the date in mmddyyyy format or double-click on the field to access the Calendar viewer to do so.

Consolidated Report?
If this box is blank and your firm uses GL Entities, separate lines will be generated for each applicable entity per entry per account. If you check this box, entity information will be consolidated into one line.

GL Entity Group:
Limits the report to entries pertaining to this GL Entity Group(s).

GL Account Id:
To display history for a single GL Account, enter its Id. Multiple GL Accounts may be specified by separating their Id's with a comma (10011,10012). A range of GL ID's may be defined by the first and last items of the range separated by an underscore (10011_10067).

GL Entity Id:
To display history for a single GL Entity, enter its Id. Multiple GL Entities may be specified by separating their Id's with a comma (11,12). A range of GL Entities may be defined by the first and last items of the range separated by an underscore (11_67).

Source
To only display history from a particular source, enter the 2-character source of the entry (such as AP, AR, WO, PO, PR, GL).

Source Journal
To only display history from a particular journal, enter its title (such as AP Check, AP Journal, AR Receipts, AR Sales, WO Inventory, WO Labor, PR Check, GL Journal). See Source Journals Referenced in History Reports for further details.

Transaction Id:
Limits the report to entries pertaining to this transaction.

Invoice Id:
Limits the report to entries pertaining to this invoice.

Power-User Reporting Tip:
 You may also use the comparison operators >,<, = and ! in your report ranges to control the information output to your report. Click Comparison Operators for details.
