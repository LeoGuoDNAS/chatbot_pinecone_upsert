View Logon History Table

Important Notice: this help file is intended for use by Authorized Data-Basics Support Personnel only. Incorrect implementation of the information discussed here can create problems with software functionality and/or data.

Purpose

You may use View Logon History to both determine which version of SAM Pro each user is using and to terminate their SAM Pro session. If you terminate their session (by clicking the Terminate Session button), they will be able to complete what they are doing, but not to do anything new.

This table records the User Id, Logon Date and Time, Engine, Logoff Date and Time, Success (of logon), and Windows Logon name (Windows Logon name is not supported in Windows 98 and ME).

Usage:

We recently encountered a situation where we have implemented a fix on a user's system ... and the user reported that the fix was 'lost'.  After a certain amount of wasted time, it turned out that the user had never EXITED SAM Pro and logged back on (although he said he had) - and so was NOT seeing the fix that other users were seeing. Going directly to View Logon History would have enabled us to resolve this situation more quickly.
