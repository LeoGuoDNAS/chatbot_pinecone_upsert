Registry Report Canadian T4 (Standard)

Purpose:

This registry entry is required to print the T4 contact information in the report.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Related Registry Entries:

See Registry Entry T4 Printer for details on specifying which printer the T4’s are sent to and how they are aligned on the paper.

Company Record Support

In SAMPro Version 9.1.611.054a, the Canadian T4 reports have been revised to use the Company record / Company Registry tab to specify the Company Name and Address and other registry settings required for these reports.

Company Name, Address: These reports will use the Company Name & Company Address from the Company Record Header info if available.

Company Registry Tab Fields: Fields not defined in the Company Record header may be added to the Company record's Registry Tab.  For example, the t4sin1 for the Summary report.

First look for the Company Record's Registry entry. If not found,
Second look for the sys-report-rT4 If the registry is blank or not found,
Third look for the company specific companyname registry setting (e.g., companyname-1) and use that.  
If none of the above are available, the string 'companyname' will be used in the report instead of leaving it blank.

Option for Partial Year Reporting

This registry option enables partial year reporting. This would be desirable if, for example, an employee moved from one Province to another mid-year.

Setting Key: PartialYearReporting  Setting Value: true

Note: The system sets the Start Date (01/01) and the End Date(12/31) whenever the Year field gets changed. The system will reject attempts to edit the Start Date and the End Date unless this registry entry is set.

Special Processing for Partial Year Provincial Reporting on Printed T4 Reports

If an employee moves from one province to another during a calendar year, special processing must be followed in order to produce the correct printed T4's for that employee.

1. The registry sys-report-rt4std must include the following setting to allow the Start and End Dates to be edited when the T4 is run: Setting Key: PartialYearReporting  Setting Value: true

2. Determine when the Provincial Tax in the Employee's Tax Tab was changed from Province 1 to Province 2. You will need to determine the last Payroll Period the employee's check was printed with their Province set to Province 1.

3. Run the Employee's T4 with the End Date set to the last Check Date earned in Province 1, and the Province field set to Province 1.

4. Run the Employee's T4 with the End Date set to the day after the last Check Date earned in Province 1, and the Province field set to Province 2.

Canadian Electronic T4

When running the Report PR Canadian Electronic TA the Provincial field should NOT be completed. It is not necessary to specify partial year processing; the electronic file will contain the appropriate information for provincial tax for each employee.

 

Important Notes on Other Viewer Settings :

Tax Authority Records

For any provincial Tax Authority Records, you need to put the province code in the ‘Prov:’ field

Deduction Records

For the deduction records for Union Dues or Charitable, you will need to populate field ‘Deduction User 1’ with Dues or Charitable, respectively. The Deduction Type must be set to Other Pre-Tax.

Employee Records

For any employee that is CPP, EI or PPIP exempt, in the Employee Viewer, on the ‘User Fields’ tab, need to populate Employee User 8, 9 or 10 respectively with a ‘Y’.
