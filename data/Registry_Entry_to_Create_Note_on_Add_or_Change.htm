Registry Create Note on Add or Change (sys-viewer-afunction)

Purpose:

These options will create and display a Note each time a record is added (AutoDocumentAfterAdd=true) and/or an existing record is saved (AutoDocumentAfterChange=true).

In previous releases, these Notes were set to Public by default, resulting in them being uploaded to Service Channel when certain records were saved.  In version 9.1.611.054 and above, the Notes default to Private when generated via this registry.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id: 

sys-viewer-vFunctionId

 where vFunctionId is replaced by the Id of the viewer you wish to adjust.

Specify Autodocumentation

Setting Key: AutoDocumentAfterAdd Setting Value: true
Setting Key: AutoDocumentAfterChange Setting Value: true
These options will create and display a Note each time a record is added and/or an existing record is saved. The user should then enter the reason for the changes to / addition of the record, and save and close the note. This registry option pertains to all static records (such as the client site) as well as journals.

In previous releases, Notes added via the AutoDocumentAfterChange option were set to Public by default, resulting in them being uploaded to Service Channel when certain records were saved.  In version 9.1.611.054 and above, the Notes default to Private when generated via this registry.

For example, if the following registry entry were implemented, a Notes window would open when an Employee record was changed, so that the user could document the reason for the change.

Id: sys-viewer-vemplye
Setting Key: AutoDocumentAfterChange Setting Value: true
