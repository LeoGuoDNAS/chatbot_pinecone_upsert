Tax Tables
The following Id formats for your FICA Social Security and Medicare records must be followed or boxes 4 and 6 will not print on the W2's:

 FICA SS (note the blank space separating FICA and SS)
 FICA MC (note the blank space separating FICA and MC)
FICA MCAD (additional tax for high-wage earners)

Information on completing each year's Federal Tax Tables with be provided by Data-Basics when this information is officially release by the IRS.  It is the user's responsibility to update the applicable state local tax tables when needed and to correctly maintain the federal tables.

Employee Record
W2 fields must be completed as needed.

W2's should be run after processing payroll is complete: all payrolls are entered, checks/direct deposit advice printed, payroll posted, and year-end adjustments made. Next, run the appropriate payroll reports to verify that the wage and tax amounts are correct.

Box 14
To implement SDI (Box 14 on form W-2), the W-2 Category Employee field in the applicable Tax Authority or Tax Table must be set to SDI. The Tax Authority must also be set up as type Other.

Box 10
To support Dependent Care Benefits (Box 10), the W-2 Category field in the appropriate Deduction record must be set to DC.

Box 12 code C
This will be completed from Employee fringes.

Box 12 Code DD Support for Employer Sponsored Health Coverage

This reporting requirement was introduced by the 2010 Affordable Care Act. The idea is that reporting the cost of healthcare will enable employees to be better informed about their healthcare costs and the choices available to them. For 2014 the reporting is required for employers filing more than 250 W-2s. The 250 limit may be reduced in the future but for the moment small employers are not required to report the cost of healthcare coverage.

SAMPro handles this by treating the Employer Sponsored Health Coverage as a non-taxable fringe and using the PR Adjustments Journal to record the dollar amounts for each employee.

1. If necessary, create the appropriate Earnings record for this fringe, and enter 12:DD into the W-2 Category field.

2. Create a PR Adjustment entry for each employee; enter the appropriate Fringe amount on the Earnings tab.  The GL transactions would already have been made; the PR Adjustment entry gets it on the W-2.  

Alternate Box 12 Code DD (HSA Employer Costs) Processing
Some users wish to set up an Earnings Id to process W2 12:DD (HSA Employer Costs) and include this Earnings Id in regular PR processing. This procedure populated both prhstryernngs_amnt_ernngs & prhstryernngs_amnt_fringe fields. The W2 report sums (prhstryernngs_amnt_ernngs + prhstryernngs_amnt_fringe) which is cause the amount in box 12:DD to be overstated.  NOTE: This processing does not follow the instructions in the Data-Basics EOY document.

In this release, setting the Earning record's User2 field to a value of FringeOnlyW2 changes the default behavior to only include prhstryernngs_amnt_frnge in the calculation for specific earnings on W2's which corrects the overstatement.

Box 12 Code P

SAMPro supports the use of the W-2 Box 12 code P for Excludable moving expense reimbursements paid directly to the employee via the W-2 Category field (or the User 2 field in older versions) in the Other Rate record.

Create an Other Rate Id for REIMB MOVE:

Set the description to Moving Expense Reimbursement.
Set the W-2 Category to 12:P
Set the GL Debit to the GL Account you want to charge.
The GL Credit should be set to the Accrued PR account.
Do not set any dollar amounts in the Rate or Cost fields.
Set the billing rate type as N/A.

Record the Moving Expense:

If the employee has not yet been reimbursed for the moving expenses, enter the expense on the Labor Journal Other tab, using the REIMB MOVE Other Rate. When the check is cut, the reimbursement will be recorded in PR History and will appear on the employee’s W-2 based on the 12:P code.

If the employee has already been reimbursed for these expenses, you will need to create a PR Adjustment Journal entry to enter this information into PR History. Create a PR Journal entry referencing the Other Rate Id on the PR Journal’s Other Tab for the Amount reimbursed. You will also have to specify the Total Other in the PR Journal header. When the PR Journal is saved, the reimbursement will be recorded in PR History and will appear on the employee’s W2 based on the 12:P code.

NOTE: Previously it was necessary to enter 12:P in the Other Rate User 2 field to support Excludable moving expense reimbursements. This functionality is still supported so that it is not necessary to change your current setup if it was implemented last year.

Boxes 07, 08, 09,11,12,14
The W-2 Category field in the Deduction and the Earnings viewers enables the user to write to various W2 boxes by entering a 2-character box number (in the case of box 12, followed by a code) in the appropriate W2 -Category box. Note that only those boxes that can be controlled by an Earnings or Deduction entry to PR History (via normal processing Compute or PR Adjustment entries) are supported.

Note: please consult the IRS website / your firm’s CPA before proceeding. The system does NOT validate that a particular earning or deduction is configured correctly for particular box/code as far is IRS rules are concerned. The W2 functionality is implemented by querying PR History for entries when the Earning or Deduction record’s W-2 Category is completed, and then summing the entries for each particular category. The user MUST ensure and test that the W-2 Category field is configured in such a way as to give the desired result.
