Electronic Work Orders

File should start with EWO.

 

How Work Orders are selected for processing

The ‘Start Taylor Universal Work Order Generation’ function allows the user to enter a date range. The database is then queried for Work Orders matching the following criteria:

The Work Order’s Close Date must fall between the indicated date range.

The Work Order must have a jchstry record having a non-blank jchstry_rfrnce_ar_invce_id value

The Work Order must not already been included in a UWO file

The Work Order must not be ‘Private’. Currently, only Work Orders having a priority of 7 are considered Private.

TYPE 1 OPEN RECORDS:

[ ] Distributor Number (registrykey=DistributorNo)

[ ] Distributor Branch (registrykey=DistributorBranch should be blank if only one)

[ ] Work Order Type (1st character of wrkordr_usr1)

  Valid choices:

  D - Deliveries

  H - Shop Work

  I – Install/startup

  N – Normal

  P – PM

  R – Install/no startup

  S – Scheduled

  U – Pickup

  V – 14 Day check out

  X – 30 Day check out

 

[ ] Model Family (clntsteeqpmnt_mnfctrr_mdl 1st four characters)

[ ] Phase Type (clntsteeqpmnt_usr19 1st charcter)

  Valid choices 1 or 3

[ ] Cooling Type (clntsteeqpmnt_usr20 1st character)

  Valid choices A or W

 

TYPE 2 PARTS RECORDS

[ ] Distributor Number (registrykey=DistributorNo)

[ ] Distributor Branch (registrykey=DistributorBranch should be blank if only one)

[ ] Part Number 1 (invntryitmbyvndr_vndr_prt_nmbr) REQUIRED FOR PARTS TO BILL.

[ ] Who Pays Flag (jb_usr10 1st character)

  Valid choices M, D, or C

 

TYPE 3 CHARGES RECORDS FOR LABOR

[ ] Distributor Number (registrykey=DistributorNo)

[ ] Distributor Branch (registrykey=DistributorBranch should be blank if only one)

[ ] Category Field (lbrrte_unt_msre 1st character)

  Valid choices L (labor), H (1st half hour), T (travel), M (misc), or X (tax)

[ ] Who Pays Flag (jb_usr10 1st character)

  Valid choices M, D, or C

 

Preventive Maintenance WOs Type 3 Record Exception:

Type 3 records are used to report the contents of a Work Order's "Other" tab (as well as the "Labor" tab). The value of "M" (for Material) entered into the Type 3 "Other" records "Category" field. In this release, an exception to the above is made for PM Work Orders. These are identified by the Work Order Type column having a value of "preventivemaint". Type 3 "Other" records deriving from PM Work Orders should have the "Category" field set to "L" (for Labor).

[ ] Service Begin Date (wrkordr_dte_erlst_strt)

[ ] Service Completion Date (wrkordr_dte_mst_cmplte)

[ ] Service Completion Time (wrkordr_tme_mst_cmplte)

[ ] Recall? ‘Y’ if wrkordr_type is 'Call Back'; otherwise ‘N’

[ ] Rewrite? ‘Y’ if wrkordr_orgnl_wo_id is not empty; otherwise ‘N’

 

EXCLUSION RULES:

Inventory, Labor or Other records with a job cost category where the jbcstctgry_usr2 field is set to “Y” will be excluded.

 

ADDITIONAL INVENTORY EXCLUSION RULE:

Inventory items with invntryitm_usr2 field is set to “Y”.
