Remote Access Query: dynamic_content_list

Purpose:

The purpose of the dynamic content list to specify all the RAQ structures that should be in the local datastore. Also, RAQ's that are in this list are inspected in the local datastore and if updates are available for these RAQ structures then they are updated.

The steps that occur at the beginning of the update datastore process include:

1.    The dynamic content list is downloaded.

2.    If there is a item in the list that isnt in the local datastore then it is created as an empty structure and the "headings" for this raq are downloaded.

 Example:
So for example, if I wanted to distribute a new set of code and datastore, the datastore having a new structure I have added called "wo_hammer_list" then I would add "wo_hammer" to the "dynamic_content_list".  I would give everyone with a version of sampro the new "dynamic_content_list" raq, the new "wo_hammer_headings", and ,depending on whether Techanywhere was going to download the new structure or not, the new "wo_hammer_list".

You would increase the datastore_version number of your "next_techanywhere_versions_list" remote access query then start the update process in TechAnywhere.

The new datastore stuctures will be implemented and then the new code will be installed. Then at this point you should have exactly the same state of techanywhere as myself.

 Usage:

Remember to also update the datastore_version number of your "next_techanywhere_versions_list" remote access query
