Registry Employee PDF PIN

Modify Setting of Employee PDF PINs

The option for password protecting an Employee's PR Check, PR Check Detail, and Employee Timesheet reports (SectionPasswordProtect=true) is specified in the report registry and uses the last 6 digits of the Employee's SSN or SIN as the password. This release supports the ability to modify how the PIN is generated. The original default behavior (last 6 digits of SS#) will be used if this registry entry is not enabled.

The following registry modifies the behavior of the SectionPasswordProtect option in the report registries to use an SQL statement to define how the PIN # will be created from fields in the Employee record. The following entry uses the last 4 digits of SS#  followed by the first character of the employee's first and last name.

Id: sys-behavior-EmployeePin  

Setting Key: SQL  

Setting Value: RIGHT(emplye_scl_scrty_nmbr,4) + LEFT(emplye_nme_frst,1) + LEFT(emplye_nme_lst,1)

Note: Other correctly configured SQL statement may be used to set the Employee PIN.  Note that this registry currently impacts PR Check, PR Check Detail, and Employee Timesheet reports that are saved as password-protect PDF's.
