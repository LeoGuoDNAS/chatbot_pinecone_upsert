Special Bonus Posting and Accrual Configuration

This configuration addresses the scenario where an employee doesn’t get wages over and above the union wages (Earnings Group) specified in the Employee record's Default Tab.

Version 9.1.611.057 And Above (Simplified Method)

In Version 9.1.611.057 and above, all that is needed is to set up the employee with an Earnings Id for $.00 on his Employee record - thus eliminating the need for that special Earnings Group for $.01 noted below. This Earnings Id will provide the information needed to derive the posting accounts. This enhancement eliminates the requirement that an hourly rate must be determined for posting values to populate from the bonus amount.  

Versions Below 9.1.611.057

When entering a Bonus amount in the Labor Journal, the following special process is required so that their bonus amount will post:  

Create a special Pay Type record (e.g., OnCall for On Call Bonuses).

In the OnCall Pay Type Record, set the Default Earning Group to an OnCall group that has one line with $.01 as the pay rate.

The OnCall Pay Type should also reference a Deduction Group with no child lines.

 

When the Bonus is entered in the Labor Journal using the OnCall Pay Type, the penny will be ignored when the flat rate Bonus is entered.
