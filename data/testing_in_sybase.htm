Testing and Demo’s in Sybase – number of connections exceeded

In more recent versions of our software, if you are running Sybase locally in a test environment, it is possible to exceed the number of allowable connections (10) under dbeng9.exe if you are running everything – scheduled event manager, report manager, document and SQL logging, etc. When you attempt to open just one more connection (for example, by archiving a document), you may get some sort of error. For example, I got the ‘argument is not a collection error’ message because the connection it tried to open was the one needed to create the document.

Fortunately, the solution is simple – just run the database under dbsrv9.exe instead. You just need to change the name of the executable in the ‘Target’ property of the icon that you use to start the test database.
