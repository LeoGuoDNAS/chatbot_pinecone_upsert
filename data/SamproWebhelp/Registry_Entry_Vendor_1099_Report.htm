Registry Vendor 1099 Report (sys-report-rap1099)

Purpose:

Supports a secondary address line for the and the inclusion of the state EIN.  

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

TIP: Help Completing Vendor 1096

The Report Vendor Payment History may be run to assist with the completion of the Federal Form 1096 which summarizes the 1099 Vendor forms for that year. The Start and End Dates should specify the calendar year reported on.

Related Registry Entries:

See Registry Entry Federal EIN for details on specifying your firm’s EIN number that will print on your 1099’s.

Intercompany Users:
For Intercompany users, the Federal EIN may be completed in the Company records if the Report AP 1099's (RAP1099v82b01 dated 2019-12-31 or later is being used.

Important Fields:

Id:  sys-report-rap1099 (or the function id of the customized report that supports this functionality)

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Intercompany Users: the following options may be entered on the Company Registry tab if the Report AP 1099's (RAP1099v82b01 dated 2019-12-31 or later is being used.

Print Second Address Line

Setting Key: printAddressLine2 Setting Value: true

This registry option specifies that a second Vendor address line will be printed on this revised report.

Include State EIN on 1099

The revised Report AP 1099 (rap1099.rpt) dated 2018-01-09 supports registry entry which will set the State EIN. Note that this report may also be titled Report Vendor 1099's.

Id: sys-report-rap1099

Setting Key: EINState   Setting Value: company's state ein

Exclude GL Accounts (for Credit Card Payments)

Setting Key:  1099exclude  Setting Value: User1 or User2

When this registry is present, the report will look for a "Y" in either the GL Account User1 or User2 fields (glaccnt_usr1 or glaccnt_usr2), and, if present, that GL Account will not be considered for 1099 payments. Payments made to vendors via Credit Card do not count for 1099 reporting (the IRS gets that from Credit Card companies). The use case is you may want to exclude payment from some "bank/payment accounts"....i.e. if you record invoices as paid with a Credit Card liability, those you would want to exclude.
