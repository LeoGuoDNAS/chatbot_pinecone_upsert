Payment Type Viewer

Purpose:

This record is typically used to define how the client says they will pay (e.g., Cash, Check, Account, CreditCard, etc.).

Registry:

Registry to Force Setting the AR Cash Receipts Bank Account from the Payment Type.

Usage:

Referenced in New Call, the Work Order Viewer (for COD work orders) and the AR Cash Receipts Journal.

Credit Card Processing

A Payment Type record is required if PM credit card processing with OpenEdge or Electronic Payments is being used. This Payment Type must be reference in the appropriate registry setting for OpenEdge or Electronic Payments. We recommend setting the Id to CreditCard or EPCreditCard and the GL Bank Account set to the appropriate bank or sweep account for these. This will be used for recurring payment processing (e.g, PM's).

Important Fields:

Payment Type Id:
Uniquely identifies this Payment Type. This field is required.

Description:
Briefly describes this payment type. Optional but highly recommended.

GL Bank Account Id:
This field may be completed for Payment Types associated with COD payments or automated credit card processing with OpenEdge or Electronic Payments.

When a Payment Type is selected on the Work Order COD parent or child tab (or posted from TechAnywhere), the GL Bank Account specified in this viewer will be defaulted in. Note that the Work Order will deny changes to the GL Bank Account on the COD data entry forms when it has defaulted from the Payment Type.

Payment Type User 1-5:
These fields may be used by TechAnywhere assist in filtering records for TechAnywhere access, or for entering any information your company wishes to retain.

Obsolete?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be cleared when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record.
