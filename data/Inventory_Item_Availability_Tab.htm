Inventory Item Availability Tab

Purpose:

Lists current and future availability by location. All fields are maintained by the system.

Important Fields:

Note: this Help Topic lists the standard configuration of all fields that may be included in this record. Note that all fields on this screen are display-only.

Location:
The Location being evaluated for this line.

On Hand:
The number of items that are physically at this location.

- Allocated:
The number of items that have been allocated (picked) for Client Orders.

- Reserved:
The number of items that have been reserved for Shop Orders.

= Available:
The number of items currently available for use:

 OnHand – Allocated – Reserved = Available

+ On Order:
Ordered (via Purchase Order) but not received.

- B/O
Back Ordered for a Client or Shop Order.

= Future:
Future availability at this location:

 Available + On Order –B/O

Promised:
Most recent Date Promised from vendor PO’s for this item that have yet to be received.

LastActivity:
Most recent activity in Inventory History.

Security Id: Supports the use of Security (including Intercompany). This column more clearly identifies the items available for Fill Backorders when Intercompany is enabled.

Obsolete Location:
This field is set to Yes if the Location is flagged as Obsolete. This allows for easy filtering of active locations.
