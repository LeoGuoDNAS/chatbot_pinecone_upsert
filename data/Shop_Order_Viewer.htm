Shop Order Viewer

Purpose:

Create and maintain shop orders used for an item or assembly that must be manufactured for a client or for stocking purposes. For an overview of the steps required to take a shop order from initial created to transferring the manufactured item back into stock, click on Shop Order Overview.

Shop Orders are also automatically generated as part of the Requisitioning process. See Requisitioning Overview for details.

Usage:

This viewer is required to create and modify shop orders for items that must be assembled. The PO Reorder Advice Report will specify which items must be manufactured (and therefore must have Shop Orders created). If a manufactured item is itself comprised of several subassemblies, this information will be automatically entered into the system when the first Shop Order is created. The new subassemblies will appear on the next day's PO Reorder Advice report, which will be used to generate shop orders for the subassemblies. The items on the shop order must be flagged as Released to Pick in order for a Transfer Ticket to be printed.

Note: Omitting Transfers: depending upon the way in which your firm operates, you may choose to employ the Omit Transfers option which simulates the procedures normally completed by printing and entering a transfer ticket.

Shop Order Modes
There are several methods of handling Shop Order creation and closure that are controlled by your customization file and/or registry entries. The most typical mode is discussing in the following help topic: Registry Entry Shop Order Mode For additional information, contact your support representative.

Example:

Create a shop order for gadget1, a manufactured item.

Important Fields:

This viewer is divided into two sections. The upper section includes the header information for the order: Job Id, Job Cost Code, etc.

Upper Section:

Main Tab

Order Id:
If you are running in an inventory-centric (keystone mode), this field will be completed for you when you save this record. If you are not running in this mode, you will need to enter an id. It is recommended that the Shop Order and Job Order Id's be related. For example, if you were creating Shop Orders for Job 1098, you might create shop orders with Id's 1098-1, 1098-2 and so on.

Order Description
Briefly describes this order for internal use.

Inventory Item Id:
The item to be assembled.

Quantity:
The quantity of the item to be assembled. The Quantity is constrained to zero or one if the item being manufactured is serialized. Shop Orders automatically generated via Requisitioning

Location Id:
Identifies the location in your firm where the item will be assembled. Generally, you would create a Location corresponding to the Job Id for the assembly.

Transfer From Location:
Identifies the location where the parts and components for this item are to be transferred from. In general, this will be your main warehouse or stocking location.

Released to Pick
This a sticky field; it will default to the last value you entered when you create a new shop order. If you check this box before entering the Inventory Item Id in the Main tab, all items imported from its BOM tab will be flagged as Released to Pick (you must click the magnify button to see this field). This box may only be checked if you are using the job-centric model and Omit Transfers is clear. If you are adding new entries directly to the detail section (in the lower part of the screen), check this box for all additions to be flagged as ‘Released to Pick’ automatically.

New Srl Nmbr:
This field enables you to assign a serial number to the serialized item you are creating with this shop order. This field may only be entered if the actual serial number is empty. (Note: It may never be filled in the Keystone customization. This is NOT the Keystone mode in the registry entry.) If it is filled in, a new serial/lot master will be created when the record is saved and that serial number will be then appear in the parent window. If it is not filled in, a new serial/lot number will be assigned by the system.

Date Required:
The date that this item is required by. You may enter this date in mmddyyyy format from the keyboard, or press Alt+F7 (or double-click on the field) to display the Calendar viewer.

Date Promised:
The date you promised the customer this item.

Date Entered: Time Entered: Entered By:
These display-only fields are maintained by the system.

Date Closed:  Close Cost:
If this is an Omit Transfer Shop Order, only the Date Closed field should be entered by the user.

In some customizations (e.g., Keystone) these fields should be filled in when this item has been completed, since doing so will relieve the parts/components at the assembly location and transfer the assembled item(s) to the location specified. Doing so closed the shop order (it is not to use the Shop Order Journal to do so). Note that the Date Closed will be used as the Effective Date on the system-generated journal entries. The Close Cost must match the BOM Cost.

Period Id:
This field must completed if the Date Closed is entered; it will be used when the system generates journal entries for the transfers. Note that the Date Closed must fall within the Start and End dates of the accounting period that you have selected.

Instructions Tab

Extended Description:
This optional field may be used to enter user notes on this order. These notes will be printed on the transfer ticket.

Omit Transfers
If you are running in inventory-centric mode, this box will be checked. If so, the system will automatically simulate the procedures normally completed by printing and entering a transfer ticket, allocating the BOM parts in the Transfer From location. When you filled in the Date Closed and save the record, the system will post to GL using information in the Inventory Group record associated with the Asset Account (debit) for the master item being created. In addition, the system will credit the Asset Account of the Inventory Group(s) associated with the raw materials or components. Finally, the appropriate quantity will be transferred to the Location designated in this Shop Order record.

Note: if this box is checked, you cannot flag the Released to Pick field.

Job Tab

This tab is used for job-centric shop order processing only.  It is NOT completed for shop orders generated by Materials Requisitioning.

Job Id:
This required field identifies the Job this order is for. Enter a valid Job Id or press Alt+F7 (or double-click on this field) to select from a list of Jobs. If this Job is associated with a project, the display-only Project Id and Name fields will default from the Job record.

Job Cost Code Id:
The Job Cost Code to assign to this item. Enter a Job Cost Code Id or press Alt+F7 (or double-click on this field) to select from a list.

Job Cost Category Id:
Select a Job Cost Category with a Cost Type of Material or Equipment for this field. This field supports enhanced filtering provided by the Cost Code's Job Cost Category Usage Filter.

Important Fields Lower Section:

When the inventory item being assembled is specified in the header (upper) section of the Shop Order record, its component parts and sub-assemblies will automatically be filled in from the BOM (Bill of Material) tab. The detail lines in a Shop Order can be directly changed, added or deleted if the BOM was incorrect. When a Shop Order is saved, the components will be ordered by Inventory Item Id.

TIP: Update the BOM of the Inventory Item being assembled (if this is a permanent change) and then click the Recompute button to revise the Shop Order.

BOM Tab

Note: the system will prevent deletion of BOM lines where the 'Qty Transfer Ticket Recorded' is greater than zero.

The system validates that a valid Inventory Item Id has been entered on the BOM tab when the user attempts to tab out of the line or Save or Add the record.

The system now denies the deletion of BOM lines whose transfer has been recorded. In addition, changes cannot be made to shop orders with unfilled transfers (the transfer ticket has been printed but not yet recorded).

Users are not allowed to CLOSE a shop order which has un-transferred ITEMS on the BOM and whose status is: TRANSFERS PENDING. If the Shop Order is ready to be closed and the Date or Cost fields have already been set, the user should clear the Date Closed and Cost Closed, delete any lines which have not yet been transferred, and save the work order. Once the Shop Order has been saved and the status set to Transfers Records, a Closed Date and Cost may be entered, and the Shop Order saved again as closed.

Inventory Item Id:
A component or part required to complete this shop order. This field defaults from the Bill of Material of the Item being assembled, but may be changed by the user as needed. The display only Unit of Measure and Inventory Item Name fields will appear when this field is completed. The system recomputes backorder & allocation quantities when component id's are changed in this field.

Qty Req'd
The actual number of this part or component that is required for this shop order. This field defaults from the Bill of Material of the item being assembled, but may be changed by the user if needed.

Validations Note: the system will prevent changes to this field when the 'Qty Transfer Ticket Printed' is greater than zero (Transfer Ticket must  be recorded first). It will also prevent the Quantity Required from being less than 'Qty Transfer Ticket Recorded'.

Qty Alloc:
This field defaults from the Required field.

Qty Avail: Qty Rjcted: BOrd:
 These display-only fields are maintained by the system. The Qty Avail: field shows how many of this item have been reserved for this shop order (by running the Fill Shop Backorders function). The Qty Rejected field displays information on rejected lot-controlled items. Finally, the BOrd field shows how many of these items are on backorder for this shop order.

Description:
This optional field may be used to enter a brief description for informational purposes.

Serial #:
When you double-click this field, the serial/lot number will be filtered to only those for the selected Inventory Item number.

Unit Cost: Extended Cost:
The cost of this component; these fields default from the Inventory Item record, but may be adjusted by the user if needed.

The following fields appear of the detail view of of the line, accessed by clicking the about button.

Released?
Flag this field if these items are to be released to this Shop Order. Double-click on this field (or press the Spacebar) to flag or clear it. Items will not be allocated to this Shop Order unless this field is flagged. This enables you to set up a Shop Order, but delay actually implementing it until the time is right.

Released
The date that these items are released to this shop order. This field will default to today's date.

Released to Pick?
Flag this field if these items should be transferred to the shop order assembly location. Double-click on this field (or press the Spacebar) to flag or clear it. A shop order transfer ticket will not print unless this field is flagged.

Tip: If you check the Release to Pick box on the Main tab before entering the Inventory Item Id in the Main tab, all items imported from its BOM tab will be flagged as Released to Pick (you must click the magnify button to see this field). This box may only be checked if you are using the job-centric model and Omit Transfers is clear. If you are adding new entries directly to the detail section (in the lower part of the screen), check this box for all additions to be flagged as ‘Released to Pick’ automatically.

Receipts Tab

Item Id: Description: Received: Location:
Items that have been received for this order (by recording the Shop Transfer Ticket) will be listed on this tab. Note that ‘Location’ refers to their current location.

This tab also reflects Items that have been transferred from STAGING Location to TRUCK.  In previous versions, this tab only included Items transferred from TRANSFER FROM Location to STAGING.

Supply Status Tab

Item Id: Description: Transferred: Allocated: Backordered: Location:
This display-only tab lists the supply status of the items listed on the BOM tab; how many are allocated, transferred, and backordered. The Location column refers to the Location they were transferred from.

This status is relative to the TRANSFER FROM Location (View Shop Orders / Vshpordr2 only).

Transfer Tickets Tab

This display-only tab is maintained by the system.

TicketNo:
If this field displays as zero, a Shop Order Transfer Ticket has not yet been printed for this item. A positive number indicates that this pick ticket has successfully been printed.

Item Id: Description: Needed:
 These fields indicated how many of each item are needed for this transfer ticket.

Transferred: Date: Serial No: BO Filled:
These fields will be completed when the Shop Order Transfer Ticket has been recorded.
