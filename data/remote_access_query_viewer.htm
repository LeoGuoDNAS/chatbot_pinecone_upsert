Remote Access Query Viewer

Purpose:

The Remote Access Query table is currently used to define and configure the information that will be uploaded to the TechAnywhere web server and ultimately to the wireless tech devices themselves. When the TechAnywhere java script software requests information during synchronize, the Remove Access Queries define what will be retrieved. This information includes the static lists of inventory parts, labor types, active pay periods, and so on.

Important Fields:

Application: TechAnywhere TimeAnywhere
Select the appropriate application for use with this record.

Dataset Name:
This required field must be unique, and should not contain blanks between characters. It must match exactly the dataset name being requested by Techanywhere for a particular purpose.

Version:
This field identifies the application's version to which a dataset (RAQ) applies.  The VW code will only execute RAQs that match the calling JS codes version number.

Date:
This display-only field gets updated whenever a record is added or changed.

Protected:
The 'Protected' Boolean allows an installation to protect an RAQ from being overwritten during an import.

Test Query Button:
The Test Query button should be clicked to execute a Query against the SAM Pro database; the results will be displayed so that you can verify that it is operating correctly.

Query Id:
This field is assigned by the system when the record is saved.

Query Format: SQL Frame Token

SQL:

Queries define the information that will be returned when PHP/TechAnywhere requests this dataset. SQL’s are handled in the following manner:

1. TA middleware asks SAM Pro server for dataset name.

2. If the server finds the dataset, it sends back a query to middleware.

3. Middleware asks server to execute the query.

4. The SAM Pro server sends the results of the query to the middleware.

5. Middleware delivers the results in the correct code to the Java script code in the TechAnywhere wireless devices.

SQL type Remote Access Queries may be customized to limit the amount of information being sent to the server.

Frame:

Frames are typically used to screen labels and static lists such as the labor hours list that lets TechAnywhere users select there time from. When PHP sees a Frame type dataset, it doesn’t have to do the secondary retrieval because the necessary information is already contained in the Dataset Query box.

Token:

Reserved for future use.

Dataset Query:
This field provides the query itself (for SQL’s) or the contents and structure (for a frame). Consistency must be maintained in this field or TechAnywhere’s ability to function properly (or at all) will be severely compromised.

For Frames:

If the Query Format is set to Frame, the Dataset Query must be of the form of an array with the elements separated by commas: [element1,element2,element3] where elements 1, 2, and 3 are replaced by the actual text or headings. Note that literals must enclosed in double quotes. So long as the brackets, elements and commas are correctly entered in the right order, carriage returns and spacing may be used to enhance readability as shown in the example below.

 Example:

This example, for static_labor_hour_list, specifies the hours worked that will appear on the drop-down list that technician may select from when adding time to a work order. Since these are literal values, they are quoted.

 [

"0.5",
"1.0",
"1.5",
"2.0",
"2.5",
"3.0",
"3.5",
"4.0",
"4.5",
"5.0",
"5.5",
"6.0",
"6.5",
"7.0",
"7.5",
"8.0"

]

 For SQL’s

Dataset queries for SQL’s contain an SQL statement or statements that will be executed against the SAM Pro database when requested by the middleware. The results of this query will be picked up on the secondary retrieval.

The Test Query button should be clicked to execute the Query against the SAM Pro database; the results will be displayed so that you can verify that it is operating correctly.

 Example:

The static_work_code_list controls which work codes the will be available to choose from when creating an on-call work order in TechAnywhere:

select wrkcde_id

from wrkcde

where wrkcde_rn<>0

and wrkcde_wb_accss_allwd='Y' 

order by wrkcde_id

In this case, only those work codes whose Web Access Allowed box is checked will be returned.

Notes On Use:

It is strongly recommended that a brief description of each RAQ be included, including any modification notes. A change that seems self-evident now may not be so clear a year from now.
