Registry Where Clause (sys-start-starterfunctionid)

The 'wc' functionality has been implemented generically and is available for use with any function that sends messages to 'FFV >> rangeWhereClause'.  That should include most functions that begin with an input screen that includes a range-type section...reports and starter functions.

Id:  

sys-start-starterfunctionid

Contents: 

wc=rangename=’desiredvalue’ 

Examples:

For sys-start-scrtepmwo

wc=clntsteeqpmnt_undr_cntrct='Y'

7.2 and Up: please use the Contents field in the parent pane of the registry viewer to enter this option.

This functionality was implemented in response to an issue from client 1598 that wanted the Create PM WOs function to skip Site Equipment not tagged as 'Under Contract'.

wc=clntsteeqpmntpm_apprvd='Y'

This functionality was implemented in response a customization for client 1617. The mod enables summer service contracts that must be approved by the customer. If the service is approved, a data entry clerk will check the Approved box visible in their customization of the Site Equipment PM schedule. This registry entry specifies that the Create PM WOs function skips Site Equipment not tagged as 'Approved'.
