Display Problem - Vendor Purchases (and similar) tabs

The Vendor Purchase tab displays summary columns for $ and quantities for the last 3 years. This type of tab seems to not handle the display correctly if column compression is NOT on. This would happen if the Purchases tab exceeded the default column compression values. In this case, the computed columns are only as wide as their contents, not their headings:

The 3rd and 4th columns of the Vendor Purchases tab should look like this:

2008 Qty 2008 $

2  57

0  100

12  254

With the problem it looks like this:

20  200

2  57

0  100

12  254

For the time being, you can avoid this problem by resetting the following sys-ffv-global-settings options to something like this:

maxColumnCountToCompress=20000
maxRowCountToCompress=5000

It is a cached entry, so you will have to edit the registry, and then log off and on again.
