Start Report Import

Purpose:

User Defined reports are stored in your database, not the application image (v.exe). This function makes it easy to import one or many user-defined reports into your database. Your support representative will inform you when you should import new reports (typically, when adding a newly customized report to your database, or updating the existing reports to correspond to a new release).

Validation:

SAMPro's User Defined Reports share field definitions. Importing an older report (with a smaller field size) will reduce the field size for all reports that use it in the database; this can result in other reports throwing errors. In this release, the user is alerted when a field being imported is smaller than that found in the database (unless the global registry setting ReportImportReportFields=Keep is enabled).

Field size comparisons are calculated as follows:

Decimal fields (Precision - Scale) for both fields
Textbox fields (Precision * Scale) for both fields
All other types of fields compare only the Precision.

If the report field definition being imported is smaller that the corresponding report field in the database, the following message will be displayed:

The Field Id: xxxxx now in your database for Report: yyyyyyy

has Field Size: 15 greater than the report you are importing with Field Size: 10

Do you want to import the smaller field  (not recommended)?

We recommend selecting 'No' if this message appears.  Please take a screen capture of this message if it appears, and contact Support with this information so that the problem with the report can be corrected.

Usage:

Scheduled Reports: If a new version of a currently scheduled report is installed, some new fields included in the report (such as displayed buttons) will not be included in the scheduling parameters. In this case, the report will display the text "Unknown" for the value of the missing field. The report should be manually rescheduled so that it will be generated with the new report parameters.

To import reports, complete the following steps.

1. To import your user-defined reports, copy them to a location you can access them from when you are running the Data-Basics software. For example, if you are running the software from c:, you might want to copy the report files c:. If this directory contains earlier copies of reports, we suggest that you move these files to a backup location (c:). If your reports were delivered in a zip file format, remember to unzip them! User-defined reports are distinguished by the file extension .rpt.

2. (Optional). Next, back up your database and export your existing reports.

3. Access the Report Import function. This function may be accessed from your list of Starter functions or from the Folders viewer (the Reporting tab of the Utilities topic). Complete the Important Fields as discussed below.

Important Fields: Range Screen

Reports Directory:
Enter the path to the directory containing the reports (for example, c:\samproreports).

4. When you click the Go button, the system will display a checklist containing all of the reports that it locates in the directory you specified.

Important Fields: Checklist Screen

Do:
The system will consider the contents of the First Version No:. Install Version:, and the version of SAM Pro itself when determining whether or not to flag a report for import. A report will be flagged for import if its First Version Number is the highest allowable for the SAM Pro version running.

Report Id: Report Name:
The internal Id and Name of the report. Note that the Id may not match the report filename, which includes build and version information.

Revision Number:
The revision number of the report; in older custom reports this was set to 999.

CaseNo:
This field displays the internal Case Number (if completed) from the report that is being imported.

First Version No:
This field displays the First Version Number (if completed) from the report that is being imported. This is the first SAM Pro version number the report should be used with.

Install Version:
This field displays the First Version Number (if completed) of the report that is currently installed in the system. This field will display as ‘Not Installed’ if this report is not already in the database, or Unknown if the First Version Number is not completed in the installed report.

Path:
Displays the location and filename of the report.

Custom For:
This field displays the Custom For field (if completed) from the report that is being imported.

 

5. When you click the Go button on the checklist, the system will begin the import. Since reports must be imported in the correct order, the system will take several passes to complete the import.

6. Any reports filed in by IMPORT REPORTS that did not previously exist may require that a corresponding User Defined function be in order to be executed or assigned to user groups. In most cases, the user defined function will be created for you during the import, and it will only be necessary to flag the report in the appropriate User Groups and Functions records.
