Spell Check Setup and Registry Entries

Purpose
This enhancement provides Spell Checking for the contents of selected fields. Spell checking is implemented when the user exits the field.

Components
Spell Checking requires the following components:

1. The SpellCheckFolder.zip file (included with the master images) must be unzipped so that the SpellCheck folder and its contents are added to the Sampro folder (the folder containing the V.EXE file). This folder contains:

The Sentry Spell Check engine (ssce5532.dll)

The Spell Check configuration file (ssce.ini)

The Spell Check help file

The Spell Check lexicons (dictionaries) (*.tlx, *.clx)

SAM Pro Registry Settings

Id: sys-spellcheck

Spell Checking is not enabled unless a ‘sys-spellcheck’ entry appears in the Registry.

Contents: enabled=value

Version 7.2+: Setting Key: enabled Setting Value: value

An optional ‘enabled’ key can be used to disable Spell Checking. If the ‘enabled’ entry value begins with ‘Y’, ‘y’, ‘T’, or ‘t’, Spell Checking is enabled. Any other value disables Spell Checking. In the absence of the ‘enabled’ entry, Spell Checking is enabled.

Additional Contents: checkField=fieldname1,fieldname2, …

Version 7.2+: Setting Key: checkField  Setting Value: =fieldname1,fieldname2, …

Registry entries with the key ‘checkField’ are used to indicate the names of fields to check. Multiple field names separated by commas may appear in entry’s value. Multiple ‘checkField’ entries are allowed.

Example: Work Order

Check the WO Work Requested and Performed, and WO Equipment Work Performed fields.

Additional Contents: checkField = wrkordr.wrk-rqstd, wrkordreqpmnt.wrk-prfrmd, wrkordr.wrk-prfrm

Version 7.2+

Setting Key: checkField  

Setting Value: wrkordr.wrk-rqstd, wrkordreqpmnt.wrk-prfrmd, wrkordr.wrk-prfrm

Example: New Call

Check New Call’s Work Requested field

Additional Contents: checkField=wrkordreqpmnt.wrk-rqstd-nwcll-6x40

Version 7.2+

Setting Key: checkfield Setting Value: wrkordreqpmnt.wrk-rqstd-nwcll-6x40

How It Works
The Spell Checking logic is invoked by a Viewer each time a field is exited. This is the same time that other field validations are performed.

The Spell Checker takes no action if Spell Checking is not enabled by the appropriate Sampro Registry entries or if correct spelling is not required for the field.

If Spell Checking is enabled for the field, the Spell Check configuration file (ssce.ini) is examined for options. These options identify the Lexicons (Dictionaries) which are to be consulted as well as enabling or disabling a number of spelling rules (for example, identifying whether words containing a mixture of letters and digits be spell checked). A section below describes some of these settings.

The spell checker then checks the spelling of the field contents. No visible action is taken of the field passes this test.

If the field fails the preliminary spelling check, a Spell Check Dialog Box is displayed. This box allows the user to accept or correct the spelling or to add words to one of the Lexicons. Note that at this time, there is one global Lexicon for the entire installation.

When the Spell Check Dialog Box is exited, the field contents are set to the corrected value. If the user Cancels the Spell Check Dialog Box, the field contents is left unchanged.

Detail help for all spell checking option may be obtained by clicking the Help button on the Spell Check dialog box.

Setting Spell Check Options
A number of the spell check options which appear in the ssce.ini file are editable via an Options Dialog. Use of this dialog relieves an administrator of the need to edit the ssce.ini file directly.Some of the options that can be set in this file are:

Ignore words containing all capital letters

Ignore words beginning with a capital letter

Ignore words containing any mixture of upper and lower case letters

Ignore words containing a mixture of letters and digits

Ignore words containing no letters.

Ignore case

There are many other options which may be set.

The function Start Spell Check Option Maintenance can be run to access this dialog box from SAM Pro; you may have run the Update Function List to see this function. Help for this function is available from the dialog box.

It is strongly suggested that you only allow access to this function to the system administrator at each installation.  

Maintaining Dictionaries
There are cases when System Administrators may want to modify the contents of a dictionary. This might be done to remove improperly added words or to add a list of words used within the installation. The function Start Spell Check Dictionary Maintenance can be run to access this dialog box from SAM Pro; you may have run the Update Function List to see this function. Help for this function is available from the dialog box.

It is strongly suggested that you only allow access to this function to the system administrator at each installation.
