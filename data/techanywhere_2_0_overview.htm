Data and Functions Considerations

Your data must be set up correctly in order for TechAnywhere to function correctly. A vgsrtry.cus file with TechAnywhere enabled and the number of user licenses specified must be in place in the directory containing your SAM Pro executable file (v.exe).

Payroll Period

If your Technicians will be recording labor hours via TechAnywhere, the Payroll Period that they record them to must

Be Active

Must have the Remote Technician Applet: Ok to Use box checked

Technician Record

On the Remote Access tab:

Web Access must be set to Technician

Remote Device Type set to either CE or XP

The TechAnywhere Prefix field must be completed (this will be used on PO’s generated from TechAnywhere)

The Technician record must point to a valid employee record, and there must be a one-to-one correspondence between the Technician record and the Employee record.

Document Viewer

Document Attachments (to work order, client or equipment records) must be flagged as Web Viewable Tech in order for your TechAnywhere users to see them.

Scheduled Event Manager

TechAnywhere requires that the Scheduled Event Manager be running. The Scheduled Event manager will check for and process status changes and time sheet downloads from TechAnywhere.

Client Site Equipment

A default service job should be specified in all active Equipment records.

Escalation Profile

If one has not already been created, an Escalation Profile should be set up to alert the appropriate individuals that a web log has been created (as a result of the Technician entering a Recommendation in TechAnywhere). We recommend an initial notification to the dispatcher, and a followup if the web log has not been attended to in an appropriate period of time. Registry Settings

The following registry entries should be created/checked prior to implementing TechAnywhere:

sys-email

If you will be running TechAnywhere in a test environment, we suggest using the RedirectAll= option to redirect all email to the individual in charge of the testing. This will avoid confusing regular personnel by sending them test date.

sys-weblog-escalation

This registry entry specifies which Escalation Profile will be attached to weblogs created from TechAnywhere.

sys-techanywhere

This entry is required to assign the applicable Job Cost Category to Inventory written to Work Orders from TechAnywhere. It may also be used to email the contents of the posting logs to the appropriate email address.  

sys-web-access-technician

If you want your dispatch to control the release of work orders to your technicians’ wireless devices, the NewWorkOrderStatus=Pending option may be used. When this option is implemented, all new wo’s will be created with a status of Pending. When the Release to Tech button (on the dispatch board or the work order Technician tab), the line’s status will be changed from Pending to the Default Status for the technician – when this is Unacknowledged, the dispatch will be eligible to be sent to the Technician’s device.

Process Overview

1. When a TechAnywhere User first logs on and synchronizes his/her data, any unacknowledged work orders assigned to that user will appear in the Pending Work Order list.

Purchase Orders associated with work orders assigned to TechAnywhere Techs will be uploaded to the TechAnywhere device if the PO exists when the WO is uploaded. These PO’s will display both inventory and misc. in TechAnywhere regardless of whether or not it has been received. This allows the technicians to see that those items that are already associated with the work order. The PO number, quantities, part numbers and descriptions will be displayed in TechAnywhere.

2. In addition, the Tech may use the On Call Work Order function to create a new work order.

 a. This will be sent as a Message to SAM Pro.

 b. When SAM Pro receives the Message, it will automatically generate a work order.

 c. The next time the tech synchronizes, this new created work order will be uploaded to the wireless devices as a pended work order.

3. When the Tech acknowledges the work order (whether created in SAM Pro by the dispatcher, or as an On Call Work Order in TechAnywhere), a Message will be sent to SAM Pro. The Scheduled Event Manager will process this Message and change the status of the work order to waiting service in SAM Pro.

4. The Tech may change the status of the work order to Travel / Onsite / Waiting Parts as appropriate. These status changes will be delivered as Messages to SAM Pro; the Scheduled Event Manager will automatically pick them up and update the work order in SAM Pro.

5. The Tech may:

a. Add time, materials and inventory to the work order as needed.

b. Create Purchase Orders.

c. Capture customer signatures

d. Create Service Recommendations; that will be transformed into web logs.

e. Add additional existing equipment to the work order, or create new equipment.

6. The Tech may review and submit his time sheet if a pay ending is ending; this information will be sent as a message to SAM Pro and automatically processed by the Scheduled Event Manager as Labor Journal Entries.

7. When the Tech changes the status to ‘Completed’ and synchronizes, all detail lines (including previously unsent labor lines) and purchase orders will be downloaded to SAM Pro.
