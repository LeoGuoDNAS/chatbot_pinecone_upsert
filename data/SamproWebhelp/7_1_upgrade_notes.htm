sys-currency-adjust

JCHenabled=false

If the JCHenabled value is not found or is blank, the default behavior is to enable currency adjustments. Note that this flag is cached the first time it is referenced during a logon session.

If you are upgrading a client from 7.0 or earlier AND EXISTING PO’s have been generated from QuoteMaster or Create PM WorkOrders, either set this registry option to disable the tab mismatch test to JCH, or update the client’s PO data via sql. The currency factor was NOT being set properly in versions 7.0 and earlier for these PO’s. If the user attempts to edit a PO (created in 7.0 or earlier) with a currency factor of 0.0 in 7.1 or 7.2, a Tab Mismatch Error will be displayed, preventing the PO from being edited.

Note: this registry option should not be implemented if foreign currency processing is or will be implemented! Instead, use the sql to reset the currency factor in existing PO’s discussed in the next section.

Material Requistioning

Start Update Workorder Reference in IM History

2931

This starter function should be run for all installations that might have fill backorder records that are associated with a workorder (e.g., using Materials Requistioning).

Fixes imhstry records written by Material Requisitioning prior to case 2331.  It updates the workorder reference fields in IMHistory, and allows requisitioned items to be successfully be imported into the work order.

Bank Rec / AP Journal

SQL to fix History

2979

If customer generates AP Credit memo's for Credit Card/EFT transactions AND uses the Bank Rec journal.

Previously, when the AP Journal posted these transactions to the Check table, they were not written as negative amounts (this is corrected in images 7.1.461.162 and up).

Use the SQL attached to the case to identify and then update existing data that was incorrectly configured on AP Journal transactions generated prior to this fix.

COD Jobs

SQL to fix history

2904

Run to correct JCH postings for COD Jobs invoiced prior to images 7.2.474.035 / 7.1.461.153.

Previously, the Job Invoice tab did not show COD Invoices as received due to failed posting from AR Sales Journal to JCH. This has been corrected for new invoices, but an SQL is need to fix history.

An SQL to fix the failed posting retroactively is available from the case itself (or from Jean). This SQL must be run against database to correct these issues for existing data.

An SQL update script is available in the case to correct existing data.

Purchase Orders

SQL to set the Purchase Order Currency Factor

update prchseordr
set prchseordr_crrncy_fctr = 1
where prchseordr_crrncy_fctr = 0
  and prchseordr_rn <> 0

If you are going up upgrade a client from 7.0 or earlier, AND foreign currency processing is enabled or planned AND EXISTING PO’s have been generated from QuoteMaster or Create PM WorkOrders, you will need to use the sql to reset the currency factor to avoid a Tab Mismatch error when editing these types of PO’s.
