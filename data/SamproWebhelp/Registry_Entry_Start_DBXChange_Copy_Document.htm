Registry DBXChange (sys-dbxchange)

Id: sys-dbxchange

Work Order viewer (XOi link)

To further the SAMPro-TechAnywhere/XOi integration, an Open XOi Vision Info link button has been added to the Work Order's Main tab. When clicked, this button will launch a link/URL to XOi's page, to display XOi "Jobs" linked to the selected Work Order (these are not SAMPro Jobs). The user will be alerted if XOi is implemented but XOi "Jobs" are not associated with this Work Order. If XOi is not implemented, nothing will happen when the button is clicked. This functionality is supported in MS SQL installations.

Implementation Details:

The way this works is that SAMPro will talk with Sampro Secure Communication on a local GlassFish/Payara web server. Sampro Secure Communication will then talk to DBXchange. DBXchange to XOi, and then back down the chain to SAMPro. This will require the following setup:

1. Installation/upgrade of the latest SamproSecureCommunication WAR file. It can be found in the usual location \\shares\FTP Updates\SAMPro Secure Communication\ . The SamproSecureCommunication2.1.zip or above is required.

 2. Installation of DBXchange (this should be in place because this integration is used for TechAnywhere).

Registry:

Id:  sys-dbxchange:

Setting Key: websitename  Setting Value: Full dbxchange URL   (ex. http://SERVER:8080/dbxchange)

Setting Key: server  Setting Value: Server/IP/DNS that hosts SamproSecureCommunication   (ex. SERVER)

Setting Key: port  Setting Value: Port that hosts SamproSecureCommunication     (ex. 8080)

Important: The registry entries for sys-dbxchange are cached and only take effect after logging out of SAMPro and back in.

Push Documents (Obsolete)

This functionality enables you to ‘push’ certain documents to a specified location where they are accessible for

TechAnywhere, Client Web or FMAnywhere users. This would typically be on your firm’s DMZ in order to handle firewall considerations.

Please consult with your support representative on the implementation of this functionality.

Notes:

NOTE: the following restrictions and requirements apply when first implementing this functionality...

1. You must have document archiving implemented. The applicable documents will be copied from the archiving location (behind your firewall) to the location specified in the sys-dbxchange registry entry.
2. Implement the new registry entry sys-dbxchange described below.
3. You may need to run the Start Function List Update function to add Start DBXChange Document Copy to the ‘All’ user group.
4. You may need to assign the new starter function, Start DBXChange Document Copy to the appropriate User Groups.
5. A Scheduled Event entry should be implemented to automate the Document Copy functionality.

The following registry setting is necessary to specify a location to push the documents marked FM / CSR / TechAnywhere viewable in the Document Viewer from their archived location to the one specified in this entry. Note that if this entry is in place, TechAnywhere and Client Web users will look to the directory specified in this registry for attached documents.

Id: sys-dbxchange

Contents: DocumentLocation= (location where you want the documents pushed)

Once this registry entry is in place, the new starter function Start DBXChange Document Copy can be run on demand (from the menu system) or automated (by scheduled event manager) to push the documents to the location specified in the registry.
