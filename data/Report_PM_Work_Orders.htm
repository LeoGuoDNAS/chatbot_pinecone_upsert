Report PM Work Orders

Purpose:

In previous releases, it was very difficult to understand what PM Work Orders would be created from Master Contract until after the fact. This report is a 'predictor': it evaluates the Master Contract data and the data residing in the Client Site Equipment PM Schedule, and lists what PM Work Orders will be created when Start Create PM WorkOrders is run. This report (RPMWRKORDRLST.rpt) must be imported into your database before it can be run.

Range Screen:

Beginning Date: Ending Date:
These fields define the range of Scheduled Inspection Dates that you wish to report on.

Client Id: Client Site Id: Branch Id: Project Id: Work Code Id: Service Category Id:
These ranges enable you to filter the report, just as you would the actual Create PM Work Order function. If you leave these fields blank, Work Orders for the specified Inspection Date(s) for all applicable clients will be within your security level will appear on your report.

Master Contract Id:
If you wish to only report on work orders for equipment assigned to a specific Master Contract enter the desired Id.

WorkOrder Id:
By default, this field is set to = so that only inspections that do not yet have a work order generated will appear on the report. To report on predicted work orders as well as those that have already been created, clear this field.

Report Format:

The report format mirrors the ‘checklist’ screen of the Start Create PM Work Order Function, displaying the following fields for each work order or predicted work order:

Site Id: Site Name:  Service Tasks Id: Est Hours: Work Order Id:
 Equipment Id: Inspect Date: Agreement # PM Invoice:

Important Note: Work Order Id Begins with a #
If the Work Order Id begins with a #, it represents a place holder for a Master Contract PM Work Order that has not yet been generated.

Report Order:

Predicted Master Contract PM Work Orders (yet to be generated) will appear first in the report, followed by predicted PM Work Orders not associated with a Master Contract. If the Work Order Id range was cleared so that actual PM Work Orders would be included, Predicted Master Contract Work Orders will appear first, and actual and predicted PM Work Orders will appear second.

Unless overridden by registry, the work orders in each section will be ordered as follows:

By Client Site Id, Client Site PM Date Scheduled, Work Code Priority, Client Site Equipment Id.
