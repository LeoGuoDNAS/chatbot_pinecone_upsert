Registry Employee Viewer (sys-viewer-vemplye)

Purpose:

Set defaults/configuration for the Employee viewer.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Related Registries:

Employee Id Auto-Sequencing: please consult your support representative for details

See Compute Overtime - No Checks Printed or Posted for details/additional registries for computing overtime when checks will not be generated.

 A Labor Journal Registry setting may be used to include Earnings and Deduction Groups from the Employee Recurring Wages tab for Recurring Labor entries.

Important Fields:

Id:  sys-viewer-vemplye

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Specify the Event Types that may be entered on the Employee Events tab

Setting Key: EventTypes Setting Value: type1,type2,type3,type4 ...

Where the types are replaced by the name of the event you wish to track. If this registry entry is present, the user must enter a value from registry list.  Double-click functionality is also supported.  If the registry entry is not present the user can key in any value in the Employee Events table.

Example:

EventTypes=Hire,Fire,Layoff

SSN / SIN Number Duplicate allowed

There are legitimate reasons for the same person (SS#) to be reused in the employee table...such as rehires or to force multiple W-2's, etc. This option enables you to Allow Duplicate Tax Ids. Setting the new registry entry effectively changes the duplicate Tax Id message from an error to a warning.

Id:  sys-viewer-vemplye

Setting Key: AllowDuplicateTaxId Setting Value: true


Disable Prcnt: Field on PTO Tab (US Clients)

If your firm will not be implementing vacation accruals with the 'percent of wages' option (most US installations), you may wish to disable the Prcnt: field on the Employee PTO tab to avoid inadvertent data-entry in this field:

Setting Key: disabledFieldschildPTO Setting Value: emplyepto.prcnt

Disable Annual (Standard) Hours Field on Other Tab

The new Annual Hours field in the employee record Other tab is used in various payroll calculations for salaried workers. When the field’s value is zero, the system uses the global standard (as set by the StandardAnnualHours option in sys-ffv-global-setting, or by 2080 (if this registry entry does not exist). If your firm will NEVER need to define standard hours by employee, disable this field to prevent users from entering random values.

Setting Key: disabledFieldsParent  Setting Value: emplye.annl-stndrd-hrs

Require that the Payroll Period Per Year is Set

Id:  sys-viewer-vemplye

Setting Key: RequiredFieldsParent Setting Value: emplye.py-prds

Sort Earnings / Deductions By End Date

In previous releases of version 9.1, the Employee's Earnings and Deductions group tabs were automatically sorted in descending order by the End Date of each child line. Due to issues maintaining complex Earnings and Deduction Groups, the default behavior has been changed back so that the lines are NOT resorted when the record is saved. If you wish to have the child lines sorted, the following registry must be implemented: This functionality is also available in the Earnings Group and Deductions Group registries.

Id: sys-viewer-vemplye

Setting Key: NoSortOnChildren  Setting Value: false (default is true to prevent sorting)
