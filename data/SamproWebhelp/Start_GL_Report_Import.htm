Start GL Report Import

Purpose:

GL reports are stored in your database, not the application image (v.exe). This function makes it easy to import one or many gl reports into your database. Your support representative will inform you when you should import new reports (typically, when adding a newly customized report to your database, or updating the existing GL reports to correspond to a new release).

Usage:

To import GL reports, complete the following steps.

1. To import your GL reports, copy them to a location from which you can access them when you are running the Data-Basics software. For example, if you are running the software from c:, you might want to copy the report files c:\glreports\import. If this directory contains earlier copies of reports, we suggest that you move these files to a backup location. If your reports were delivered in a zip file format, remember to unzip them! GL reports are distinguished by the file extension .GRPT.

2. Next, back up your database and export your existing reports.

3. Access the Start GL Report Import function. Complete the Important Fields as discussed below.

Important Fields:

Reports Directory:
Enter the path to the directory containing the reports (for example, c:\glreports\import).

4. When you click the Go button, the system will display a checklist containing all of the reports that it locates in the directory you specified. By default, all GL reports found in this directory will be selected.

5. When you click the Go button on the checklist, the system will begin the import.

6. Any reports filed in by IMPORT GL REPORTS that did not previously exist require a corresponding User Defined function in order to be executed or assigned to user groups.

a. Go to the view menu and select VIEW USER-DEFINED FUNCTIONS.

b. Add a new function for each new gl report imported.
