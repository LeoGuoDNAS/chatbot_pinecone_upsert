Reset SQL to Rerun EFT ACH File for Bank

How To Fixup The Chck Table So That The EFT File For Checks Can Be Regenerated

1. This script should only be run under exceptional circumstances. It should be reported to and cleared by the company CFO / Owner. The user must be able to tell the Date and at least the approximate Time of the payroll run that should be regenerated. Once you have ascertained that this request is valid, the following SQL may be used to reset the Check Table flags so that the Report EFT function may be rerun.

2. Query the ‘chck’ table to identify the exact date and time the EFT file was generated. The appropriate columns are:
 chck_dte_trnsmt_chck
 chck_tme_trnsmt_chck

A sample query is:

select distinct chck_dte_trnsmt_chck,
 chck_tme_trnsmt_chck
from chck
order by chck_dte_trnsmt_chck,
chck_tme_trnsmt_chck

3. Once the appropriate date and time have been identified, update the chck table setting the above columns to their empty values only for those rows with the identified date and time.

The empty value for a date is ‘1900-01-01’. The empty value for a time is ’00:00:00’

A sample update is:

update chck set chck_dte_trnsmt_chck = '1900-01-01',
chck_tme_trnsmt_chck = '00:00:00'
where chck_dte_trnsmt_chck = '2007-05-16'
and chck_tme_trnsmt_chck = '13:37:36'
