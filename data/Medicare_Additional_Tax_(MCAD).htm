Medicare Additional Tax (FICA MCAD)

Version 9.1.611.036 and up supports payroll processing for Medicare Additional .9% Penalty so that it is no longer necessary to predict when the user will exceed the $200,000 and enable / disable the Always W/H flag on the Employee Tax Tab's FICA MCAD entry. However, this code did not account for pre-tax deductions (section 125, AFLAC Pretax, etc) when using the multi-bracketed, annualized tax tables for MCAD. This meant that excess additional medicare tax was withheld for high-wage earners.

Version 9.1.611.050 supports a registry (Refine Medicare Additional Tax for High Wage Earners) that, when enabled, refines the calculations to properly handle pre-tax deductions (section 125, AFLAC Pretax, etc) when computing additional medicare tax for high-wage earners. This functionality will require payroll history adjustments if implemented after the first payroll run of the year: please contact your support representative to set up a time to do so.

Version 9.1.611.050 and Up Setup (Recommended)

Step 1. Payroll History Adjustments

PR History changes are required if the new functionality is to be implemented AFTER the first payroll computation of the year. Contact your support representative to schedule time for assistance.

Step 2: Enable the new Compute PR registry

Enable the registry Refine Calculation for Medicare Additional Tax.  This registry used in conjunction with the FICA MCAD tax table discussed in Step 3 triggers the new behavior. This will ensure that FICA MCAD & FICA MC taxable & exempt earnings should match.  

Step 3. Tax Tables

FICA MCAD - the Base Tax tab must specify the following lines so that the first $200,000 of earnings will be exempt from the 9% penalty.

 

Limit:

Rate:

Add'l Amount:

200,000.00

0.000

 

999,9999.99

0.900

 

Step 4. Tax Authority:

Verify that the FICA MC Authority is NOT set to Annual Compute.

Step 5. Employee Record

In the Employee record's Taxes Tab, employees should have the FICA MC AND the FICA MCAD tax groups flagged as Always W/H all year.

Step 6. Compute Payroll

When Compute Payroll is run, pre-tax deductions (section 125, AFLAC Pretax, etc) will be taken when additional medicare tax is withheld for wages exceeding $200,000.  

 

Versions 9.1.611.036 - 9.1.611.049 (NOT recommended for Version 9.1.611.050+)

This functionality is enabled by changing the FICA MCAD Tax Table as describe in Step 1 below. If you change the Tax Table, you must also complete steps 2-4 below or the additional Medicare tax will NOT be deducted properly.  If you do NOT change the FICA MCAD table, you may continue as before.

Step 2. Tax Authority:

Verify that the FICA MC Authority is NOT set to Annual Compute.

Step 3. Employee Record

In the Employee record's Taxes Tab, employees should have the FICA MC AND the FICA MCAD flagged as Always W/H all year. This configuration can be applied to ALL employees, however this is most conveniently applied at the beginning of the Payroll Year.

Step 4. Payroll Adjustments (if new functionality is enabled mid-year)

If this functionality is to become effective for the current Payroll Year in process then a PR Journal Adjustment will need to be applied for each employee subject to the tax in order to synchronize Taxable and Exempt values pertaining to Tax Table FICA MCAD.

There are 3 possible starting points in such cases that are best identified by running Report PR Summary for the period from the first of the year – date of the last posted PR Check.

#1. Employee has no prior tax history related to FICA MCAD

Option #1: Delay adding or designating FICA MCAD as “Always W/H?” until commencement of Payroll for the next year.  Recommended if employee is not expected to exceed $200,000 for the current year.

Option #2: Create a PR Adjustment transaction to initialize FICA MCAD for YTD activity.  Value for FICA MCAD “exempt earnings” should equal FICA MC “taxable” + “exempt” per the PR Summary Report.  Recommended if employee is remotely likely to exceed $200,000 for the current year.

#2. Employee does have some prior tax history related to FICA MCAD and has not yet exceeded $200,000 in subject wages this year.

MUST!!!: Create a PR Adjustment transaction to initialize FICA MCAD for YTD activity.  Value for FICA MCAD “exempt earnings” should equal FICA MC “taxable” + “exempt” per the PR Summary Report.

#3. Employee does have some prior tax history related to FICA MCAD and has already exceeded $200,000 in subject wages this year.

MUST!!!: Create a PR Adjustment transaction to initialize FICA MCAD for YTD activity.  However an exact prescription for the adjustment that will be required is subject to additional and specific analysis.
