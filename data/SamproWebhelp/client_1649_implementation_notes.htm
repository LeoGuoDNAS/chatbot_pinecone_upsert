Performance Air is using a new v.cus file that enables them to correct the Tax Type Id on WO-LAB lines.  It is accessible from the full-screen view only. This was the scenario encountered...

1. Job is sales-taxable.
2. Tax Group taxes labor but has one labor-type Cost Category as an exception.
3. Separate Labor Rate Ids exist for taxable labor and non-taxable labor.
4. Labor Rate records have default Cost Categories - one taxable, one not.
5. Taxable time is entered in Labor Journal. (they use Tech Anywhere)
6. Costs imported into WO for invoice review and preparation.
7. Labor Rate Id switched to non-taxable on WO-LAB lines.
8. Printed invoice still shows time as taxable.

The problem stems from the fact that during the WO-LAB line edit, the Tax Type had already been set to 'sales'.  When the Labor Rate Id was changed, the Cost Category was also changed to the default from the Labor Rate.  However, the tax status of the new Cost Category wasn't re-evaluated.
