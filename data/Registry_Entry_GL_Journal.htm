Registry GL Journal (sys-viewer-vgljrnllst)

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id: sys-viewer-vgljrnl lst

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Require a GL Entity for Income / Expense Accounts

Setting Key: ValidateGLEntityIE Setting Value: Error or Warning (pick one)

Specify Error if users should be prevented from saving the entry without specifying a GL Entity (if the GL Account is an Income or Expense account) or Warning if the user should simply be warned of the situation.

Deny a GL Entity for Balance Sheet Accounts

Setting Key: ValidateGLEntityBS Setting Value: Error or Warning (pick one)

Specify Error if users should be prevented from saving the entry with a GL Entity specified for a Balance Sheet Account, or Warning if the user should simply be warned of the situation.

Import .CSV file Registry and File Definition

The Import .CSV button supports the import of a .csv file use to generate a GL Journal entry. When the GL Journal header information is entered and the icon is clicked, a browser window will be opened to a registry-defined folder to allow the user to select a .csv file to import as the GL Journal child lines.  The import process includes an option that allows imported lines to APPEND to or REPLACE(default) existing child rows.  The Import loads the child rows. The User is responsible for saving the transaction.

This optional registry setting supports the specification of a default path to CSV files.  

Id: sys-viewer-vgljrnllst

Setting Key: ImportFilePath   Setting Value: full path

This is the directory you want SAMPro to open to for selecting the CSV file; this directory must exist. The Windows browser will open to the this folder when you click the Import CSV button.

CSV File Definition:

Each CSV file must contain a header row which serves as a column map.  There are five required column headers ...

GLAccountId - a valid GL Account Id (required)

GLEntityId - a valid GL Entity Id

Debits - Decimal field, 99999999.99 / -99999999.99 max, no commas

Credits - Decimal field, 99999999.99 / -99999999.99 max, no commas

Description - Text, no commas, 40 characters max.

Column headers can be in any order.  Spelling counts but the header values are not case-sensitive.

Data row values must be consistent with header rows.

File can include additional columns.  All but the five noted above are ignored.

Data values can be wrapped in single- or double-quotes but this is completely optional.  The import process strips them away if it sees them.

Commas embedded in any of the values will cause the import to fail. This is true with or without quote wrappers.

Disable Fields to Prevent Data Entry Errors

The following fields may be disabled to prevent users from erroneously completing fields that were previously hard to see, but are more obvious in due to recent changes in the screen design.

Setting Key: disabledFieldsParent Setting Value: gljrnl.rvrsl: accntngprd.id;gljrnl.clsng-entry

Note: since this is a multiple entry, the system will clear the setting key / setting value fields on this line when the record is saved. The Additional Contents field, however, will be completed properly as disabledFieldsParent=gljrnl.rvrsl: accntngprd.id;gljrnl.clsng-entry and the and the Multiple Settings box will be Flagged.

If needed, create a separate registry entry for the controller or other responsible user without this option (e.g., sys-viewer-vgljrnllst--userid).
