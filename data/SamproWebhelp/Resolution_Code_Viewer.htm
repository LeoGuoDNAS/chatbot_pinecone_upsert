Resolution Code Viewer

Purpose:

Create, maintain or select from your company's list of Resolution Codes that describe how a problem was actually resolved during a service call, or what the original source of the problem was.

Example:

Fixleak, replaced (describing how problem was resolved), or badpart, neglect (describing origin of the problem) etc.

Usage:

Optional but recommended for tracking service and pm issues.

Important Fields:

Resolution Id: (10 character, only . and - special characters)
Uniquely identifies this code. This field is required.

Resolution Name: (40 character)
A brief description of how the problem was resolved (or the original cause of the problem). This field is optional but recommended.

Billing Description: (400 characters)
A more detailed description of the resolution of this problem; optional for invoicing. It will be defaulted into the Work Order Equipment Work Performed when the user selects the Resolution Code on the Work Order Equipment tab.

Resolution Code User 1-5:
These optional fields may be used for entering any information your company wishes to retain.

Obsolete?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be un-done when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.
