Registry Start Workorder Entry (sys-workorder-entry)

Purpose:

This registry entry must be implemented in order to use the Start Work Order Entry function.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id: sys-workorder-entry

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Setting Key: CostCategoryST  Setting Value: costcategoryid

Setting Key: CostCategoryOT  Setting Value: costcategoryid

Setting Key: CostCategoryDT  Setting Value: costcategoryid

Setting Key: CostCategoryInventory Setting Value: costcategoryid

Where costcategoryid is replaced by an appropriate job cost category for Regular (Straight) Time (ST), Overtime (OT), Doubletime (DT) and Inventory Items.

Note that a default cost category for Other charges is not required since this information is stored in the Other Rate record.
