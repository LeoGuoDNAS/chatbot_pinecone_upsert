Setting Key: default

This option controls the default behavior for the defaulting of the blank City and State fields when a zip code (from the zip code table) is entered. Exceptions to this setting may be specified by record.

Setting Key: afunction  

Setting Value: true or false

This option enables you to specify exceptions to the default behavior

Example: enable in all but Vendor

In case 320, the ability to set related fields from the Zipcode table was implement via registry. At some point, the software stopped recognizing the registry, and always defaulted from the zipcode table. This was a problem because you could not turn the functionality off by record. In this release, the 320 functionality has been restored. What this means is that if the registry is not in place, defaulting from the zipcode table will no longer work. Obsolete View Issues – Service Zone

One of our (very old) existing customers reported that prior to the upgrade to version 7.2 they had full access to service zones; after the upgrade they did not. This was caused by the fact that SAMPro once had 2 service zone viewers, vsrvceZne and vsrvcezne (both are named View Service Zones). Notice the "z". Z did not get along with z and caused headaches for programmers when some users were using Z and some users were using z, so the programmers obsoleted Z. This then made it so that "View Service Zones" (the Z version) was no longer on the list.

Solution:

The solution is to have an administrator grant access to the correct Service Zone function after the upgrade.

Support Hint:

1. Before upgrading your customers, access the Service Zone viewer and see if it has an uppercase letter in the function id. If not, you will not have a problem.

2. If it does include an uppercase letter in the function id, determine which user groups have access to View Service Zone.

3. After the conversion, give the appropriate User Group and Functions records access to the Service Zone viewer.
