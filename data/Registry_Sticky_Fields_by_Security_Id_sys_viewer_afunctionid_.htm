Registry Sticky Fields by Security Id (sys-viewer-afunctionid)

Purpose:

This registry option, when implemented, replaces the User Id with the Security Id when managing sticky field values. When this registry entry is first implemented, the sticky fields will initially be blank because their contents are not yet associated with a Security Id. This registry setting should be applied on a viewer-by-viewer basis and within a viewer would apply equally to ALL the viewer's sticky fields.  In other words, the setting does not support selecting specific fields.  Also note that this enhancement is by Security Id, not Security Id by User Id. Therefore, all users within a specific security id would use the same sticky field values for viewers where this option is implemented.

Usage:

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id: sys-viewer-afunctionid

Setting Key: StickyBySecurityId Setting Value: true

Example

The setting for the AR Cash receipts Journal looks like this:

Id: sys-viewer-varrcptslst
Setting Key: StickyBySecurityId Setting Value: true
