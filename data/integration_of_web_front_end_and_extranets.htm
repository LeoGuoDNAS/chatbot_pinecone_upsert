Integration of Web Front End and Extranets

Purpose:

This enhancement implements a web front end mechanism to enable the users of extranets (who are logged into the extranet) to transition smoothly to parts of Sam Pro’s CRM/CSR web interface without having to log in again.

This feature extends the Login web RPC to allow extranet users to pass on a pre-generated Session ID string (and a timeout length), along with the username and password (for authentication by SAM Pro). The idea is that when external users log into their extranet and receive a session id, they can also log that session id with SAM Pro, so it can be used at any time during their browsing session.

Usage:

An external application (within the customer’s extranet) must be able to generate a valid session number and call to SAM Pro’s web front-end using a regular web request (http GET) to the address:

http://domain:port/action?rpc=registerSession&session=[session-id]&user=[username]&password=[password]&timeout=[number_of_seconds]

Parameters:

[session-id]  -- (Required) an alphanumeric session id, generated by the external application.
[username]    -- (Required) a valid Sam Pro web user (either a technician with 'client' or 'csr' web access, or the new style Web User (version 6.2 only)
[password] -- (Required) a valid password for the user, plain text (either from the technician record with 'client' or 'csr' web access, or the new style Web User (version 6.2 only)).
[number of seconds]  -- (Optional, defaults to 10000 if not present) session timeout period, in seconds.

Note that the external application must be able to generate the session id. The extranet log in and the SAM Pro web logins should be identical.

Login In Results:

If the username, password or session id is missing, an HTTP

400 Bad Request
status code is returned, along with an explanation for the error, in the http body.

If the username or password is invalid, disabled or expired, an HTTP

403 Forbidden
status code is returned, along with an explanation in the http body.

If neither of the previous results is returned, the 'registerSession' rpc method then registers the session with the web server (and creates the necessary temporary tables, for technician users).

If the operation is successful, an HTTP

200 OK
status code is returned.

Accessing the SAM Pro Web Interface After Log In:

The Sam Pro web interface can now be accessed at any time (within the timeout period) by the external application by, linking to any valid rpc action.

Example: To link to the Main menu:

http://domain:port/action?rpc=main&session=[session-id]

(the session id has to be one that was previously registered).

Logging Off / Ending the SAM Pro Session:

The external program can either let the session time out (at which point it's removed), or
can call the logout rpc when the user exits the extranet:

http://domain:port/action?rpc=logout&session=[session-id]
