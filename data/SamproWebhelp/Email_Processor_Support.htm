SAMPro Email Processor Setup

See Email Processor Customer Notes for details on using the Email Processor when it has been set up.

Files and Database Entries

The software queries the sys-document-archive registry in order to place documents directly into the Archive Path. When documents are created, the software extracts the email body out as HTML and creates an HTML file in a temporary location. The system then checks for images embedded into the HTML and creates the corresponding files in the same temporary location. The temporary location is then used to create a PDF file which is put into the document archive location. The path to the PDF file is then used as the path for a newly created Document Viewer record. The record is then populated with the keyword locations for the Document List children records. When external attachments are added to the document, the document record's “Other Files” tab is populated with these files.

The software creates entries into the dcmnt, dcmnt_lst, dcmnt_fle_lst, nte_lst, notes, and wblg tables. It generates RN's just as SAMPro does. The entries into all these tables are truncated to the length of the corresponding field.

Installation

  1. Install latest version of program (\\shares\FTP Updates\SAMPro Email Processor). Currently 1.4. Visual C++ runtime must be installed on server.

  2.  Setup config file with customer-specific information.

  3. Be sure the SAMPro Email Processor Services are running (Windows service). If you've made any adjustments to the config file you must restart the services.  

  4. Confirm a valid UNC archive path is set in sys-document-archive. A setting has been added to version 1.3 that will allow an override if a mapped drive is specified in this registry.  

  5. Confirm you will have access/permissions to Local Disk on customer RDP.

**During the installation, a window will appear asking for a Username and Password. This will be the user used to run the service, it will communicate to the SQL Server and must be setup to a user with permissions to the database. The login credentials will be the users WINDOWS or computer login credentials.

FAQ/Tips

General:

-The application should be used for internal purpose only. External emails may be successfully parsed but could very easily interrupt the service until manually cleared from the inbox.

Gmail Settings:

-Enable POP forwarding

-Enable Less Secure Apps
