General Ledger Report Viewer (Old Version)

Purpose:

Maintain older GL Reports.

Usage:

This split-screen function should not be used to create new GL Reports.

Requires GL Report Formats.

Important Fields:

This viewer is divided into two sections. The upper section contains the Id and Name of this report, and also includes buttons that enable you to print and display the report (Show Report) and view your chart of accounts (Show GL Structure). The lower section defines the GL Accounts, Entities and Formats for each line of the report.

Upper Section: GL Report Header

GL Report Id:
Uniquely defines this report. This field is required.

GL Report Name:
Briefly describes this report. Optional but highly recommended.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

Show Report
If you click this button, the system will automatically display the Report Range viewer for this custom report, enabling you to easily print a hard copy of it or simply display it on your screen.

Printing Orientation:
Select Portrait or Landscape for this report by double-clicking the adjacent radio button.

Show GL Structure
If you click this button, the system will automatically display the Report Range viewer for your GL Chart of Accounts, enabling you to easily print a hard copy of it or simply display it on your screen.

Dollar Sign Control:
Check the Show Dollar Signs box (by double-clicking on it) to display dollar signs in your report.

Sign Control:
Enter the character to be used by the report to indicate negative values, typically (.

Font For Report Lines

The following fields enable you to control the font that is used for report lines. If these fields are not completed, *** will be used.

Font Name:
 The name (ex., Courier New, OCR-A, Prestige Elite) of a fixed-width true type font that is available to the users on your system. A list of fonts is located on in the Fonts directory of your Windows Control Panel.

Points:
The size in points of the font: the larger the number, the larger the letter. Note that the point size for the Heading Lines and Report Lines should be the same.

Bold:

Check this box if you want the bold version of the font to be used.

Examples:

Courier New 8pt Bold Courier New 10pt (not bold)

Font For Heading Lines

The following fields enable you to control the font that is used for heading lines. If these fields are not completed, *** will be used.

Font Name:
 The name (ex., Courier New, OCR-A, Prestige Elite) of a fixed-width true type font that is available to the users on your system. A list of fonts is located on in the Fonts directory of your Windows Control Panel.

Points:
The size in points of the font: the larger the number, the larger the letter. Note that the point size for the Heading Lines and Report Lines should be the same.

Bold:

Check this box if you want the bold version of the font to be used. You may wish to distinguish between heading and report lines by making one of them bold.

First Page Text: (lines 1-5)
Optional. Enter up to 5 lines of text that should appear on the first page of this report.

Report Row Id:

Report Column Id:

These fields must not be completed for old-style GL Reports.

Lower Section: Report Detail

This section would be completed for old-style GL Reports only. Each line included in this section will generate a line in your printed report.

GL Account Id:
If this line is being used with a Report Format that specifies GL Account information, enter a valid GL Account Id (or double-click to select from a list). If the GL Format specified for this line includes only text, is blank, or calculates report totals, this field should remain blank. The display-only GL Account Name field defaults from the GL Account record.

Entity:
If you enter a GL Entity Id in this field, only the portion of the specified GL Account pertaining to this entity will be used for extraction formulas on this line. Normally, this field would only be filled in if you wished to individual entities (departments) on separate lines of the report. This field should not be filled in if the associated GL Report Format also specifies a GL Entity; doing so may lead to unexpected results.

GL Report Format Id:
Enter a valid GL Report format Id or double-click to select from a list. The display-only Report Format information will appear when a Report Format Id is selected. If this Format is designed to display GL Account information, be sure that the GL Account field for this line is also completed.
