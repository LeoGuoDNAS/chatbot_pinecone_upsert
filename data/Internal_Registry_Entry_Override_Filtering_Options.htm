Internal Registry Override Filtering Options (sys-viewer-afunction)

All, Active (Open) and Closed Records

Purpose:

This registry option may be used to override one of the default settings for the Open, Closed and All filter options on a viewer-by-viewer basis.

If you specify the registry Id to the user level (sys-viewer-afunction-branchid-userid or sys-viewer-afunction--userid) you can accommodate the requirements of individuals in your system. For instance, an ‘active’ work order would be very different for a dispatcher or a billing clerk or a parts manager.

Usage:

Registry overrides impact filters are used for viewers opened from folders, menus, and right-clicks, and the PICK-LIST’s that are displayed from "double left-clicks". Since these registry entries will interact with pre-filtering done by the system for ‘take-backs’, care must be taken to test the results and ensure that unintended consequences are not obtained.

Dates may be specified (e.g., ‘1900-01-01’).

The token replacement of ^today^ is also supported.

In version 6.2, the user may set filter ranges that will be saved by the system. However, this registry entry will still be supported, and may be desirable in specific implementations.

Important Fields:

Id:

 sys-viewer-afunction-securityid

 sys-viewer-afunction-branchid-userid

 sys-viewer-afunction- -userid (if branches are not implemented)

 sys-viewer-afunction

Additional Contents:

Filter{type}:{display field name}={filter string value}

{type} must be 'All', 'Open' or 'Closed'.

{display field name} must match the name of the range display exactly.

{filter string value} can be anything supported during manual entry of filter values.

Dates may be specified (e.g., ‘1900-01-01’).

The token replacement of ^today^ is also supported.

Example:

In the Work Order Viewer, the system compares two fields, Date Closed and Date Work Completed with the current system date, to determine if the work order is active (open) or closed.

The following values (entered in sys-viewer-vwrkordr--billingagent) override the system values for the Date Closed and Completed settings, and base the filter parameters solely on the escalation status of the work order. Since this entry is designed for a billing agent user (with user id=billingagent), the escalation statuses are set in a corresponding fashion.

FilterOpen:wrkordr.escltn-stts=Invoice Ready, Invoice Review,Wait Paperwork

FilterAll:wrkordr.escltn-stts= Invoice Ready, Invoice Review,Wait Paperwork,Billed

FilterClosed:wrkordr.escltn-stts=Billed

FilterAll:wrkordr.dte-clsd= !! The following lines override the system values

FilterOpen:wrkordr.dte-clsd=

FilterClosed:wrkordr.dte-clsd=

FilterAll:wrkordr.dte-wrk-cmpltd=

FilterOpen:wrkordr.dte-wrk-cmpltd=

FilterClosed:wrkordr.dte-wrk-cmpltd=
