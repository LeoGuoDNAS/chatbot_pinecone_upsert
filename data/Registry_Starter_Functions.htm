Registry Starter Functions (sys-start-sfunctionid)

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Specify Initial Starter Function Settings

The software supports general functionality to allow the user to specify the initial value for Start function input fields and ranges.

Setting Key: InitialValueFor~fieldId Setting Value: fieldvalue

Note that ”fieldId” is displayed on the start input screen when you lay your cursor over it. It is the SAMPro field name, not the database name. For boolean fields (check boxes), the field value of true is for checked, and false is for blank. Note that ^today^ is supported as a token initial value.

If the initial value of a range field is being set, the fieldvalue may be set to a comma-delimited list of values (see the Start Work Order Daily Edit example below).

Example: Start Material Requisitioning
Set the Date as Of field to today, and the Work Order Type field to exclude Preventive Maintenance work orders.

Id: sys-start-smtrlrqstnng

Setting Key: initialValueFor~dte-as-of   Setting Value: ^today^

Setting Key: initialValueFor~wrkordr.type Setting Value: !PreventiveMaint

Example: Start Work Order Daily Edit
This entry sets the parameters used when this function is run standalone (launched from the Starter menu or Workflow Folders. It will also control most initial settings when it is launched from the Dispatch Board (see the Important Note below for an exception).

Id: sys-start-swrkordrdlyedt

Setting Key: initialValueFor~wrkordr.type Setting Value: !PM Invoice (excludes PM Invoices)

Setting Key: initialValueFor~wrkordrlgtype.id

Setting Value: Pending,Waiting Service,Unacknowledged

Important Note: If this function is launched from the Dispatch Board’s Edit Daily WO button, the Log Type Id range will always be set from the Dispatch Board Filter Calls tab instead of the : initialValueFor~wrkordrlgtype.id registry option. Other options set by this registry will be used regardless of how this function is launched.
