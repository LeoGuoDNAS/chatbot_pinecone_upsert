CSRAnywhere Installation

If you are also running the Client Web, be aware that the initial login graphics and stylesheet will be identical for CSRAnywhere and the Client Web. The software will only make the distinction of ‘who’ the user is after the login is complete. Therefore, when planning these graphics, the login screen should be compatible with both.

Installing the CSRAnywhere Interface

Verify that you are running a release that is v6 or higher.

Load the appropriate graphics and default.css files into a subdirectory of SAMPRO (see the graphics files specifications below).

Create a temp files subdirectory

Make the appropriate Registry Entry Client Web Interface record to specify the location of the graphics files and temp files.

Restart the web server

Import the Web reports

Make user-defined function entries for those reports

Java Runtime Environment installed

Latest jar file uploaded

Notes on SAM Pro Records

Creating accounts for your CSR’s:

Technician viewer. Create a new Technician Record for each CSR you wish to give web access to. Be sure to set the following fields, and set the Web Access to CSR.

Web User: Create a Web User record for each Technician; complete the Web User Email and Password field on the Main tab. On the CSR Profile tab enter the Technician Id.

Purchase Order numbers don’t have a pre-defined field in the New Call page in this release. You can set up a P.O.# call script field to handle it (to be entered manually into the work order on work order creation).

Header Graphics File Specifications

This enhancement may be enabled to improve the way the page headers in CRMAnywhere and CSRAnywhere are handled. If CRM was implemented in an earlier release, modifications to your existing graphics will be needed to support the change.

When these functions were first implemented, lower resolution screens were the norm, and the headers were designed be left justified on 600X800 pixel screens, with the HTML generated by SAM Pro centered.

To be more suitable for larger screen, the following registry entry may be used to enable the centering of headers with an appropriated background graphic, and a right-justified graphic to complete the screen:

Id: web-settings
Setting Key: NewHeaders Setting Value: true

  1. The right-hand side of the header should be named right.gif. This graphic will appear at the right-hand side of the header for all pages. It should be 68 pixels high, and wide enough to display the desired graphic.

  2. The portion of the header gif’s containing the title (e.g., Main Menu, View Quotes, etc.) should be wide enough to display the appropriate title, and be 68 pixels high. These graphics will be centered in header.

sign_in.gif
csr_mainmenu.gif
csr_manageleads.gif
csr_reports.gif
csr_newleads.gif

  1. A one or two pixel wide and 68 pixel high vertical slice of the graphic’s background should be saved as slice.gif. This graphic will be repeated as the header background.

  2. A sidebar area - 112 pixels wide by 440 pixels high should be save as sidebar.gif. This will stay the same for all screens. The top 80 pixels may include your logo. The software will overlay menu options on the middle 300 pixels. A message or additional graphic may be positioned on the bottom 80 pixels of the sidebar.

  3. Filenames: binocbutton1.gif
    reportbutton.gif
    starterbutton.gif
    servicebutton.gif
    or button.gif (if using only one button)

  4. Your buttons should not exceed 36 by 36 pixels. If you only wish to use one image, simply copy it and rename for each of the four instances.

  5. Additional Graphics Files

  6. Standard graphics will be furnished by Data-Basics for the labor, call script, materials and other tabs used within this viewer. The filenames begin with csr_ and they must be located in the specified web directory. We do not recommend customizing these files.

The web server should be restarted after the registry change has been made and the revised graphics installed.

Old Header Graphics Files Specifications

The CSRAnywhere Web Interface allows for your firm's corporate branding on individual page headers, a standard sidebar, and buttons. Your graphics must be named as follows (and use the following specifications) in order to work properly.

1. All graphics must use the gif format.

2. Filenames: sidebar.gif, csr_sidebar.gif

A sidebar area - 112 pixels wide by at least pixels high. sidebar_.gif will be used on the login screen; csr_sidebar.gif will be used on all others. These files may be identical. The top 80 pixels may include your logo. The software will overlay menu options below the logo.

3. Filenames: sign_in.gif
csr_mainmenu.gif
csr_manageleads.gif
csr_reports.gif
csr_newleads.gif

These graphics are used for an area at the top right-hand portion of the screen - 68 pixels high. We recommend a width of at least 681 pixels - but you can go wider if needed. This graphic will be left-justified against the side bar. You can create a new graphic for each page, including your branding and the appropriate page title (Sign In, Main Menu, Manage Leads, New Leads, CSR Reports). In the example at the end of this topic, notice that we elected to title each on the left, but you can choose whatever format you prefer.

4. Filenames: binocbutton1.gif
reportbutton.gif
starterbutton.gif
servicebutton.gif
or button.gif (if using only one button)

Your buttons should not exceed 36 by 36 pixels. If you only wish to use one image, simply copy it and rename for each of the four instances.

5. Additional Graphics Files

Standard graphics will be furnished by Data-Basics for the labor, call script, materials and other tabs used within this viewer. The filenames begin with csr_ and they must be located in the specified web directory. We do not recommend customizing these files.
