New Call Create Client / Site / Equipment / Job / Work Order

Purpose:

Take a New Call and create a work order for a new client, site and/or equipment. If you use the special residential template option, this function will also generate Job, Job Cost Code, and Billing Item records for this client.

Example:

Mrs. Jones, who is not in your firm's database, calls in with a service problem.

Usage:

Used to add the basic Client Site, Client (if you are creating a new client), and Equipment information to your database as well as create a work order. If you have implemented residential templates, new Job, Job Cost Code, and Job Billing Item records will also be generated for this customer.

Important Fields:

Client Site Fields

Site Id:
If you are creating a new equipment record for an existing site, enter its Site Id (or double-click on this field to select from a list of Sites).

If this is a new residential customer and the residential template is implemented on your system, the client site residential should be selected; then type over residential with a unique Id for this new site. The Site Id that you enter will also be used to generate the Client Id and Job Id. This required field may include letters, numbers, and the special characters hyphen - and period ..

Site Name:
The name of the Client Site. This field will also be used to default in the Client Name if you click the Copy Site to Client button.

Client Site Alpha:
This field defaults from the Client Site Name, but may be changed by the user if needed.

Site Voice Phone:  Site Fax Phone:
The main numbers for this Site; this information will be copied to the new Client Site record.

Tax Group Id:  Tax Exempt Id:
If you are creating a new site, enter the applicable sales tax information.

PO Required
Double-click on this field (or press Spacebar when the focus is on it) if a Purchase Order is required to schedule service for this Site. A check mark appears in this field when it is flagged.

Store #
The Store # for this site (if any).

Market Source Id:
The advertising source (yellow pages, radio spot, etc.) that generated this call. Double-click on this field or press Alt+F7 to select from a list of Market Sources. This field is optional.

Service Zone Id:
Enter a valid Service Zone Id if they are implemented at your firm, or press Alt+F7 (or double-click) to select from a list.

Work/Ship Address:
The actual (street) address of this Site, as it will appear in the Site Record and on the Work Order.

If you are creating a new Client record, clicking this button defaults the Work Ship (Site) address to the Client Mailing Address further down the screen. Clicking this button will also move the Site Name to the Client Name field.

Contact 1:
Enter the First Name, Last Name, Salutation, Title, Email, Phone and Fax numbers of your primary contact for this site. These fields are optional but recommended: they will be used to generate the Contact 1 fields in the Site and Client records that are being created. The system also writes the Contact 1 Email field to the Client Site Email Address field when the Site is created.

Dispatch Note:
Any special notes (e.g., guard dog, alarm system).

Equipment Fields

Equipment Id:
Required. An Id that identifies this piece of equipment. If you have implemented Equipment Master records on your system, you may double-click on this field to select from a list of your firm's standard equipment designations from the Equipment Master table. If the equipment Id is unknown, enter "unknown"; this field can later be edited when the equipment is clearly identified.

Equipment Name:
This optional field briefly describes the equipment.

Log Form Id: Escalation Profile Id:
These fields default from the template (if applicable) but may be changed by the user. They will be included in the new equipment record, and used to generate metrics/scheduled events on the newly generated work order.

Work Order Fields

Version 9.1.611.060 and up provide reminders for past due payments for invoices made via the New Call 2 COD Job's setup. When users create a work order, the Past Due warning messages appear if their client didn’t pay upon delivery. 

Original Caller:
The individual requesting service.

Work Code Id:
A valid Work Code Id identifying the problem. You may double-click on this field (or press Alt+F7) to select from a list of Work Codes. This field is optional.

Priority
The priority for this call. This field defaults from the Work Code but can be edited: 0 is the highest priority, and 9 the lowest (least critical).

Estimated Hours
The estimated hours for this task. This field defaults from the Work Code but can be edited.

Date Scheduled:
The date on which this call is scheduled. Press Spacebar to enter today's date, or access the Calendar Viewer to enter the date by double-clicking on this field or by pressing Alt+F7.

Time Scheduled:
The time this call is scheduled to start; this field is optional. Enter the time using a military or 24-hour format, or press Alt+F7 (or double-click on this field) to access the Clock viewer.

Call Sequence:
The sequence number for the new work order. This field is particularly useful if you are using the Classic board for Dispatching, as it is used to determine the order in which the Technician should attend to his work orders on a given day. If you do not set the call sequence explicitly in new call, the system will attempt to derive an appropriate call sequence number based on the Time Scheduled and the Technician assigned to the Work Order. In the case of ties, the original work order will retain its original sequence number. See Registry Entry for New Call Options for a registry entry to alter this behavior.

Technician Id:
The technician assigned to this call. This field is optional and can be set from the dispatch board. Double-click on this field (or press Alt+F7) to select from a list of active technicians.

PO Number:
If the PO Required field is flagged, enter the client's purchase order number for this work order.

Work Requested:
Describes the work the customer requested for this task.

Work Order Name:
This optional field may be used to further identify the Work Order on the Dispatch Board and on the Work Order list. If blank, it will default from the Work Code Name.

Service PreventiveMaint Installation Estimate
CSR  Call Back Warranty Quote  Other

The type of workorder; defaults to Service but may be changed by clicking the adjacent radio button.

Client Fields

Client Id:
If you are creating a new Client record, leave this field as is: it will default from the Client Site Id you specified at the top of the screen. If you are creating a new Site record for an existing Client, enter a valid Client Id (or press Alt+F7 or double-click on this field to select from a list of Client Id's).

Client Name:
The Name of the Client. If you are creating a new Client record, it will default from the Site Name you specified at the top of the screen if the field is blank.

Credit Limit
The credit limit for this Client (for reporting purposes).

Branch Id:
If Branches are implemented on your system, enter the Branch (if any) that this Client is assigned to.

Voice Phone:  Fax Phone:
If you are creating a new Client record, enter the main voice and fax phone numbers for the Client. If you are creating a new Site record for an existing Client, these fields will default from the Client record.

AR Terms Id:
Enter a valid AR Terms Id, or press Alt+F7 (or double-click) to select from a list. This field is optional.

New Job for Each Wo (Deprecated)
This field is now deprecated in favor of the Service Compute Job option. If you flag this field, a new job (and its associated Billing Items and Cost Codes) will be created for each Work Order for this new Site.

Material Price Level:
The price level to use for Inventory Items flagged as Use Unit Pricing (1-9).

Charge Interest
For report purposes.

Client Mailing Address:
The mailing (billing) address for this Client. If you leave this field blank, it will default from the Client Site Shipping Address. If you are creating a number job, this field will default to the new Job record. If you are creating a new site record for an existing Client these fields will default from the Client record.

When you click this Client Mailing Address will default to the Invoice Address (in the Job record) further down the screen. In addition, the Client Name will be copied to the Job Name field.

Job Fields

Job Name:
This field defaults from the Client Name when the Copy Client to Job button is clicked.

Invoice Address:
Name on Invoice:
The name that should appear on this invoice. This field defaults from the Client Name when the Copy Client to Job button is clicked.

Attention Line:
This field enables you to specific an attention line for invoices generated for this job.

Street Address: City: State: Zip:
The address that invoices for this job should be sent to. This field defaults from the Client Mailing Address when the Copy Client to Job button is clicked.

Override Tech Status:
By default, this field will be set to the value ‘ default’: the technician status will be set to the appropriate value for that type of technician (referring to the sys-web-access and other registry entries, if present). The ‘ ‘ blank Override Tech Status is processed in the same way as ‘ default’ if the user accidentally cleared the field.

If the user needs to select a special status (e.g., Return Trip), double-clicking in this field (or clicking the Magnify button) displays a quick pick list of valid statuses for the user to select from. By default, all statuses will be display, but a registry entry may made to control the contents of the list: Registry New Call (sys-newcall) .
