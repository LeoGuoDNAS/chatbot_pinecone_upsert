Start Database Validation

This new starter function allows access to several validation routines previously accessible from development mode only. It is the equivalent to executing the lines shown below ...

"--checks for indexes missing from system tables"

FieldTest dbTableValidateIndexes 

"--checks for foreign keys missing from system tables"

(FieldTest dbTableValidateForeignKeys) edit 

These lines continue to be available from Smalltalk's main transcript window for those preferring to execute in development mode.

Important Fields:

Validate Table Indexes

Flag this box to check for indexes missing from system Tables. If any problems are found, an error report will be generated. This report identifies the missing indexes and duplicate keys causing the problem; it will be displayed in a workspace window which may be printed or saved for further evaluation.

Validate Table Foreign Keys

Perform Integrity Check

Box of these fields would typically be flagged to identify any foreign keys missing from tables and perform an integrity check. If any problems are found, an error report will be displayed in a workspace window with may be printed or saved for further evaluation.
