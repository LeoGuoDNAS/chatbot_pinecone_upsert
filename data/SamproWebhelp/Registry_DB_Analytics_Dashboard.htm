Registry DB Analytics Dashboard

Purpose:

This registry setting provides the defaults for launching the web server via Start DB Analytics or via or is automatically started when a user logs on.

Related Registry Entries:

These fields may be overridden by the Dashboard Keys/Values in the User Registry Tab .

Certain Custom PM Workbench analytics depend on knowing which of a job's Cost Codes is designated as the 'Fee Code'. This analytic uses a Job Progress Registry to determine which of the Cost Code User fields is being used.

Important Fields:

Registry Id: sys-dashboard

Email Address for Dashboard Event Manager

Setting Key: EmailFrom

Setting Value: emailaddress@example.com

Specify the Email address the Dashboard Event Manager uses to handle the automatic emails from SAMPro to the email address associated with the dashboard user. [Converted files that couldn’t be viewed online, username / password requests}  

Default Address

Setting Key: AppPath Setting Value: url or ip address/DBAnalytics/DBAnalytics.html

This defines the port that SAM Pro uses to communicate with a SAM Pro Dashboard (e.g., DBAnalytics). This is NOT the same as the Glassfish Server port.

Registry Option to Specify A WebServer Port Range

Each user session must occupy exclusive control over a LocalHost IP: Port to maintain a token based WebService session between DB Analytics and SAMPro to enable launching SAMPro viewers by extension from DB Analytics.

In previous releases, the regimen was to deploy and maintain dedicated port number assignment via User Registry and/or DB Registry settings which works well enough for FAT-Client situations. This configuration got cumbersome and tedious when Terminal Services enters the picture and as a result the feature wasn't exploited and utilized to the degree it might otherwise have been.

In this releases, the web session start code has been revised to alternatively assess currently occupied "ports" that fall within a Registry-defined "Port Range" so that an available port conforming to the range can be dynamically assigned.

Id: sys-dashboard

Setting Key: WebServerPortRange   Setting Value: firstport:lastport

Example:

Setting Key: WebServerPortRange   Setting Value: 50000:51000

Note: the old setting is WebServerPort; it will still be used if the newer WebServerPortRange is not implemented.

Web Server Port

Note that the WebServerPortRange option described above is preferred the this option.  

Setting Key: WebServerPort Setting Value: NNNN

The setting values for these options will depend on your system’s implementation – consult with your support rep for detail.

 Example – running on local host:

 Setting Key: AppPath: Setting Value: http://localhost:8080/DBAnalytics/DBAnalytics.html

 Setting Key: WebServerPort Setting Value: 6011

Default Browser (obsolete in Version 9.1.611.058 and Up)

Important: The following registry option is obsolete in SAMPro Version 9.1.611.058 and above. The user's default browser will be used instead of the registry specified option.

Setting Key: BrowserName Setting Value: executable file

This setting specifies the default browser which is launched when DB Analytics is launched via Start DB Analytics or is automatically started when a user logs on (via the StartDashboard User registry option). This default may be overridden on a user by user basis in the User Registry tab discussed in User Registry Entries. Note that DB Analytics users must have a current version of the browser specified in this setting (or overridden in their User Registry tab).

Example: Chrome (preferred browser for DB Analytics)

Setting Key: BrowserName Setting Value: chrome.exe

Example: Firefox

Setting Key: BrowserName Setting Value: firefox.exe

Example: Internet Explorer

Setting Key: BrowserName Setting Value: iexplore.exe

Important Fields: 

Registry Id: FieldFormViewer.openViewerFromDBAnalytics

Open Quote Manager Instead of Quote Master

This registry allows users to open the new Quote Manager Viewer instead of the Quote Master Viewer. 

Id: FieldFormViewer.openViewerFromDBAnalytics 

Setting Key: @Drilldown@

Setting Value: name=View Quote; description=; value=View Quote Manager; title=; type=SAMPro; column=QuoteId; hidden=true

Note: In 9.1.611.061 and up, this registry is no longer required to open the Quote Manager. Contact support if you want to launch the deprecated Quote Master instead.
