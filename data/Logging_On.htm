Logging On the Software

Purpose:

Select a data source (database) and start Data-Basics' Windows' software.

Usage:

Required to start the application.

Important Fields:

To move the cursor from one field to the next, press the Tab or Enter key or click on the field you wish to select. The Backspace key deletes typing errors.

Data Source:
Enter the name of the data source of your database. Typically, this will be the same as the name of your database.

User Id:
To access your company's database, enter your personal user Id.

Password:
To access your company's database, enter your personal password. Your keyboard entry in the Password field does not display on the screen for security reasons. If you think you have made an error entering your password, press Ctrl+Delete to delete the entire contents of the Password field.

Important Buttons:

After entering completing the above fields, click on the Log In button to start the application (or press Spacebar when the caret is located on this button).

Data-related issues can occur if an Intercompany installation runs with a non-Interco v.cus and vice versa. The system will display a warning on login if the v.cus file and global registry intercompanyenabled=true don't match.  'Special' users will not see the warning to avoid issues with agent type users started via batch file.

WARNING: The Inter Company settings on your v.cus and global registry do not match. Please contact your support representative for assistance.

To exit without starting the application, click on the Cancel button (or position the cursor on the Cancel button with the Tab or Enter key, and then press the Spacebar).

 Login Errors:

The system will alert you if there is a problem logging on as follows:

WHEN the Id is valid AND the Password is valid AND the user's Failed Login Limit is exceeded

THEN the 'User Account has too many Failed Logins' error is issued.

WHEN the Id is valid AND but Password is invalid AND the user's Failed Login Limit is exceeded

THEN also, the 'User Account has too many Failed Logins' error is issued instead of the 'Invalid User Id and/or Password' error.

WHEN the Id is valid AND the Password is valid AND the Logon process has been successfully completed

THEN the user's Failed Logon Count is reset to zero. Otherwise, the Failed Logon Count will remain in effect for all future logon attempts.

TIP: Default Database / DataSource
The Data-Basics' Registry may be used to default in the Datasource for you. For details on doing so, click on Default Database Registry Entry.
