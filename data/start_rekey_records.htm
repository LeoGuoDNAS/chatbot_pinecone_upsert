Start Rekey Records

Purpose:

This function is designed to enable Data-Basics support to "point" all internal references (rn’s) from one record to another. It was originally designed to enable users to combine GL accounts, but may be used (with caution) to aggregate other records.

Usage:

This function should only be used by Data-Basics personnel, never a client.

Make sure a good backup is available; the results of this function are not reversible.

This function may be run in in developmentmode. If it is run in user mode, it may only be run by a superuser.

Special procedures must be observed for tables that are referenced by records with compound keys such as the Client Site. See example 2 below for details.

Important Fields: Range Screen

Table Name:

Double-click to select the the table that you wish to rekey.

Important Fields: List Screen

Id: Name:
These display-only fields identify each record in the table.

Change to Id:
If you want all references to the current line to point to a different Id, enter that Id. Note that this must be a valid Id.

Delete:
If you wish to delete this record after the function has pointed all its references to a different Id, check the Delete box.

You may also check this box to delete a record with no referenences to it (without entered a Change to Id). The system will warn you to if this record has references (and you have not selected a different Id), and will not allow you to procede.

Important Tip: Always tab out of the Delete box if you have selected it.

Click the Go button to proceed.

Example 1: Account Manager

Suppose that two of your Account Managers, AL and JK, are going to retire, and that all of their jobs, work orders, and so should now be handled by JT. To accomplish this, you would run the Start Rekey Records function for AccountManagers. Let’s suppose the list screen resembles the following"

Id: Name:  Change to Id: Delete

AL Allen Letz 

JK Jean Kerry

JT Joe Taylor

TC Tim Carson

To point all current reference of JK to JT, and then delect JK, you would edit the list as follows:

Id: Name:  Change to Id: Delete

AL Allen Letz  JT  X

JK Jean Kerry JT  X

JT Joe Taylor

TC Tim Carson

Note that if you double-click on the Change to Id field, all active records EXCEPT the one you are currently on will be listed.

After tabbing out of the Delete field you just flagged, click the Go button.

 

Example 2: Client Site

Suppose that your customer has inadvertently created 2 records that represent the same site, NW02 and NW005. Further, suppose that both sites include 2 pieces of equipment with the indentical Id’s HVAC and Electric, and both sites have work orders. We will assume that we want to ‘keep’ the NW02 site. Because of the situation with the identical Id’s, the rekey must be done in 2 steps.

Step 1:

Go to the Equipment records for NW005 and change their Id’s to, for example, HVAC1 and Electric1, so they will not be identical to NW02’s equipment.

Step 2:

Run the Start Rekey Records function, changing references to NW005 to NW02 and then deleting NW005. Remember to tab out of the Delete box after checking it!

…..

NW005 Northwest Br 5 NW02  X

NW02 Northwest Br 2 

….

Note: Rekeying Client Site Equipment

Although this function can be used to rekey Client Site Equipment records after you have rekeyed the Sites, we do NOT recommend doing so. Instead, use the Client Site Equipment view to change the Site the Equipment is assigned to.
