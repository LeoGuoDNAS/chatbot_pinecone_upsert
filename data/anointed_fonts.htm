Anointed Fonts

Case 2795 (implement in build 067 of version 7.2) handled a number of issues with font handling in our SAM Pro User Reports. In some cases, the font in the report did not match the report displayed on the SAM Pro report preview, and in other cases the SAM Pro report did not match the PDF’d version. These issues included incorrectly specified font names in the reports themselves, as well as incorrect handling fonts in the XmlToPdf.jar file.

Report Line – Font Validation:

SAM Pro User Reports (not the User Report Definitions that may be generated by our customers) now validates, during data entry, that the Font Face Name is an ‘anointed’ SAM Pro face name (see list below) and is entered correctly. The Font Face Name field may be double-clicked to select from the list of anointed face names.

‘Anointed Fonts’ 

‘Anointed’ fonts will be converted efficiently to .pdf with the newest xmlToPdf.jar file. The following anointed fonts are commonly installed on Windows systems, and may be selected from the pick list:

Arial
Courier New (fixed width)
Lucida Console (fixed width)
Microsoft Sans Serif
Tahoma
Times New Roman

If the customer is already on 7.2, the font names may be updated by running the following from a workspace:

Wsql dbCommit.

You can check the font names before and after with the following SQL statement:

select count(*),rprtlnelst_fnt_nme from rprtlnelst
 where rprtlnelst_rn > 0
 group by rprtlnelst_fnt_nme

New xmlToPdf.jar File:

We will begin to distribute the updated xmlToPdf.jar file with the image once anomolies in the Quote form have been handled. The new .jar file handles bolding correctly, and manages common font face errors in SAM Pro User reports.
