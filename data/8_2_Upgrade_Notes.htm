8.2.545.081 (2015-10-12)

AP Journal (Important Note)

Main Tab - GL Entity
In V8.2, this field also sets the GL Entity for manual checks, EFT and CC/DR Card transactions when posting to a Bank Account. Previously, it only assigned an Entity to the AP Control Account and Discounts Earned. Because of this change:

If InterCompany IS enabled, set to a Corporate GL Entity for the company.

If InterCompany is NOT enabled, this field should be BLANK for manual checks, EFT and CC transactions to avoid Bank Reconciliation issues.  

Do I Need an Updated v.cus File?

When a system upgrade triggers a data-conversion, the v.cus is automatically upgraded, too. If there is no data conversion, the system does not update the v.cus. The most recent 8.2 change requiring a revised customization was an Inventory Adjustment Journal enhancement which went in as of 2014-08-05 - if the v.cus is older than that date,a refresh wouldn’t hurt.

For installations using one of the standard v.cus files (WinII, CA, etc.), relabel the current v.cus as v.cusOUTdate, and then simply pick the new version of the applicable file in the zipped release, and RENAME it as v.cus. If the client has their own special customization, a revised version may be obtained by from Support.

Open SAMPro Attachments in Windows Picture Viewer:

This release corrects a problem when certain users couldn't open picture files (jpg and jpegs) in Windows 8 running on Server 2008 R2 64 bit (may also be present in other environments).  In some cases a registry entry may also be required; see sys-openapp-commands.

Java 8:

Some users have experienced an issue where the Java 8 upgrade did not remove Java 7. When Java 7 & 8 are installed at the same time errors occur. Un-install Java 7 to correct this problem.

Converting Files to PDF's:

A problem has been reported converting SAMPro reports to pdf documents by using the Save As PDF function. The message 'you are not permitted to run this function' was displayed, but in reality the call to see if java existed was returning a 'file not found'. The work-around to solve this problem is to place the path to java in the Window's Path environment variable.

Important Autoemail Registry Change!

If any registry entries with autoemail=true AND autosavewhenprinted=true enabled, the following line must be added to the appropriate registry in order for the auto-mail to occur:

Setting Key: autoemailwhenprinted   Setting Value: true

Registries which may be impacted:

Service Invoice (sys-report-serviceinvoice): This entry requires autoemail (to change the address on the Invoice to the e-mail address) AND autoemailwhenprinted to do the e-mailing.  

Vendor Expiration Report (sys-report-rexprdinsrnce); if autosavewhenprinted is enabled, add the autoemailwhenprinted line and disable autoemail or two copies of the report will be sent.

Customer Statement (sys-report-rcuststmnt): if autosavewhenprinted is enabled, add the autoemailwhenprinted line and disable the autoemail or two copies of the report will be sent.

Multiple Tech Lines Per Day Per Work Order:

The system now allows multiple lines on the Work Order Technician tab with the same Tech Id and Date but a different Time Scheduled (compatible with TechAnywhere 3.0.048.006+). A change to the Work Order indices is required for current 8.2 installations; this may be done by dropping the revlev and letting the database convert up. Contact Data-Basics support for help.  

DB Analytics/FMAnywhere:

If these modules are used, verify that your GL Reporting Periods are correctly defined for at least the next full year: they are required for date-related functionality. If a ‘beginning’ period was used for loading data, the Start Date should be set to the beginning of the loading year, not the ‘blank’ date (01011900) before upgrading.

The GL Account Type record includes a new Category field. SAMPro will continue to use the old Account Type Category field, but DBAnalytics used the new field. Support or a designated user should categorize the new field in all GL Account Type records if DBAnalytics is implemented.

New XmlToPdf.jar

The revised file dated 2015-07-08 corrects a problem in the previous version (2015-06-22) which failed to save PFD's properly if the save path and/or File name contained SPACES.

WebHelp:

See Customer Note 1.7 for details on configuring Internet Explorer to run SAMPro WebHelp correctly.

Required Files:

The change.log, recover.log, v.bak and master~pushbuttons.obj files must be in the same directory as the image (v.exe) file.

Colors:

To fully support Bitmap Skinning, display settings must be set to 24 or 32 bit color. If the Color Quality is set to 16 bit (High Color), the background and mouse over color changes will not appear.

VM Memory:

SAM Pro’s Smalltalk virtual memory was expanded to 600MB to better support the running of extremely long reports and large record viewers without encountering out of memory errors.
