Internal Registry Job Number Auto-sequencing (sys-ffv-global-settings)

Purpose:

Auto-number jobs created from the Job viewer and the Clone Job button in the Job viewer.

Usage:

Important Implementation Notes: During data conversion, the system will create a new database table named nextjb to that enables the implementation of the auto-sequencing for job cloning.  By default, this counter will be Initialized to100000. Before implementing this registry setting, PLEASE REVIEW YOUR CLIENT’S JOB NUMBERING SCHEME FOR ALL JOBS.

As of the 2/7/2007 image, the software is smart enough to iterate to the next Id when a collision with an existing would occur. Nevertheless, the auto-sequence id’s should be started at an appropriate number for the client.

Please see Auto-sequencing Issues in Job Clone and New Call Backscreen for important details on this topic.

Important Fields:

Id:

sys-ffv-global-settings
Contents:

AutoSequenceJobId=true
If this option is set to true and the user doesn't provide a Job Id (in the Job Viewer or during the cloning process in the Job viewer), then the auto-sequencing should be used to provide the Job Id. Note that this registry setting does not impact the auto-numbering scheme used on the back-screen of New Call.
