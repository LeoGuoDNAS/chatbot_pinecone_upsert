Inventory Item Locations Tab

Purpose:

Lists the location(s) and quantities of this item. Note that Shop Orders are considered a location.

Example:

At the Cleveland location, 5 items on hand and 3 items on order.

Usage:

Used to track inventory location, quantity and order status. The Reorder Level and Cycle fields are maintained from this tab.

Use with Inventory Restock

The "trigger" mechanism is that when "on hand" quantity is less than the "reorder level", an item will qualify for replenishment, and the Quantity to PICK will be however many are required to bring the "on hand" quantity up to the "Order Up To Level.

for instance...if 2 are on hand
                  3 is the reorder level
                10 is the order up to

then an item would qualify for 8 to be picked.

Important Fields:

Note: this Help Topic lists the standard configuration of all fields that may be included in this record. If you are using a customized system, the fields displayed on your screen may not exactly match the fields described in this Topic.

Location Id
The location of this item on hand (e.g., cleve) or part of a shop order (order Id); press Alt+F7 or double-click on this field to select from a list of locations. New location lines will be created as needed by the system when purchasing or receiving an item.

Note: Obsolete Locations are displayed at the bottom of this tab. The Obsolete flag appears as a column on the far right of the tab.

Pick Seq:
The shelf/bin position for items on hand at this location.

Reorder Level
The Level to which this item should be Reordered. This field is used for the Reorder Advice Report.

Order Up To
This field is used to determine the quantity needed by the new Start Generate Purchase Order function. If this field is not completed, the Reorder Level will be used to calculate the Quantity Needed instead.

Note: Setting the Reorder Level and Order Up to Fields Automatically

Start Compute Corporate Metrics may be used to automatically set these fields based on historical usage and user-specified values. See Computing Corporate Metrics for details.

On Hand: In Test: Rejected: Pick: Shop BckO: Cust BckO: On Order:
These informational fields are maintained by the system as PO's are released or changed, items entered in the Inventory Receipts Journal, Shop Orders generated, etc. The In Test and Rejected columns pertain only to lot-controlled items.

Note for COP/Shop Order Users:
The quantity available at a location = On Hand – Pick. Note that Pick / On Hand fields are recomputed when a Client Order is invoiced, not when the pick ticket is recorded. 'On Hand' and 'Pick' are also updated when recording Shop Order Transfer Tickets.

Average Cost - Last Cost:
By default  the Average and Last Cost calculation is no longer applied when loading the viewer. To improve performance, the Average and Last Cost will only be calculated on each line when clicking "Show Computation" on the detail view.  The Average and Last Cost columns will display as "Needs Compute" until they are individually computed.

Note that the overall costing information for the Inventory Item is displayed on the Pricing/Costing tab as usual.

See Registry for Location Tab Average and Last Cost Calculation to restore the previous behavior and show the Average and Last Costs.  

See Inventory Item Unit Cost Calculation for a more detailed description of how unit costs are determined.

Turns:
This statistic will be displayed by location if your firm has implemented the computation of corporate metrics. See Computing Corporate Metrics for details.

Obsolete:
This field will be flagged if the Location referenced on this line is Obsolete; Obsolete Locations will appear at the bottom of the tab.

Click this button to display the calculations used to derive the Average Cost at this location for this item.

Custom Systems Only:

Purchase Manager:
Identifies the purchasing manager for this location. Enter a valid Id or double-click to select from a list.

Date Last Counted:
The date of the most recent physical inventory in mmddyyyy format.

Cy
Cycle. An optional, two-character field describing the inventory cycle (mo, 30, yr ) for this item. This field is used in the Inventory Count report.

Ratio  Stock UM: PrchseUM: Qty/Yr:
These memo fields will appear if your organization maintains different Stocking and Purchase quantity (e.g., purchase by the box and stock by the item); they are for reporting purposes only. The Ratio defines the mathematical relationship between these two fields. For example, if the Stock Unit of Measure is "each" and the Purchase Unit of Measure is "box-12", the Ratio should be entered as 12.

LSN Sent:
If this custom field appears on your screen, it should be completed by the user went the Low Stock Notice is first sent.
