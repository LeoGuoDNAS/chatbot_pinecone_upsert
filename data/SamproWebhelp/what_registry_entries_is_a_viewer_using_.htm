What Registry Options is a Viewer/Function Using?

If you are running in development mode in the more recent versions, you can easily tell which registry entries a viewer or function is responding to. To do so:

1. Run in Development Mode

2. Be in the record or function of interest – preferably logged on as specific user (remember that registry entries can be by user).

3. From the menu bar at the top of the window, select:

Help >> DataValues

4. An Inspecting: Dictionary will appear. Double-click on myRegistry.

5. Another Inspecting: Dictionary will appear: the registry options will appear on the left. If you select one, it’s value will appear on the right.

6. If you need to figure out the registry id the option belongs to, use the Search functionality of your Support Help to find the registry option text string.

 Example:

In New Call, one of the registry option you might encounter is ‘defaultstartdate’. To reverse-lookup the registry Id, go to Help and click the Search tab. When it gets through compiling its search database (if this is the first time you have done it), enter the string defaultstartdate in the search field. Help will display the reference to Registry Entries for New Call in the lower left-hand pane.

sys-ffv-global-settings entries are NOT included in these inspection viewers. To viewer a users global settings and other ffv items, log on in development mode, and select Help >> Inspect FFV Settings.
