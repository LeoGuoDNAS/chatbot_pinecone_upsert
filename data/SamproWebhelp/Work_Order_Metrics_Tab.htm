Work Order Metrics Tab

Purpose:

This tab displays the system metrics recorded by the technician when working on a specific piece of equipment.

When a PM Work Order is created, the Log Form referenced in the Site Equipment record will be written to the WO's Equipment tab; and will be used to default the Work Order Metrics tab.

When a Work Order is created in New Call, the used may override the Log Form Id from the Equipment record. The Log Form Id selected in New Call will be written to the new WO's Equipment tab, and will control the WO's metrics.

Usage:

The Log Form Id is referenced in the Client Site Equipment record. If a Work Order is created for equipment that has its Log Form Id field completed, the Log Form detail will automatically be imported to the Metrics tab of the Work Order if the Technician’s Web Access is set to ‘None’. These metrics lines may then be completed manually by the dispatcher or service coordinator.

If you create a Work Order for a Tech with the Web Access set to ‘Technician’, the Metrics tab will be empty when the Work Order is first created since this information will be completed via TechAnywhere. TechAnywhere retrieves the Metrics (Log Form), from the Work Order Equipment tab. The Metrics readings would then be completed by the Technician using the TechAnywhere device. TechAnywhere Post, the Work Order’s Metrics tab will be updated with the information that the Technician entered.

Note: If an Equipment Id associated with Metrics is deleted, only those lines on the Metrics tab that have been recorded (a date and/or value has been entered) will be retained. If the same Equipment Id is re-entered, the system does not restore blank Metrics lines that were not recorded. To restore these lines, click the Sync Metrics button.

Cycle Code Filtering

The Log Form specified on the Client Site Equipment record may include lines/metrics that should be recorded at every service or maintenance call, as well as some that need only be recorded on (for example) system startup or installation. Both Sam Pro and TechAnywhere filter (via the specified "Cycle Code Filter" associated with PM schedules and service Work Codes) the lines written to the Work Order Metrics tab.

When a Service Work Order is created in SAM Pro’s New Call, the system will check the Cycle Code Filter in the Work Code; if this field is completed, only those log form line metrics whose CC: field corresponds to the filter or is blank will be written to the Work Order (for Techs without remote access) or uploaded to the Technician’s device.

When a PM Work Order is created, the system will check the Cycle Code Filter in the Client Site Equipment PM Schedule (or the Master Contract Schedule). If the Cycle Code Filter in the schedule is blank, the system will use the Cycle Code Filter in the Work Code instead. Only those metrics whose CC: field corresponds to the filter or is blank will be written to the Work Order (for Techs without remote access) or uploaded to the Technician’s device.

Important Fields:

Equipment Id: Equipment Name:
These display-only fields are maintained by the system, and specify the piece of equipment for which the reading should be/was taken.

Metrics Date:
The date that the reading was taken.

Value:
The value of the reading itself.

UM:
The unit of measure this reading should be taken in (psi, volts, etc.).

Log Form Header: Log Form Line:
These fields default from the Log Form Record and define the reading that should be entered in the Value field.

Source Rn: Tech Id:
If these fields are completed, this record/line was created/revised from TechAnywhere.

Important Buttons:

Sync Metrics
This new button will import missing metrics (e.g, if work order was assigned to a remote tech, then reassigned to a tech with web-access = none) and tidy-up orphaned lines (if going from a non-remote tech to a remote tech). Cycle Code filtering is based on the Work Code on the Equipment tab (if present) or the Work Code in the Work Order header if the Equipment tab’s Work Code is not completed.

Note that if you add a piece of Equipment, it's log form defaults from the Equipment record, not the override the user may have selected for this Work Order in New Call.
