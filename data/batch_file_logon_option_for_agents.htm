Batch File Logon Option for Agents

Batch File Logon Option

Agents for ReportAnywhere, TechAnywhere, and EscalationManager need to be restarted via the appropriate batch file each time the PC they're running on is rebooted. Putting the batch files into the Windows Startup directory enables these agents to be restarted automatically each time the machine is reboot (advertently or inadvertently - i.e. MS Update reboot) without user/administrator intervention. However, in previous releases, SamPro displayed the meesage "user agent**** is already logged in....." requiring operator intervention to click "Ok" to disconnect the previous session.

This release implements a parameter that may be added to the existing batch file start up logic that automatically kills existing sessions and allows the logon to continue. When user adds the following option in the startup script the Active Session Found message will not display and active sessions will be removed:

-omitUserPrompt
