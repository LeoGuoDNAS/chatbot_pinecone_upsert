Client 1617 Implementation Notes

5/12/2005

1617' custom screens and registry entries are designed to enable the processing of their summer service mailing and reporting.

Client Site
------------
A custom field labeled "Send 2nd Mailer?" was added.  In the 1617 cus file, this field is located on the Service tab of the Client Site viewer.  If I remember right, Tim indicated they had some customers to whom they never send a second mailer.  This provides a way to eliminate specific sites from the second mailing.

Client Site Equipment PM
------------------------------
A custom field labeled "Approved?" was added.  In the 1617 cus file, this field is located on the PM Schedule tab of the Site Equipment viewer.  They should check this box when a customer returns the summer service mailer and indicates their approval for one or more of the proposed services.

For sys-start-scrtepmwo use one of these entries:

wc=clntsteeqpmntpm_apprvd='Y' or InspectionMustBeApproved=true

This registry entry specifies that the Create PM WOs function skips Site Equipment not tagged as 'Approved'.


Summer Service Custom Report
--------------------------------------
Jeff Boyd wrote a custom report to 1617' specifications.  It makes use of the previously mentioned custom fields.  It also requires that they identify the summer services using these specific Service Task Ids...

       FC (Fireside Cleaning)
       WC (Waterside Cleaning)
       BC (Burner/Boiler Checkout)
       MC (Mid-season Check)

Jeff has report samples and can provide additional details concerning the processing if necessary.

Create PM Work Orders
---------------------------
To prevent this function from generating WOs for unapproved services, they need the following registry entry.

        Id: sys-start-scrtepmwo
        Contents: InspectionMustBeApproved=true

Hope this helps.  You might also discuss this with Tim C.  He probably remembers some additional pertinent details.
