Registry Adobe Reader (pdfreaderpath)

Purpose:

In earlier releases, in order for SAMPro to automatically display or print a PDF file, the location of Adobe Acrobat Reader needed to be entered in the SAMPro registry. This entry is typically not needed current releases.

Usage:

This functionality is currently implemented for MSDS (Manufacturer’s Safety Data Sheets) printing.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id:

pdfreaderpath- -userid

pdfreaderpath-branchid

pdfreaderpath-securityid  

pdfreaderpath-usergroupid

pdfreaderpath

The first formats let you specify the path to the adobe reader by branch id and (optionally) user. If this entry is specific to a user, the branch must be specified if a user belongs to a branch. If branches are not implemented, two dashes (not separated by a space) must precede the user id.

The fourth format enables you to specify the location by security id, and the fifth by user group id.

The last format would apply to all users unless another registry entry was created for their security or branch/user id.

Setting Key: path Setting Value: pathtoreader

where pathtoreader is replaced by full path to the Adobe (Acrobat) Reader.

Examples:

Setting Key: path Setting Value: c:\program files\adobe\acrobat 6.0\reader\acrord.exe

Setting Key: path

Setting Value: \\myserver\ program files\adobe\acrobat 6.0\reader\acrord.exe
