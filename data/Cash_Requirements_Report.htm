Cash Requirement Report

Purpose:

This report would be run to view vouchers payable. This report can be generated to list all vouchers not already paid, or by selecting specific vendors and/or next pay dates. Invoices will appear on the Cash Requirements Report even if the only outstanding amount is unpaid retainage. Only vendors with outstanding balances will be included on the report: if a vendorâ€™s credit balance exceeds his outstanding balance, he will not be included on the report.

This report will total all vouchers selected.

Date to Pay Column

If a number sign (#) follows the date in the Date to Pay Column, the invoice requires approval.

Usage:

The cash requirement report lists vouchers not paid either under past due, current due and future due columns. At the end of the report, the vouchers in each column will be totaled to show total amounts past due, current to pay, and future due. By selecting the start date and end date, you are selecting your current period. All vouchers with dates to pay before the start date will be under the past due column. Likewise, all vouchers with dates to pay after the end date will fall in the future due column. This report in combination with the Date to Pay List allows you to select the vouchers that you want to pay.

Example:

Select the Start Date of April 1, 1999 and the End Date of April 15, 1999. All vouchers that have a date to pay from April 1 to 15, 1999 will fall under the current column. A voucher with the date to pay of March 31, 1999 will fall in the past due column and a voucher with the date to pay of May 2, 1999 will fall into the future due column.

Important Fields: Upper Sections

Report Options:
The Report Options selector pane controls how the report will be output. Click Report Options to learn more.

Report Ranges:
The Report Ranges selector pane enables you to select a customized range for the report. When you click on a range, the criteria defining it will appear in the ID field(s). Creating a customized report range is discussed below. Notice that you must name a custom range in order to save it.

Important Fields: Lower Section

Range:
Enter a unique, alphanumeric name to save and identify your customized report Range. Ranges are defined by the contents of the ID field(s).

Hint: You can create user-defined report ranges that are available to all users with access to that report. To do so, the first character of the named range should be an asterisk. Named ranges beginning with an asterisk will appear on all users' report ranges options so that they can be selected (but not altered) by any user with access to that report. Only the user that created the report range is allowed to alter it.

Start Date: End Date:
These are the settings that define the Current column of the report. For example, suppose the Start Date is March 1, 2002 and the End Date is March 31, 2002. Items that appear in the Past column of the report were due before March 1st., 2002. Items in the Future column of the report are due after March 31st, 2002.

Invoice Description: Print?
If you flag the Invoice Description Print box when running this report, the Invoice Description from AP Journal Vouchers will be printed when the field has been filled in.

Invoices With Future Date to Pay: Skip?
If you flag this box, invoices with date to pay that is defined as Future (e.g., whose Date to Pay is after the End Date) will not appear in the report.

Held Invoices: Skip?
If you flag this box, invoices that require approval will not appear in the report.

Vendor ID:
Fill a vendor ID, or a range of vendor ID's to select only certain vendors. If you leave this field blank, all vendors will be selected. Several vendors can be selected by separating them with a comma, or select a range of vendors by placing an underscore between two vendor numbers.

Next Pay Date:
Select a particular pay date or a range of dates to pay to run this report for selected pay dates. The date should be put in the yyyy-mm-dd format. If putting in a range of dates, use the yyyy-mm-dd_yyyy-mm-dd format. If no dates are entered in this field, all dates will be selected for the report.

GL Account
An AP Control Account may be entered to limit the report to a particular type of payables (trade payables, other payables, etc.).

GL Entity
Enter a GL Entity Id to limit this report.

Currency Id:
If multi-national currency is implemented on your system, enter the Currency Id that this report should be run for. Only those vendors whose Currency Id matches the one entered in this range will be included.

Note: In 9.1.611.060 and up, this update allows the total of the Current column of the AP Cash Requirements Report to take partial payments into account specified in the Select Invoices for Payment screen. The values will remain sensitive to the partial payments specified in the Select Invoices for Payment function.

Power-User Reporting Tip:
You may also use the comparison operators >,<, = and ! in your report ranges to control the information output to your report. Click Comparison Operators for details.
