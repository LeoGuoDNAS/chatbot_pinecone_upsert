Registry Control Initial Window Size and Position

Purpose:

This functionality applies to record Viewers, most initial Starter screens, and Report ranges. The size and position settings in this registry will be overridden is the user has individually set the size and positions via the File menu: Save Viewer Attributes functionality.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id: sys-screen

Note that this setting may be customized by user with the sys-screen-branchid-userid or sys-screen- - userid formats.

Setting Key: nomaximize Setting Value: true

(default is false: all record viewers are maximized when initially opened)

Setting Key: topleftratio Setting Value: x@y

‘at’ allows you to specify the initial location of the top left corner of the SAM Pro window (as a percentage of the screen) of the upper left corner of the image. For example, the setting value 10@5 moves set the initial open location to the right by 10% of the screen width, and down by 5% of the screen height.

Setting Key: sizeratio Setting Value: x@y

‘at’ allows you to specify the initial size of the SAM Pro window (as a percentage of the screen). For example, the setting value 40@60 sets the initial open size to forty percent of the screen’s width and 60% of its height. This might be an appropriate setting if you have a very large monitor (e.g., 1900 x 1080, for example).
