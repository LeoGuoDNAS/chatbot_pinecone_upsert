Internal Registry Zip Code Lookup (sys-zipcode-lookup)

Purpose:

This registry entry is required to enable defaulting by ZIP code. The City, State, Service Zone, Branch and Tax Group fields will only default from the zip code record if they are blank; existing values will not be overwritten.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Usage:

The Zipcode table in SAM Pro must be populated with the appropriate entries; these records may be entered by hand or imported from a database.

The Branch Id was added to the Zip Code record for defaulting purposes. The Branch (if completed) defaults in the Job, Site, Vendor and New Call / COD Site records. If you do not want the Branch to default in Vendor records, be sure that is it is not included in the registry entry that sets the zip code enablement, sys-zipcode-lookup.

Related Registries:

By default, data entry in the State / Province address field is limited to the official abbreviations of the US States and Canadian Provinces. See the State / Province Field Validation Registry if you need to add to or change the list of valid State / Province codes.

Important Fields:

Id: sys-zipcode-lookup

Control default behavior

Setting Key: default Setting Value: true or false

This option controls the default behavior for the defaulting of the blank City and State fields when a zip code (from the zip code table) is entered. Exceptions to this setting may be specified by record.

Specify Exceptions to Default Behavior

Setting Key: afunction Setting Value: true or false

This option enables you to specify exceptions to the default behavior.

 Example:

Setting Key: default Setting Value: true

Setting Key: vvndr Setting Value: false

In this example, blank fields will default from the zip code table in all records except the Vendor record.
