Registry Entry Comments

All Registry Entries – Comment Functionality for Version 7.1 and earlier

Contents / Additional Contents:

Starting with release 6.1.358.031, comments may be included in the Contents/Additional Contents fields of Registry records. This new feature allows the user to attach comments to entry values without interfering with SAM Pro's use of the values.  The double exclamation point (!!) is used to indicate the beginning of a comment.  SAM Pro terminates the registry value when it encounters the comment marker. 

Example:

COPDropShipReceipts=manual  !!options are manual, serialized or automatic (default)

To SAMPRO, this is equivalent to...   COPDropShipReceipts=manual

You may wish to use your demo database as a storage vehicle for registry entries that you've created and/or used.  It makes a good reference source when searching for vaguely remembered registry options.

Comments in Version 7.2 + Up:

Comments now have their on field that is accessed by clicking the Magnify button of the Registry child lines. During data conversion, existing comments will be written to this line.

Disabling Options:

Starter Databases (7.1 and Earlier):

If you wish to include possible registry settings or options in your starter databases, they may be included as disabled (preceded by an ‘x’, but not as blank options:

 Disabled Entry: Correct (note initial x)

 X COPDropShipReceipts=manual  !!options are manual, serialized or automatic (default)

 Disabled Entry: Wrong (may lead to unexpected and undesirable results!)

 COPDropShipReceipts=

Starter Databases: 7.2 and Up

A new field, disabled, has been added to the registry child table. It is recommended that entrys be disabled by checking this box.
