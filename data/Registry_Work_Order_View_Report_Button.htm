Registry Work Order View Report Button

The View Reports butto hasn been added to the sidebar of the Work Order viewer. When enabled, it will allow users to select from a list of appropriate reports for the viewer. The following registry settings allow you to specify one or more reports to be run from the Work Order. One registry would be made for each report to be accessed from the Work Order.  

If only one report is specified, it will be launched when the button is clicked with the selected Work Order Id inserted in the report's Work Order Id range.  If more than one registry has been generated, a list of reports to select from will be selected.

Registry Id: sys-rprt-vwrkordr:report name

A separate Registry Id must be made for each report.  Note the colon that separates the first part of the Id field from the report name.

Setting Key: MenuTitle    Setting Value: report name or other text

This option defines the title displayed on the Menu for this report if multiple reports are specified.

Setting Key: FunctionId  Setting Value: user-defined function Id of report

This key specifies the user-define function associated with the report.

Setting Key: Range~wrkordr.id  Setting Value: wrkordr.id

The registry specifies that, when the report is launched, the report's Work Order Id range will set with the selected Work Order Id it was launched from.

Example: Entry for Work Order Profitability

Registry Id: sys-rprt-vwrkordr:Report Work Order Profitability
Setting Key: MenuTitle    Setting Value: WO Profitability
Setting Key: FunctionId  Setting Value: DWOPRFT
Setting Key: Range~wrkordr.id  Setting Value: wrkordr.id

Note: Initial Values and Graphics
The standard sys-report-functionId registry should be used to set initial report values other than ranges. Initial values and bitmap registries are not supported from the sys-rprt-vwrkordr:report name registry.
