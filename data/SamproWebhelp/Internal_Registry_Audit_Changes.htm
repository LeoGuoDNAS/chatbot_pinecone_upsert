Internal Registry Audit Changes (sys-viewer-vfunctionid)

The basic idea behind this entry is to capture all field changes, buttons pushed, etc. that occur while a user is in the viewer.  When the user saves the record, the captured activity will be automatically documented as a Secure Note attached to the changed record.

As of version 7.2, accepted and rejected activity will be recorded: rejected entries will be flagged with *REJECTED*

Note: Record Change Log
This registry entry provides a much greater level of detail than is maintained in the record change log table, which only records the user, date, time and record being changed.

The existing Note record already had an access-type field that supported two values ... Public and Private.  'Secure' has been added as a third option.  Note records that are auto-created for auditing purposes are assigned the new Secure access type.  Secure means that the record cannot be edited or deleted by SAMPRO users although, at this point, they can be created. 

Secure notes can be viewed, however. Secure-type notes are filtered out of the Notes list when the Notes viewer initially opens.  To include the secure type of notes in the list, the user must change the filter option for the Access field.  By default it gets set to '!secure' in the filter ranges list.

To implement for a specific viewer:

Id: sys-viewer-vemplye
Contents: AuditChanges=true

To implement for all viewers...

Id: sys-ffv-global-settings
Contents: AuditChanges=true

This option is not recommended, as it will generate more data than is probably needed.
