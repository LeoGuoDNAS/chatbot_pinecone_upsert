GL Reporting Period

Purpose:

This viewer provides fiscal period information for gl reporting and for DB Analytics and Time Slotting .

IMPORTANT: Only Global Users (Security Id blank) should create/maintain these records.

Usage:

Required for User Defined GL Reports and DBAnalytics.

Tip: Create a beginning reporting period
Note that you must define a beginning GL Reporting period. Fill in the End Date with the last date before your actual data starts. The Start Date should correspond to the beginning of the year you are loading – not the blank date.

DB Analytics:
GL Reporting Periods should be created and maintained through the end of the current and following year to provide appropriate Calendar Date/Fiscal period information (see Calendar Date ). In particular, if a ‘beginning’ period was established for loading data, the Start Date should be set to the beginning of the loading year, not the ‘blank’ date (01011900). See DB Analytics Overview for detailed information on this module.

Time Slotting:
GL Reporting Periods are also required to enable the date calculation for Time Slotting.

Important Fields:

Reporting Period Id:
Uniquely identifies this record. This field is required.

Reporting Period Description:
This optional field provides a brief description of this record.

Report Title:
This display field is generated by the system based on the Fiscal Year and Date fields when a new record is added. This field may be edited by the user is enabled via customization as well as for Intercompany users.

Company Id: (Intercompany Installations Only)
The GL Reporting Period viewer supports optional definition of company-specific Reporting Periods to allow for differing fiscal year definitions. Any install using an Interco v.cus file will see the Company Id in this viewer. This also exposes the Report Title field.  If left blank, this field will be 'built' using the logic already in the viewer. A global registry setting is required to enable this feature and allow the Company Id to be set. 

When running a GL Report, if UseCompanySpecificGLReportingPeriods is enabled, the GL Reporting Period will now be reset if the selected Company does not match the reporting period.

Year:
The fiscal year.

Period:
The fiscal month.

Start Date:  End Date;
Defines the Starting and Ending Dates (inclusive) of this period. Note that reporting periods should never overlap. For Intercompany installations, these fields should not overlap within the same company. These fields can be used to calculate the number of weeks in this period. Validation ensures that the End Date does NOT precede the Start Date when the record is saved.
