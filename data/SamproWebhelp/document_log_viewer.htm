Document Log Viewer

NOTE: if earlier versions of document logging were implemented, the document log table must be dropped before installing this version to avoid scrty_rn errors.

Purpose:

Entries to the Document Log table are made when a Document record is created (by attaching or scanning a file or saving an invoice). A log file will also be generated when SAM Pro’s Document Archive Maintenance function is run if a file is successfully archived or if there is a problem with the archiving directory.

Registry Entries:

The following new registry options have been added to to sys-ffv-global-settings:

Create a Local Temp Directory on the User’s Computer:

SamproTempDirectory=C:\sampro\temp (Must be a valid local directory on ALL user’s systems!).

During report saving, if the regular directory the report is to be saved to is not valid or accessible the temporary directory will be used. If the temporary directory is valid then any specified subdirectories can be created. This allows for files to be saved locally and a document record created even it there is a network ‘hiccup’ that interrupts writing to the server.

Turn On Document Logging:

UseDocumentLogging=true

If this registry setting is implemented, the system will create and write a document file log of any document records saved or archived. Failures to archive or save a file will also be recorded. The registry UseDocumentLogging=true option entry MUST be made in sys-ffv-global-settings in order to implement document logging.

Related Functions:

Document Archive Maintenance

Important Fields:

Document Log Id:
Generated by the system files are archived and when the Document Archive Maintenance function is run. Note that only ‘successes’ generated by Start Document Archive Maintenance will be logged.

Date Logged: Time Logged: Logged By:
Records the date and time the maintenance was run, and the user that ran the function.

Status:
The result of the operation (e.g., success or failed)

Operation:
savetodisk (saving a report to disk), archive, etc.

Document Id:
The Id of the Document record being logged.

Function Id:
This field will be completed if a function (such as running a report) generated this operation.

Description:
A system-generate description of the operation.

Path Name:
The path to the original file (e.g, the location of the file being attached and archived).
