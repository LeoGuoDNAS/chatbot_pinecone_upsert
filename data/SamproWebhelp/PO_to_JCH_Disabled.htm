PO to JCH Disabled

This entry essentially allows us to revert to pre-6.0 methodology that postpones the costing of P.O. items until A/P vouchering occurs. This is a nuclear option when the new costing at PO receipt simply will not be accepted by existing users migrating from version 5.1. Everything possible should be done to discourage adoption of this solution. This will not be mentioned in customer notes.

Id: sys-ffv-global-settings
Contents: POReceiptsToJCH=false

Implementation of the above registry option will cause double-costing during the WO Import Costs process.  The following registry entry must be made to prevent this:

Id: sys-viewer-vwrkordr
Contents: ImportPOs=false

Work Order ‘Invoice Ready’ Status

If PO-to-JCH has been disabled, then there will be no cost to import to the WO until AP Vouchering happens for that PO. As such, it is probably desirable to prevent a work order from being set to "invoice ready" and billed knowing that all of the costs ARE NOT yet included. This forces that the Received PO items MUST have been vouchered in AP (where the costs will come from) in order for the WO to go "invoice ready:".
