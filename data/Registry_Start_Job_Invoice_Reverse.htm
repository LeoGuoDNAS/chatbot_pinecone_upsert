Registry Start Job Invoice Reverse

Purpose:

Configure Start Job Invoice Reverse.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields

Id: sys-start-sjbinvcervrse

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Allow Service Invoices to be Reversed

This registry provides a way for users to self-serve resolving situations where invoice post-process is completed, but document management fails. Users will be able to reverse and reuse to regenerate invoices, postings, and successful documents consistent with the original.
This registry also revises Service Invoice posting processing orders so that XML, Document records, and new scheduled event records can be created or committed as ARSJ postings do. This saves processing time for users and provides enhanced recoverability in the event of externalities that currently result in document attachment failures.

Setting Key: OkToReuseInvoice        Setting Value: true

Note: An Invoice Id can never be re-used when its Avalara Status is either LOCKED or VOIDED.

Allow PM Invoices to be Reversed

The default behavior is to deny PM Invoice reversal. A registry setting may be implemented to allow PM Invoices to be reversed with a warning instead of an error. This registry option is deprecated, and should only be implemented after discussion with your support representative.
