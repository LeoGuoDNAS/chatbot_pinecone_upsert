Registry AR Sales Journal (sys-viewer-varslslst)

Purpose:

Specify AR Sales Journal defaults for manual entries.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Require a GL Entity for Income / Expense Accounts

Setting Key: ValidateGLEntityIE Setting Value: Error or Warning (pick one or the other)

Specify Error if users should be prevented from saving the entry without specifying a GL Entity (if the GL Account is an Income or Expense account) or Warning if the user should simply be warned of the situation.

Deny a GL Entity for Balance Sheet Accounts

Setting Key: ValidateGLEntityBS Setting Value: Error or Warning (pick one or the other)

Specify Error if users should be prevented from saving the entry with a GL Entity specified for a Balance Sheet Account, or Warning if the user should simply be warned of the situation.

Require a Work Order for Service Compute Jobs

Jobs indicated as "Service Compute" get billed by the Work Order, yet we allow costs to be entered and posted to Jobs w/o reference to any specific work order, leaving these unbilled costs in limbo. This feature may be activated to warn or error if a Work Order is not specified for Jobs flagged as 'Service Compute' when manual entries are recorded.

Note: This setting may also be made globally in the sys-ffv-global-settings registry. If so, the local setting by Journal will trump the global entry.

Registry Id: sys-viewer-varslslst

Setting Key: NeedWOInServiceComputeJobs  Setting Value: Error, Warning or Neither (default Neither)

When the Setting Value is set to Error, the user will be alerted when leaving a line lacking a Work Order Id and will not be able to save the record. Note that implementing Error on an existing system impacts your ability to edit existing entries. It should be implemented with caution.

When Warning is implemented, the user will be alerted when leaving a line lacking a Work Order Id but will be able to save the record.

If Neither is implemented (or the registry option does not exist) the user will not be alerted if a Service Compute type Job lacks a Work Order reference.

Disable Account Type Validations on Price lines

In version 7.2.476.098, validations were added to the AR Sales Journal to prohibit the entry of AP Control accounts on Prices lines. Some users find that these validations conflict with their processing procedures: the following registry entry may be implemented turn the default error behavior into a warning.

WARNING: disabling these validations is NOT recommended.

Setting Key: IgnoreCase3193  Setting Value: true

Allow Entry Type to be Changed

Validation has been added to prevent the user from changing the Entry Type on an existing entry. By default, the Entry Type may only be changed in the Sale Journal is in ‘add’ mode. The following registry entry overrides the above validation, effectively restoring the previous behavior. Implementing this option is not recommended.

Setting Key: OkToChangeEntryType Setting Value: true (default is false)

Allow Limited Edits to the Sales Journal

This registry allows authorized users to edit or delete AR lines for Credit Card paid cash receipts. Previously edit and delete options were not allowed to full users. Now authorized users can make edits to the sales journal after it has been paid except for invoice amounts, invoice Id, invoice date, gl account, and accounting period.


Registry Id: sys-viewer-varslslst

Setting Key: AllowLimitedChangeToReceiptedInvoice     Setting Key: true
