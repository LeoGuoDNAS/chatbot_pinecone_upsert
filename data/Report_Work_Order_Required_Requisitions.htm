Report Work Order Required Requisitions

This report lists items included on the required resource tab of work orders that are not on hold and have not been requisitioned (e.g., have either a shop order id or a po id on the line).

Report Range Screen

The input screen allows the user to order the report by either Work Order Id or Job Id, and to choose whether to include Inventory Items, Other Items, or Both on the report. The user will enter an End Date for the report, which is compared against the Required By field on the Work Order Required Resource tab.

Report Ranges enable the user to filter by Work Order Id, Job Id, and Work Order Type (note that Preventive Maintenance Work Orders are initially excluded by default).

Report Contents:

The report itself lists the following fields:

Job Id - Job Name

Work Order Id – Type

Required Resource Id – Description – Quantity - Date Required
