Market Sources Viewer

Purpose:

The Market Sources viewer enables you to create, maintain or select market source records.

Example:

Typically, market sources are used to identify the source of advertising (newspaper ad, yellow pages) which generated the client or service call. By specifying a market source in your Client Site records, you can track the effect of your advertising and promotional activities.

Usage:

Optional but recommended in the Client Site record for tracking advertising/marketing effectiveness.

Referenced in Records: Client, Client Site, and Work Order.

A related record, Job Source Viewer , is used to track the source of specific jobs.

Registries to Set Disabled Fields and Set Required Fields.

Important Fields:

Note: this Help Topic lists the standard configuration of all fields which may be included in this record. If you are using a customized system, the fields displayed on your screen may not exactly match the order or status (required, optional, default, display-only, hidden, etc.) of the fields described in this Topic.

Market Srce Id: (10 character, only . and - special characters)
Uniquely identifies the market source. This field is required.

Market Srce Name: (40 character)
A brief description of the market source. This field is optional but highly recommended.

Obsolete?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be un-done when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record: see References List for details.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

TIP
Include a market source record for customer referrals to track your word-of-mouth advertising.
