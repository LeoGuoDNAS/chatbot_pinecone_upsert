Remote Access Query: next_techanywhere_versions_list

Purpose:

Specifies the current version of TechAnywhere that SAM Pro expects TechAnywhere users to be running.

Usage:

The order in which things are done is most important:

1. Install any applicable new raqs provided by programmers (including the new dynamic_content_list raq).

2. Copy the new TechAnywhere code files (e.g., hta’s and j’s) to the expected directory. This will typically be in [C:\Program Files\Cincom\vw7.4nc]\web\TechAnywhere\ta_updates – note that the part of the path in the square brackets is the path to the visual works installation which could change with each install.

3. Increase the code and datastore versions as necessary in next_techanywhere_versions_list.

4. Check Remote Access Query: code_update_list to verify that the correct files are specified for the update.

5. Start the update process within each TechAnywhere application.

See Remote Access Query: next_techanywhere_versions_headings.

Important Fields:

Application: TechAnywhere

Dataset Name: next_techanywhere_versions_list

Query Id: This field is assigned by the system when the record is saved.

Query Format: Frame

DataSet Query:

[

nextcodeversion#,

nextdatastoreversion#

]

Example:

[0.05,0.04]

This would initiate the update of the TechAnywhere wireless devices to TechAnywhere code version 0.05 and datastore version 0.04
