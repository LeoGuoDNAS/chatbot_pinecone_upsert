Recent Updates

Revised Reports:

Report W2 Electronic File: The Audit Report option of Report W2 Electronic File (report RW2EF.rpt dated 2/21/2012) has been modified so that the totals row is offset so that larger payroll amounts do not overlap. In addition, a new registry option is REQUIRED for federal processing. See user note 1.1.5 for details.

Drill Down AP Aging: The updated report (DAPVAGEv73b033.rpt dated 2/28/2012) corrects a problem that only occurred if you ran this report with the Invoice Detail option set to Yes. Previously, this resulted in the associated drill-down displaying the incorrect Vendor Name.

Revised Report: Service Invoice: Certain custom formats specify that the work requested / performed is suppressed. Previously, if these formats were included in a service compute run with (for example) FORMAT_11 formats, the suppression was not turned off for the FORMAT_11 lines, resulting in the Work Requested not printing when it should. The service invoice (serviceinvoicev73b032.rpt) report dated 2/13/2011 corrects this problem.

Dispatch Board: Previously, the Description field on Work Order List defaulted from the Work Order Name. In this version, if the Name is blank, the first 40 characters of the Work Order Work Requested will be displayed in the Description instead. A registry setting allows the Work Requested to be displayed first, and the Name to be displayed if the Work Requested is blank: see user note 2.8.2.

Document Scanning: improved message appears if the user attempts to show a document before adding the document record: “Document must be added to be available to show.”

Project Manager Workbench: Enhancements includes a Create Quote button and a new Quote tab. See user note 7.5.2 for details.

Credit Purchase Orders: Revised posting of credit POs costed to a Job or Work Order.

Job Budget Tab: The new Date Effective will be used by the system when writing entries to Job History from this tab (see user note 7.6.1). See user note 2.17 for an enhanced registry option to control the date.

Inventory Item: If the existing Inactive flag is checked, an item may not be entered on PO’s, but may be entered on WO’s and Client Orders (to use up existing stock. If the new Obsolete? flag is also checked, you may not enter this record on anything. See user note 8.3.1 for details.

WO Inventory (and other journals): This release implements enhanced journal validation logic to prevent sporadic posting reversals with inappropriate Transaction Ids.

Log Form Metrics Cycle Code Filtering: The Log Form specified on the Site Equipment record may include lines metrics that should be recorded at every service/pm call, as well as some that need only be recorded on system startup/installation. In this release, Sam Pro and TechAnywhere have been enhanced to filter lines written to the WO Metrics tab. See user notes 4.2.1, 12.5, 12.6 and 12.7 for details.

Log Form: Enhanced error handling and validation. See user note12.5.3 for details.

TechAnywhere: CC (Cycle Code) functionality has been added to Log Form (metrics) processing in the TechAnywhere upload and post.

CRM Anywhere: Previously, if the option disablecrmmanagequotes was set to true in the web-settings registry to remove the Manage Quotes page, the Quote table was also suppressed in the Site View page. In this release, the Quote table is also suppressed when this option is implemented in the registry tab of the Web Profile or Web User record.

Email: This release supports alternate email ports as well as support for secure servers (SSL). See user note 2.16 for the registry options needed to support one or both of these enhancements.

Work Order: A TechFinder button has been added to the Technicians tab. When clicked, the TechFinder opens, with the work order's Zone and the Certification for the first piece of equipment on the Equipment tab passed to the TechFinder filters. When the user completes the TechFinder processing, the system returns the selected Technician Id, Date and Time to the Technician line.

New Call (Original Screen):

When a user tabs out of the search text box in New Call 1, all fields except the Service Category are now cleared in preparation for selecting a new Site and Equipment..

The Log Form Id: field has been added to this screen. It defaults from the Equipment record, but may be changed by the user to select which log form id will be specified on the new work order.

Quote Master Approval Wizard: The work order registry option DenyChangesToQuoteMasterLines prevented Quote Master from writing to work orders in recent releases. This problem has been corrected.

New Call New Client (Flashlight) Screen: two new fields, Log Form Id: and Escalation Profile Id: have been added to the Equipment section of this screen so that the user can specify them when creating new equipment and generating a work order.

Recurring AP Journal

Corrects the calculation of the Recurring Lines Amount field based on the Quantity and Cost; previously, this calculation only supported Amounts under 100.00.

Release Recurring AP Journal: The Release Recurring AP Journal function now successfully evaluates the sys-viewer-vrcrrngapjrnl for the IgnoreCase3193 option.

Sub Anywhere: In this release, the Work Performed is always posted to the Work Order. Previously, Work Performed was not being posted if the Original Work Order Id was completed.

Inventory Receipts Journal: Corrects a problem where Use Tax was not correctly written to the Tax History table when items were received in this viewer.

Corrects issue with Labor Rate erroneously defaulting from the Employee record if the client’s customization includes reordering of the Labor Journal Hours tab.

This release corrects a problem where the Work Order Updates from TechAnywhere caused a WO Tech RN change. Also improves email alert processing.

Corrects Tech Anywhere intermittent issue posting COD multiple times.  

Corrects 8.0 problem where metrics were not written to work order from TechAnywhere.

Work Order: The Metrics tab is now correctly updated when equipment is added, inserted or deleted from the Equipment tab, as well as when the Equipment Id is changed. See user note 12.1.5 for details.

COP Acknowledgement Report: This report now suppresses the estimated sales tax if the Site COP tab has the Tax Exempt box flagged.
