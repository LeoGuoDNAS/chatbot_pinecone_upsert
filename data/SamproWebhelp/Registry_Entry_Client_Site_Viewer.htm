Registry Client Site Viewer (sys-viewer-vclntste)

Purpose:

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Related Registries:

See Default PM GL Accounts for a Global Settings option which provides default GL Accounts for the system to use if the corresponding fields on the Client Site PM Billing tab are blank when Start Create PM Billing is run.

By default, data entry in the State / Province address field is limited to the official abbreviations of the US States and Canadian Provinces. See the State / Province Field Validation Registry if you need to add to or change the list of valid State / Province codes.

Important Fields:

Id: sys-viewer-vclntste

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Option to Enable Auto-loading of Location / GL Entity From Technician record

By default, the Location and GL Entity fields in the Client Site COP tab are not over ridden when a Technician is entered Contacts tab (note that the Technician record also contains these values). This default behavior is in place to avoid having service-related field (technician) impact COP related fields (Location and GL Entity on the COP tab).

To override the default behavior, the following registry entry must be implemented. If you do so, that entering or changing the Technician will overwrite the Location and GL Entity on the COP tab from the Technician record.

Setting Key: LocationFromTechnician Setting Value: true

Control Display of WO Tabs

Setting Key: WorkOrderWhereClauseOpen  Setting Value: valid SQL expression

Setting Key: WorkOrderWhereClauseOther  Setting Value: valid SQL expression

Example:

Setting Key: WorkOrderWhereClauseOther  Setting Value: wrkordr_stts_ctgry not in ('Open','Archived')

Registry to Inform User if Alias in Client Site is Set and User Changes Site Name

This registry displays a warning or error message when the payment Alias has changed owners so it point at the new owner.

Id: sys-viewer-vclntste

Setting Key: DisableAliasValidation    Setting Value: warning/error/disabled   (warning is the default option)

If this is set as a warning, a warning will pop up with:
Warning: Is the Payment Alias specified for (old client site name) in this record still appropriate for (new client site name)?  If ownership of this Site has changed, be sure to clear the Payment Alias in the Client Site record.  Note: a Payment Alias may also be specified in the Master Contract which may need to be changed. 

If set as error:


If disabled, edits will be allowed to be made with no message.
