Registry Crystal Report Viewer (crv)

Purpose:

This registry setting is required to enable Crystal Report Viewer functionality within SAM Pro’s Service and Back Office modules.

Usage:

In addition to this registry entry, the Crystal View application from Data-Basics must be installed. This is available in Install Shield format from Data-Basics support. It may be installed as a fat client on the Windows Desktop, or under Citrix.

A User Defined Function record must be created for each Crystal Report. The following fields must be completed:

Function Id: Function Name:
Both fields must be unique.

Application:
Set this field to ‘crv’

Menu Category:
Select ‘Report’.

Application Parameters:
Specify the Crystal Report as follows: report=areportname.rpt

 The system will look in the directory specified in the registry entry for this report.

Remember to enable your new User-Defined Functions for all applicable User Groups.

Crystal Reports run from within SAM Pro are designed and created using off-the-shelf Crystal Reports software. Documentation and training courses for this software is readily available.

Important Fields

Id:

 CRV-securityid

 CRV-branchid-userid

 CRV--userid (if branches are not implemented)

 CRV

Description:
Although this field is optional, we strongly recommend that you use it to briefly describe this registry entry (e.g., Crystal Report Viewer).

Contents:

 RegistryASStringSeparatedBy=~

 This line is required and must be entered as listed.

Additional Contents:

exe=path\CrystalView2005.exe
where path is replaced by the full path (share path or drive designation) to the directory containing the Crystal Viewer executable. Avoid specifying a path with a blank in it.

Example of a Share Path:
exe=\\myserver\sampro\crystal\crystalview2005.eve

rptpath=path
where path is replaced by the full path (share path or drive designation) to the directory containing the Crystal Reports used by this company.

Example of a Share Path:
rptpath=\\myserver\sampro\crystal\reports

dsn=samprodsn
where samprodsn is replaced by the data source name for your SAM Pro database.

mode=u or mode=a
Enter mode=u to enable users to specify report settings, or mode=a to automatically launch the selected report. One of these options must be specified.

companyname=^companyname^
user=^user^
branch=^branchfilter^
pm=^pmfilter^
location=^locationfilter^
These contents lines are required for correct filtering of reports, and must be entered exactly as exactly as shown above.

AdditionalRegistrySettings=registry1,registry2,...

This option will pass the registry settings implemented by the specified registry entries to Crystal View. Note that if multiple registry entries are specified, they should be separated with commas.

Example:

AdditionalRegistrySettings=sys-email

This passes the sys-email registry options settings to CrystalView so that email settings used for SAM Pro will also be available for Crystal View.

ReportAnywhere=Enable_ReportAnywhere
This option is required in order to enable the scheduling of Crystal reports. This causes SAMPRO to pass the licensing info to the Crystal Viewer app so that its facilities may be used for scheduling.
