Registry to Auto Fill Checks as Not Cleared

The default behavior of the Checks tab determines the "Cleared" boolean by comparing each imported check's date to the Date Effective in the parent. If the date of the incoming check is less than or equal to the Date Effective, the new line is set to "Cleared", otherwise it is left blank. The registry determines whether to override the date comparison and simply always set the new line to not cleared, but only when not in Add Mode.

Setting Key: AutoFillChecksDefaultCleared  Setting Value: T/Y/F/N/Prompt

If the registry is set to T or Y or not set at all, the behavior remains the same as before the registry, where the date of the check and effective date are used to determine if the new line is Cleared.

If the registry is set to F or N new lines will never be marked Cleared.

If the registry is set to Prompt, when pressing the AutoFill button, users will be asked "Prevent Incoming Checks from being Cleared?". A No response is treated the same as the F/N registry, a Yes response is treated the same as T/Y registry. Note that if there are no new incoming check lines, the prompt will not display because there is no data to process.
