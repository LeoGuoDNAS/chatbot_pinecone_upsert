Registry Report EFT AP

This registry sets the default GL Account / GL Entity when the printed EFT register is run.

A separate registry (sys-eftap-glaccountid) is required for the output of the ACH file itself.

Options to Set the Default GL Account And/Or GL Entity

Registry Id: sys-report-reftap

Options to Set the Default GL Bank Account (use one or the other)

Setting Key: DefaultGLBankAccount   Setting Value: GLAccountId  or
Setting Key: GLBankAccount   Setting Value: GLAccountId

The GLBankAccount and DefaultGLBankAccount options provide a means of defaulting the bank account when these reports are run. Use of GLBankAccount and DefaultGLBankAccount should be thought of as mutually exclusive even though the Registry viewer won't prevent it. Do not use both!

Intercompany Processing Note: If the DefaultGLBankAccount option is set, the GL Bank Account specified in the registry (e.g, sys-report-reftap-securityid) defaults when the user selects the Company (or it defaults if only one Company has been specified in the database). The Account will be cleared if the user selects a different Company for which the registry is not set.

Options to Set the Default GL Entity  (use one or the other)

Setting Key: DefaultGLBankEntity   Setting Value: GLEntityId  or
Setting Key: GLBankEntity   Setting Value: GLEntityId

The rules for Entity defaulting and validation are similar to the above description for GL Accounts.
