Registry Custom Folders Entry (sys-folders)

Purpose:

This registry entry enables you to customize the Topics that appear on the Workflow Folders viewer, as well as which Tabs appear on which Topics. This registry entry may apply to all users, or multiple registry entries may be created to apply to specific user groups or users.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Usage:

Note that the registry entries for an updated, standard folders setup may be imported using the Start Registry Import function. Contact your support representative for details.

Important Fields:

Id:
A registry entry may now be created to customize the Topics that appear on the Folders viewer, as well as which Tabs appear on which Topics. This registry entry may apply to all users, or multiple registry entries may be created to apply to specific user groups, branches, security settings, or users.

The Id of the Registry Entry must conform to one of the following formats:

sys-folders-usergroupid-branchid-userid
sys-folders-usergroupid-securityid
sys-folders-branchid-userid
sys-folders-securityid
sys-folders

In the first format, -usergroupid is replaced by the User Group Id you wish to create the folders. Note that if you also wish to customize the folders for a particular branch, the optional -branchid is replaced by the Branch Id for which you wish to create the folders. If this entry is for an individual user, the -userid option would be replaced by the User Id for which you wish to create the folders.

In the second format, -usergroupid is replaced by the User Group Id you wish to create the folders. Note that if you also wish to customize the folders for a particular security level, the optional -securityid is replaced by the Security Id for which you wish to create the folders.

The third format enables you to specify folders by security id and (optionally) user. Note that -branchid is replaced by the Branch Id for which you wish to create the folders. If this entry is for an individual user, the -userid option would be replaced by the User Id for which you wish to create the folders. If branches are not implemented, two dashes (not separated by a space) must precede the user id.

The fourth format enables you to specify folders by security. Note that -securityid is replaced by the Security Id for which you wish to create the folders.

The fifth format would apply to all users unless another registry entry was created for their group, security id or branch id.

During program execution, the software attempts to find a match for the user that logged on and a sys-folders registry entry in the following order:

User Group, Branch and User
User Group and Branch
User Group and Security
User Group Only
Branch and User
Branch Only
Security Id Only
sys-folders entry with no options

Name:
Optional but recommended for describing what this entry is used for.

Define Folder Contents

This option is used to defined the names of the topics that will appear on the Folders. These names are user definable and use the following format:

Setting Key: topics Setting Value: topicname1,topicname2,topicname3,..,topictopicnamez

This entry is required. Replace the topicname fields following the = sign with the topic name as you wish it to appear on the folders viewer. Note that topic names must be separated by a comma. For example, the following entry would be used to include 4 topics in the Folders viewer:

Setting Key: topics Setting Value: Accounting,Service,Job Costing,Other

Define Folder Tabs

The Additional Contents lines of this registry entry are used to defined which folder tabs appear on a topic. One Additional Content line should be entered for each topicname you defined. Note that a tab may appear on more than one topic.

Setting Key: topicname Setting Value: tabname1,tabname2,...,tabnamez

In this format, topicname must be replaced by a topic name that you defined in the Contents section. The tabnames, however, must conform to the standard Data-Basics naming conventions as specified on the following list (note that Sales pertains to the Client Order Processing modules, and Special includes maintenance features for ODBC, function list updates, and empty record updates):

 AccountsPayable   AccountsReceivable
 ClientSites   GeneralLedger
 JobCosting  Inventory
 Invoicing   Lab/H&S
 Payroll    Purchasing
 Reporting   Sales
 Security    ServiceSetup
 Special    WorkOrders

For example, to include the Payables, Receivables and General Ledger tabs on your Accounting topic, you would create an Additional Contents line as follows:

 Setting Key: Accounting    Setting Value: AccountsPayable,AccountsReceivable,GeneralLedger

Specify Font and Font Size

The topicfont option controls the font and size of the topics list and tab labels.  The buttonfont option controls the font and size of the button labels.

Setting Key: topicfont  Setting Value: aFacename,points

Setting Key: buttonfont  Setting Value: aFacename,points

where aFaceName is replaced by a valid font name (e.g., Arial, Tahoma). This font must be installed on the users’ computer or it will not be used.

points should be replaced by the desired point size (typically 8, 9 or 10).

Example: To make the Workflow Folder labels a little bigger than the standard settings:

Setting Key: topicfont  Setting Value: Arial,9

Setting Key: buttonfont  Setting Value: Arial,9

Specify Menu Ratio

Setting Key: listratiopercent  Setting Value: ##

This option defines the percentage of the WorkFlow window that the left-hand menu uses. Typically, the Setting Value: 20 produces a good proportion.

Specify Tree or List on Left-Hand Menu

Id: sys-folders

Setting Key: listtype  Setting Value: tree or list

This option controls when a list of topics is shown on the left-hand menu or an expandable 'tree' what allows the user to expand each topic to show and access its component tab.

Folders Layout and Style

Setting Key: layout   Setting Value: sidebyside

Setting Value: folderstyle  Setting Value: web

These settings are required to display the workflow folders correctly for current versions of SAMPro.
