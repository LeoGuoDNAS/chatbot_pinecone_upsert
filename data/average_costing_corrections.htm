Inventory Average Cost Corrections

In recent releases, SAM Pro’s methodology for computing average cost of inventory by location has been significantly improved. However, inventory with historical data (particularly if Shop Orders or Inventory Adjustment Journals were involved) may require some adjustments to provide reasonable average cost if your customer indicates that there is a specific problem.

Process for fixing Average Costing

First: Correct the Average Cost by Location

1. Go to the inventory item. Determine each location with an On-Hand Qty and wrong average costs.

2. Do an inventory adjustment for each location to make the on hand quantity zero (0)

- If the On-Hand qty at the location was 12 then the qty in the adjustment is -12

- Leave whatever the average cost that the system automatically fills in and save the transaction

3. Do another inventory adjustment and put back original On-Hand Qty. **** It is VERY important to use the correct average cost for this transaction – overwrite the wrong average cost and save!

Second: Correct the Average Cost for All Locations

Once you are done and the average cost is correct for every location with an On-Hand qty, you must now fix the average cost for all locations.

1. Go to the Inventory item, go to the Pricing/Costing tab, and run the show computation.

2. Determine the last transaction in the computation - New Qty and New Unit Cost (average cost)

Example of the existing computation last line:

 NewQty NewUntCost

120.00 200.00

In this example let’s say the average cost should have been $150.00

Existing  120 x 200.00= $24000

Should be 120 x 150.00= $18000

Preferably, choose a location with no On-Hand qty if it exists. If not, you can use any location.

3. Do an inventory adjustment and (using the above example), -120 at a cost of $200 (should automatically fill in this cost).

4. Do an inventory adjustment using 120 at a cost of $150 – Again, MAKE SURE THAT THE COST IS THE LAST THING YOU FILL IN, because as soon as you fill in the location it will automatically pull in the WRONG average cost!
