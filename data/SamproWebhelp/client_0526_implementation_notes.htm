12/16/2005

Remote Time Problem:

Currently in Remote Time (RPC class - timeSetRegistryDates) even though today Friday 11/18 is the start of the next pay period the view will display last pay period as the default.  Only if the current day > first day will the period change.  Could we have a setting to allow the current period to change on the first day and not (day + 1).  Would like current day >= first day to set period default.

Current Solution:

Have them set periodsfuture=1.  We'll look at adding yet another option to the registry for a future release.
