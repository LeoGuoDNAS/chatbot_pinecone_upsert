Registry AR Cash Receipts

Purpose:

AR Cash Receipts Journal options.

Related Registries:

Disable Warning when Journal Entry has been Replaced

Important Fields:

Id: sys-viewer-varrcptslst

Option to remove the Other tab

Most users will never need to use the Other tab in AR Cash Receipts. Implementing this option restricts the child tabs to those on the list.

Setting Key: ChildTabs Setting Value: ar

Warning: If you upgrade to a newer version of SAM Pro and a new tab has been added, it MUST added to any registries that specify child or parent tab overrides or the new tab will NOT appear in the viewers.

See Viewer Ratios and Tabs for additional information.

Force GL Bank Account to be Set from Payment Type

The following registries may be enabled to force the GL Bank Account to be set from the corresponding field in the Payment Type record. When the following registries are set:

1. Do not allow user to manually enter a GL Bank Account Id.

2. Do not allow transaction to save without a Payment Type Id.

3. Always default GL Bank from selected Payment Type.

Id: sys-viewer-varrcptslst

Setting Key: BankGLFromPaymentType   Setting Value: true (default false)

Always set the Bank GL Account from the Payment Type.

Setting Key: RequiredFieldsParent    Setting Value: pymnttype.id

Require that the Payment Type is set.

Setting Key: DisabledFieldsParent   Setting Value: gl-bnk: glaccnt.id

Disable manual entry to the GL Bank Account field to prevent the user from attempting to do so and getting an error.

Important: If credit card processing is enabled the specified Payment Type must have the Bank GL Account field set. Preferably, all active Payment Type records should have the Bank GL Account field set.

Applying Overpayments to Original Invoice

If a customer pays the same invoice twice, the following registry entry may be implemented to enable the clone function in the AR Cash Receipts Journal. This allows the user to 'clone' the original Cash Receipt to create the overpayment entry.  

Registry to Implement

Id: sys-viewer-varrcptslst

Setting Key: isOkToCloneRecord  Setting Value: true  (default is false)

Note that Cloning is NOT allowed if NewReversing is enabled in the AR Cash Receipts Journal.

Procedure to 'Clone' the Cash Receipt

1. Access the original AR Cash Receipt you wish to copy.

2. On the menu at the top of the viewer, select Record > Clone Record.

3. The system will copy the original receipt, clearing the Check Number and Credit Card fields.

4. Edit the Check Number,  Accounting Period, Effective Date and other fields as needed and Add the new record. AR History will show the additional payment to the specified invoice.

Allow Limited Edits to Cash Receipts Journal

This registry gives users permission to make field changes and delete child lines for saved records with processed payments. With the enabled registry, this allows the user to zero out the A/R child rows for "incorrect" invoices and insert the new rows with the "correct" invoices, so long as the total against the processed payment matches. 
Note: If the wrong amount was processed, the users must refund the CC transaction and reprocess it.

Id: sys-viewer-varrcptslst
Setting Key: AllowChangeToProcessedReceipt    Setting Value: true
