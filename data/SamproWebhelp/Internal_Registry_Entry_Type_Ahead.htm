Internal Registry Type Ahead (sys-id-autofill)

Important Notice: this help file is intended for use by Authorized Data-Basics Support Personnel only. Incorrect implementation of the information discussed here can create problems with software functionality and/or data.

Purpose:

This registry entry enables the default type-ahead (also known as Autosearch) behavior to be altered on a table-by-table basis.

Background

In previous releases, Type Ahead was enabled in most Id fields: you would start entering the Id, and the system would complete it for you as soon as it located a ‘match’. However, due to performance issues, this functionality was disabled for the job, client, and client site within the image itself. This hard-coding of type-ahead meant that it could not easily be tuned to specific user implementations.

Changes in this Release

In the absence of this registry entry, versions 6.1.358 and up observe the following default behavior:

No Type Ahead By Default:

Numeric Id’s generated by the system, including Journal Id’s, PO’s, Work Orders, Client Orders, Quotes. In these cases, type ahead does not gain the user anything because the id is most likely not defined until the last digit is typed in.

The special cases of Job Cost Code, Job Billing Item, Client Site Equipment (because these records are children of the Job and Client Site records (respectively)).

Type Ahead Turned On By Default

All other records with alphanumeric Id’s.

Important Fields:

Id:

 sys-id-autofill

Contents / Additional Contents:

 tableid=true (turns on type ahead) or

tableid=false (turns off type ahead)

where tableid is replaced by the function id of the table.

Example:

Assume that your sys-id-autofill registry entry has the following contents:

clntste=false

wrkordr=true

In this case, all the type-ahead defaults would be observed except that type ahead would be turned off for client site id fields, and on for work order id fields.
