Record Lock Recovery

SAMPro Version 9.1.611.053a and above provides a recovery process to resolve temporary record locks for MS SQL Server databases; this functionality is not supported for Sybase databases. When a record lock condition is encountered, a dialog box will be displayed. See the sample below.

The dialog box will include information as to the user who is holding the lock. This gives the locked out user a chance to confer with the locking user who may be able to resolve the situation. If the user does not have permission to obtain the lock information, an appropriate message will be displayed instead of the lock information itself.

The dialog box contains 3 buttons:

‘Try Again’. When selected, the dialog box will disappear, and the query will be attempted one additional time.

If after the attempt, the lock is still in effect, the dialog box will be redisplayed.

If, however, the lock has been cleared by the locking user so that the query is now successful, the user will be so notified, and processing will continue as if the lock had never been encountered.

‘Try For 60 Seconds’. When selected, the dialog box will disappear, and the query will be repeatedly attempted for approximately 60 seconds.

If, after the 60 seconds, the lock is still in effect, the dialog box will be redisplayed.

If, however, anytime during the 60 second wait, the lock has been cleared by the locking user so that the query is now successful, the user will be immediately notified, and processing will continue as if the lock had never been encountered.

‘Cancel’ (or ‘X-ing’ out the dialog box). The query will be tried 1 more time.

If, after the additional try, the lock is still in effect, the query attempt will return a failure status and processing will continue as is currently implemented.

If, however, the additional try is successful, the user will be so notified, and processing will continue as if the lock had never been encountered.
