Rate Schedule Viewer

Purpose:

Create and maintain the rate schedules which determine the starting and ending dates applicable to your labor, material and other rate tables.

Example:

Create a Standard Rate Schedule for 2012, 2013, etc. labor, material and other tables.

Usage:

Required for invoicing.

When the Rate Schedule is saved, the detail lines are now re-ordered with the most recent entries at the top. Validation prevents overlapping date ranges and gaps which cause undesirable results when SAMPro attempts to determine which line of the Rate Schedule to use.

Other Rate Not Specified
If an Other Rate is NOT found on the Other Rate Table (referenced in the Rate Schedule), the system simply defaults values contained in the Other Rate record itself.

Foreign Currency Processing
Separate Rate Schedules must be created for your foreign currency customers and referenced in the applicable Job records. These Rate Schedules must reference Labor, Material and Other Rate Tables that specify the revenues in terms of the foreign currency – as you wish them to appear on invoices.

Service Provider Rates:
If this Rate Schedule is designed for Service Providers, only the Other Rate Table Id should be specified: the Material Rate Table and Labor Rate Table fields should be blank.

Important Fields: Upper Section

Rate Schedule Id:
A unique Id for this table. This field is required.

Rate Schedule Name:
A brief description of this table. This field is optional.

Obsolete?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be un-done when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record: see References List for details.

 

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

Important Fields: Upper Section

Start Date  End Date
The starting and ending dates that defines the effective (work) dates for which these tables will be used. You may include as many time periods on this table as needed. Time periods should not overlap.

Labor Rate Table Id:  Material Rate Table Id:  Other Rate Table Id:
The Labor, Material and Other Rate Tables to use for the specified time period. Double-click on these fields to select from a list of Id's.

Important Note on Service Provider Rates:

If this Rate Schedule is designed for Service Providers, only the Other Rate Table Id should be specified: the Material Rate Table and Labor Rate Table fields should be blank.
