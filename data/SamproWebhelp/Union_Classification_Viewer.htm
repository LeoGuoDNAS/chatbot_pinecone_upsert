Union Classification Viewer

Purpose:

Classify your personnel for reporting purposes, typically across unions.

Usage:

Referenced by: Union record, Earnings Group record.

If a Classification Id is included in the Earnings Group record, reports can be designed that look at employees that are linked to this Classification via their Earning Group. Therefore, you could report on Foremen, Apprentices, etc., within a single union or across all unions. Contact your Data-Basics support representative for further information.

Example:

Foreman, Apprentice1, etc.

Important Fields:

Classification Id:
Uniquely identifies this record (Foreman, Apprentice1, etc.). This field is required.

Classification Name:
Briefly describes this Classification. Recommended for reporting purposes.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

Classification User1: Classification User2: (30 character)
These optional fields may be used for entering any information your company wishes to retain (e.g., additional account numbers).
