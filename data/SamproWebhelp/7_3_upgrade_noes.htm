7.3 Upgrade Notes

7.3.498.097

New XmlToPdf.jar File: The XmltoPdf.jar file was updated as of 11/08/2011 to support a date field within the email header required by a client’s email server, and should be installed. A copy of the older file (renamed as XmlToPdf[OLD].jar is also included with the image in case it is needed.

Update your Java Version: Upgrading JAVA on the computer(s) that generate .pdf files is recommended to ensure that these files are made and e-mailed correctly. This includes the server that the Scheduled Event Manager and Scheduled Report Manager run on.

Important Installation Note: The zip file for this release includes a file containing all bitmap definitions in the image (master~pushbuttons.obj) and an executable image (v.exe) stripped of all bitmaps. The bitmaps will be loaded when a user logs on from the master~pushbuttons.obj file and the SKIN1 directory (if bitmap skinning is in effect). Running this release requires the presence of the change.log, recover.log and master~pushbuttons.obj files in the same directory as the image (v.exe) file. This configuration should allow XP users to run without ‘non-continuable protection fault errors’.

NOTE: see the registry entry for sys-gui-skin-skinname for details on enabling this functionality.

Important Note on Colors: In order to fully support Bitmap Skinning, users’ computer display settings must be set to 24 or 32 bit color. If the Color Quality is set to 16 bit (Medium in XP, or High Color in Windows 7), the new bitmaps will appear with their standard, pale gray background, but the registry settable background and mouse over color changes will not appear. Setting the Color Quality to 24 or 32 bit should provide the full bitmap skinning functionality.

VM Memory: SAM Pro’s Smalltalk virtual memory has been expanded to 250MB from 150MB. This should better support the running of extremely long reports and the unfiltered display of large record viewers without encountering out of memory errors.

Obsolete View Issues – Service Zone

One of our (very old) existing customers reported that prior to the upgrade to version 7.2 they had full access to service zones; after the upgrade they did not. This was caused by the fact that SAMPro once had 2 service zone viewers, vsrvceZne and vsrvcezne (both are named View Service Zones). Notice the "z". Z did not get along with z and caused headaches for programmers when some users were using Z and some users were using z, so the programmers obsoleted Z. This then made it so that "View Service Zones" (the Z version) was no longer on the list.

Solution:

The solution is to have an administrator grant access to the correct Service Zone function after the upgrade.

Support Hint:

1. Before upgrading your customers, access the Service Zone viewer and see if it has an uppercase letter in the function id. If not, you will not have a problem.

2. If it does include an uppercase letter in the function id, determine which user groups have access to View Service Zone.

3. After the conversion, give the appropriate User Group and Functions records access to the Service Zone viewer.
