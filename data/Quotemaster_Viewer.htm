Quote Viewer - Obsolete

NOTE: The Quote Manager Viewer should be used in Versions 9.1 and Above.

Purpose:

This viewer enables sales quotes to be created and maintained. It may be also be used to update quotes generated from CRMAnywhere or TechAnywhere (see Quotes Generated by TechAnywhere ).

A quote may contain multiple options. Each of the lower tabs has an ‘Option Code’ field. All detail lines having the same Option Code are tied together in that they pertain to the same Quote Option.

When a quote is initially created, all options default to Open. When the customer approvals one or more options on a quote, the QuoteMaster Approval Wizard can be invoked which will enable the user to automatically generate the appropriate work orders and purchase orders needed to complete the work.

Usage:

Cloning a Quote: Click for DetailsClick for Details

To ‘clone’ an existing quote, enter the word new in the quote id of an existing quote and tab out of the field: a new quote is generated with the following changes implemented:

On the Main tab, the WO Id and Job Id are cleared.
On the Scope tab, set all options to Open, clears the WO Id and Job Billing Id (if they exist) and the Equipment Id, as well as the Opened, Accepted, Approved and Declined Fields.
On the Inventory Tab, clear the PO Id and Vendor Quote No.
On the Material Tab, clear the PO Id and Vendor Quote No.
On the Subcontractor Tab, clear the PO Id and Vendor Quote No.
On the Notes tab (header info) set the Status to Open, Status Category to Active, and Status date to today.

The new quote may now be edited.  To change the Site, do the following:

1. Blank the WorkOrder Id and Job fields (if completed)
2. Blank the Equipment Id in all Scope Lines.
3. Enter the Site Id the new Quote is for.
4. Enter the appropriate Equipment Id for that Site on all Scope lines.

The system will assign the appropriate Quote Id when the record is saved.

Approval Wizard Failures: If a WO o PO fails to be generated, the software displays a message box and writes the errors as .txt files to the directory defined by the dbreportfiles registry id.

Scope Option Must Be Open to Allow Changes (unless registry enabled)
The system disallows changes to any scope lines and any associated lines if the status is set to Printed, Accepted, Approved or Declined (the user must change status to Open to make changes).  A registry option may be enabled to allow changes to Accepted options (see Registry Quotemaster (sys-viewer-vqte) .

Accounting Periods : A Default Accounting Period should be defined and maintained if Purchase Orders are to be created or updated from Quote Master. When generating the PO, the system

1. Uses the default Accounting Period (if it exists),
2. If no default exists, it uses the 'first' active period with today's date in the date range. 
3. If both of the above fail, the Wizard warns the user, and the PO post fails.

This field specifies the Estimator responsible for this Quote (optional); it defaults from the Quote's Job record anytime the Job is changed (by direct edits to the Job Id or by changing the Work Order Id).

Expiration Date:
This field indicates the last day on which the customer can accept the quote. This field is available for reporting purposes, but NO warnings or validations are performed relative to approving options.

WorkOrder Id:
If this quote is being generated in response to a Work Order, enter the Work Order Id. When you do so, the Site Id, Tax Group and Status, Sales Rep, and Rate Schedule (from the associated Job record) will default in. A Work Order may be used as the source Work Order on multiple Quotes.

Lower Section: Important Fields

Scope of Work Tab

The Scope tab acts as a master for the other tabs. A given option code must appear on the Scope tab before it can be entered on any of the other tab. A given option code cannot appear on more than one Scope tab line but may appear an unlimited number of times on the other tabs.

If an Option Code is changed on the Scope tab, that change automatically propagates to the other tabs referencing that option code. If an Option Code line is deleted from the Scope tab, all lines referencing that option code are automatically removed from the other tabs.

Pushing the button launches a new wizard that lets the user complete the selected scope line by copying the details from existing scope lines AND the Labor, Inventory, Material, Sub and Other lines associated with these Scope Options. Click Quote Scope Copy From Button for detailed information on using this button.

Unit Cost:  Extended Cost:
The unit cost for this item defaults from the Flat Rate Category (if selected) or from the Inventory Item. The system calculates the extended cost by multiplying the Quantity times the Unit Cost. The unit cost is determine as follows:

1. If inventory item references a DEFAULT VENDOR (Inventory Purchasing Tab):

That Vendor defaults into the quote line as the item is entered (unless disabled via registry).  The cost value is set to the LAST PRICE paid for the item to THAT vendor (see Inventory Item – Vendors Tab; detailed on the Purchases Tab).  

a. Selecting a DIFFERENT VENDOR will recalculate the cost:

i. If the new Vendor has a different LAST PRICE paid, then that value replaces the cost value previously loaded; if no differing history is found, then cost value remains as is.

ii. Clearing the default Vendor results in NO LAST PRICE history found for VENDOR <blank> and does NOT invoke the protocol in option 2 below.

2. If the item has NO default Vendor:

The applicable Cost Method (last, average, standard) is used at the overall level: the quote does not attempt to determine which stocking locations may be utilized in satisfying the inventory needs of the quote.

a. If a VENDOR is selected on this line, the cost will be updated as described in 1.a.i above.

Unit Price Extended Price
If this option is not Fixed Price (which requires that these fields be 0$), and this Inventory Item is not flagged as ‘Use Unit Price’, the price for this item is calculated from the Inventory Item/Rate Schedule fields or is calculated based on the Cost and Gross Margin or Percent Markup calculations. The system calculates the extended price by multiplying the Quantity times the Unit Price.

If an Inventory Item flagged as ‘Use Unit Price’ is selected, the Material Price Level from the Client Site will be used to determine which level Unit Price will default.

Price Code Handling:
The Inventory Item allows for 10 levels of Material Pricing, but Price Codes only allow for 5. If price codes are in effect, and the default price level from the Client Site is greater than 5, the system now defaults price level 1 from the Price Code. A registry entry Price Code Handling (sys-ffv-global-settings) is available to always default the value from the Price Code level 1, regardless of the Material Price Level in the Client Site.

RevAdjust Tab

Revenue adjustments enable you apply discount and management fees to the quote total. Clicking the Adjust Revenue button on the main tab populates this tab with adjustments based on the Revenue Adjustment record(s) specified on the Scope tab.

When the Quote is approved and a Work Order is generated the information on this tab will be applied to the corresponding work order RevAdjust tab. 

Option Code:
This required field identifies the Option on the Scope tab this line is associated with. Double-clicking on this field displays the list of this quote’s Options to select from ; all options are displayed in lowercase text.

Invoice Description:
This field will be printed next to this discount or addition on the printed quote and invoice generated from this quote (e.g., ‘Loyal Customer Discount’). This field defaults from the Revenue Adjustment record.

o Add o Deduct
This field controls whether the dollars on this line will be added to or deducted from the quote.

Important Note: When SAMPro calculates the AdjBase if it's value is negative, the Add / Deduct button is automatically flipped from its default setting. This supports the correct setting of Revenue Adjustments for negative scopes.

AdjBase: Rate: Amount:
For Discounts, the AdjBase would be typically be set to the Fixed or Total Quote amount for this option. The Rate should be set to the discount to be applied to this option. The Amount is calculated by the system by multiplying the Adjustment Base times the Rate. To deduct a specific dollar value, leave the AdjBase and Rate fields blank, and simply enter the amount.

‘Adds’ (such as management or trip fees) are generally entered as a flat amount. Both the Adjustment Base and the Rate will be blank, and the Amount will be the flat fee.

Cost Category:
The default Cost Category for this adjustment.

GL Account Id:
This field sets the default discount account for the entries that will be generated on the RevAdjust Work Order tab, and is typically an income type account. It will be debited (as an offset to the credits for this type of income) when the work order invoice is posted.

GL Entity:
The default GL Entity for this adjustment. This field defaults from the Default Service Job Cost Code found in the Client Site Equipment referenced on the specified scope line.

PSeq#:
The Print Sequence Number sets the order that multiple revenue adjustment lines will print on the invoice.

Tax Type:
The Tax Type will default to "n/a" and will be evaluated and properly handled by invoicing should this quote become an actual work order.
