Sales Rep Viewer

Purpose:

Create and maintain Sales Rep records that may be referenced on the Commissions tab of the Job record for the purpose of determining commissions.

Usage:

Optional for reporting. Referenced in the Job record's Commissions tab. This viewer may also be used to record sales quotas. See Sales Commission History for details on entering sales date for reporting purposes.

Related Reports:

Sales Summary By Sales Rep Report

Important Fields:

Note: this Help Topic lists the standard configuration of all fields that may be included in this record. If you are using a customized system, the fields displayed on your screen may not exactly match the order or status (required, optional, default, display-only, hidden, etc.) of the fields described in this Topic.

Sales Rep Id:
Uniquely describes this sales rep. This field is required.

Name:
The full name of the salesperson. Optional but highly recommended.

Territory:
Optional. Briefly describes the territory (e.g., Central or OH-MI-KY).

Contact Id:
This field specifies the record that supplies the e-mail, phone, and address contact information for this Sales Rep.

Contact Type Id: Firm:
Email Address: Email Fax: Email Address 2: (formerly Email Pager)
Notes: Mailing Address:
These display-only fields default from the Contact record. To update them, right-click on the Contact Id and edit the Contact record directly.

Sales Rep User1 -2:
Optional fields for informational and user reporting purposes.

Obsolete ?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be un-done when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record: see References List for details.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record.

Quotas Tab

This tab may be used to record the sales quotas, by Job Type, for this Sales Rep. This information will be written to the Sales Commission History table, where it may be used for reporting purposes. Changes to the lines on this table (including the deletion of existing lines) will also be recorded to Sales Commission History.

Year:
The year these quotas are for.

Job Type Id:
The Job Type this quote applies to. This field is required.

Total:
The Total Quota for this year. When you tab out of this field, the system will ask “Would you like to spread this amount evenly across all months.” If you answer Yes, the system will complete the Jan-Dec scheduled quota fields for you. If you answer No, you must complete the schedule manually.

Jan: - Dec:
The monthly sales quota for this Sales Rep / Job Type.
