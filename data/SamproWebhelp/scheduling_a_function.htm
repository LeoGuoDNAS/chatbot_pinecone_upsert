Scheduling a Function to Run as a Repeating Event

Purpose:

SAM Pro Starter functions, such as Start Purge Record Change Log and Start TechAnywhere Post may be scheduled to run automatically by creating a repeating event for the Scheduled Event Manager to run. When the function is run, it will use the registry entry (if any) you have set up for the function.

To set up a Scheduled Event to run a function:

1. Scheduled Event Manager needs to be running.

2. Go to the Scheduled Event Viewer, and select New Record.

3. You will need to complete the following fields and save the event - then the manager will run it at the intervals that you specify:

Profile Time Base Id: scheduled

Function Id: the starter function you need to run

Repeat Options:

Repeating Event?: flag this box

Date
The day or days the function is scheduled to run.

To specify a day of the week:

 m  tu w th f sa su

To run the function more than once a week, separate the days with a comma. For example, to run the function on Mondays, Wednesdays and Fridays:

 m,w,f

To specify a day of the month:

 1 2 3 4 5(etc.)

 first last

 Note that last would be 28 (or 29) for February, 30 for April, etc.

Time

Enter the time (in 24 hour/military format) the function is scheduled to run.

For example, to run the function at 3:00am you would enter 0300. To run the function at 8:30pm, you would enter 2030.

To run the function more than once a day, enter the times separated by
commas: 0100,0900

To run the function multiple times a day, enter the first and last time to run as a range, followed by the @ sign and the increment in minutes.

For example, to run the function every half hour from 9:00 am to 5:00pm, enter the following: 0900_1700@30

To run the function every hour from 8:00 to 6:00, and then again at 11:00pm you would enter: 0800_1800@60,2300
