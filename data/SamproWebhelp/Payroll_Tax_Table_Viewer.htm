Tax Table Viewer

Purpose:

Create and maintain the tax tables used in conjunction with your Payroll Tax Authorities.

Usage:

See Arkansas State Tax Table for special setup instructions.

Example:

Married Federal, Single Federal, Local Income Tax, FICA Medicare, FICA Social Security, FUTA, SUTA, etc.

Multiple Federal Tax Tables Required for 2020 W-4 Withholding

The 2020 W-4 form does NOT specify allowances; instead, information on additional income and deductions provided on the form will be used to fine-tune the tax withheld.  In SAMPro, this new form will be accommodated by the addition of new Federal tax tables and updates to the Employee record based on the fields completed on the new form. Since employees are not required to resubmit their pre-2020 W-4 if no changes are required, it will be necessary to maintain multiple tax tables to support the following scenarios:

Important Fields: Upper Section

Note: this Help Topic lists the standard configuration of all fields that may be included in this record. If you are using a customized system, the fields displayed on your screen may not exactly match the order or status (required, optional, default, display-only, hidden, etc.) of the fields described in this Topic.

This screen is divided into two sections. The upper section of the screen consists of the data-entry fields that identify this table and control how credits, deductions, and allowances are handled. The lower section of the screen lists the tax rate(s) assessed.

Tax Table Id: (10 character, only . and - special characters)
A unique Id for this table. This field is required.

Tip: For Local Authorities, the Tax Authority Id and the corresponding Tax Table Id should be identical to correctly support employee withholding.

Important Note: For government reporting, please use the following formats for FICA Social Security and Medicare Id's:

FICA SS (note the blank space separating FICA and SS)
FICA MC (note the blank space separating FICA and MC)
FICA MCAD (for high wage earner's additional tax)

See Version 9.1 Medicare Additional Tax procedures on for details on how to handle FICA MCAD in version 9.1. Also see Refine Medicare Additional Tax for High Wage Earners

 

Tax Table Name: (30 character)
A brief description of this table. This field is optional but highly recommended.

Tax Authority Id:
Enter the Tax Authority Id this table is used for. Double-click on this field to select from a list.

Standard Credit
If a standard tax credit is allowed, enter its annual dollar amount.

Standard Deduction
If a standard deduction is allowed, enter its annual dollar amount. Note that these fields should be completed for the FEDSD-S and FEDSD-M tables.

Std Ded Pct:
If a standard deduction percentage is allowed, enter the appropriate percent (e.g., 5% would be entered as 5.00).

Std Ded Min: Std Ded Max:
If applicable, enter the minimum and maximum amounts for standard deductions.

Amt Per Allow: (5 lines)
If an allowance amount is applicable for this tax table, (e.g., FED-S and FED-M), enter its annual dollar amount in the first line of this table. If differing categories of allowance are supported, each category amount should be entered on its own line in the table. For example, suppose a state taxing authority specified $3000 for the first and second allowances, but only $2200 for additional allowances. In this case, the Amount Per Allowance: table would be completed as follows:

 Amt Per Allow:
 3,000.00
 2,200.00

In the TAX tab of the Employee record, an employee with 4 withholding allowances would be flagged as having two category 1 ($3000) allowances and two category 2 ($2200) allowances for this taxing authority.

Credit Per Allow: (5 lines)
If a tax credit per allowance is applicable for this tax table, (e.g., some state tax tables), enter its annual dollar amount in the first line of this table. If differing categories of credits are supported, each category amount should be entered on its own line in the table. For example, suppose a state taxing authority specified a $100 credit for the first allowances, a $75 credit for the second allowance, and a $50 credit for any additional allowances. In this case, the Credit Per Allow: table would be completed as follows:

 Credit Per Allow:
 100.00
 75.00
 50.00

In the TAX tab of the Employee record, an employee with 4 withholding allowances would be flagged as having one category 1 ($100) credits, one category 2 ($75) credits, and two category 3 ($50) credits for this taxing authority.

Max FIT Allowance:
The maximum Federal Income Tax annual allowance in dollars.

Max FICA Allowance:
The maximum FICA annual allowance in dollars.

Tax Employee?
Check this box (by double-clicking or pressing the Spacebar) if this tax is to be paid by the employee.

Employee Limit: Limit Type: NA Payroll Period Monthly Annual Quarterly
If Tax Employee is checked, you may use this field to enter the maximum Employee Limit (if it exists) in TAX WITHHOLDING dollars (not the a limit on the income that may be taxed) per Limit Type (Payroll Period, Monthly Annual, or Quarterly.)

Setup Note: The 'Limit' in the Base Tax tab is for entering the WAGE brackets. As with many of SAMPRO's features, there are multiple setup options available. In many cases one could achieve the same end result using either (a) the Employee / Employee Limits or (b) the base Tax Limits or (c) a combination of both.

W-2 Category Employee
The system uses this field to handle unemployment insurance, disability, etc. (box 12 and 14 box codes) for W2 printing. It may be used to specify the titles for unemployment, insurance, disability, etc. Only the Box 14 description should be entered, i.e FLI,TDI or SUI in the User1 field.

In previous releases, users could implement SDI (Box 14 on form W-2) by setting the User 1 field in the applicable Tax Authority to SDI. This functionality is still supported. This field supercedes the use of the User 1 field for the handling of box 12 and 14 codes, and for titles for unemployment insurance. The User 1 field is still supported if this field is blank. 

Tax Employer?
Check this box (by double-clicking or pressing the Spacebar) if this tax is to be paid by the employer.

Employer Limit: Limit Type: NA Payroll Period Monthly Annual Quarterly
If Tax Employer is checked, you may use this field to enter the maximum Employer Limit (if it exists) in TAX WITHHOLDING dollars (not the a limit on the income that may be taxed) per Limit Type (Payroll Period, Monthly Annual, or Quarterly.)

W-2 Category Employer
Reserved for Future Use.

Tax Table User1:
This field has been superceded by the W-2 Category field for the handling of box 12 and 14 codes, and for titles for unemployment insurance. It is still supported if the W-2 Employee Category field is blank. 

Tax Table User2: (30 character)
Tax Table User2 may be used for entering any information your company wishes to retain (e.g., additional account numbers).

Obsolete?
If this field is checked, the record is considered Obsolete, and it may not be directly specified during data-entry (either by entering the Id or selecting it from a list). Obsoleted records can be edited and the Obsolete option can be un-done when necessary. When you check the Obsolete field, the following message box will be displayed:

“You have elected to obsolete this record. Would you like to see existing references to this record?”

If you click Yes, the system will display a viewer that provides click-through access to all tables referencing this record.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

Important Fields: Lower Section

Base Tax Tab

Limit:
The annual earnings upper limit to be taxed at this Rate.

Important Note: the upper bracket should be set to ALL 9s (including the cents) when there is no wage limit. The software includes validation so that entering 99,999,999.nn results in the value being set to 99,999,999.99.  This makes the bracket amount consistent with the Compute Payroll code's expectations.

Rate:
The tax rate percent to be applied up to the limit.

Additional Amount:
The annual additional amount to be deducted.
