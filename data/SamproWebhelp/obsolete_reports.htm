Obsolete Reports

 

Some SAM Pro reports, such as the Service History Report, are obsolete and no longer supported. They should already be marked as obsolete in the database’s function table – but if your user has been on SAM Pro for a while, they may still be out there. For example, this report can be set to be obsolete via the SQL statement:

update fnctn set fnctn_dsply='Y' where fnctn_id='rsrvcehstry'

The UD Service History Detail report replaces this one.

The following list of reports that should also be flagged as obsolete for versions 6.0 and up; alternately, you may deny access via the User Group Functions List.
