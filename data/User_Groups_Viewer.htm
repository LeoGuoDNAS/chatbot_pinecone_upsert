User Groups Viewer

Purpose:

User Groups define the various categories of users on the system. This record also enables you to select the number of days a user password is valid for members of this group, select the minimum number of letters for a password, and specify the maximum number of failed login attempts that members of this group are allowed.

This viewer can also be used to select which functions are accessible to a specific User Group. The Tabs for Viewers, Reports, Processes (Starter functions) and Analytics allow the user to easily to locate items to enable/disable. Note that changes to these settings cannot be made to the User Group 'All'. Note that in User Group All, the Viewer Export Function is supported even if the box is not flagged.

Example:

User groups enable you to maintain security and control on your system. For example, you could create groups for your sales, administrative, and supervisory personnel.

Important Fields: Upper Section

Note: this Help Topic lists the standard configuration of all fields which may be included in this record. If you are using a customized system, the fields displayed on your screen may not exactly match the order or status (required, optional, default, display-only, hidden, etc.) of the fields described in this Topic.

Group Id:
This required alphanumeric field represents a unique group Id.

User Group All should only be used by system admins or other high level personnel. It may NOT be altered by users. Note that in User Group All, the Viewer Export Function is supported even if the box is not flagged.

Group Name:
A brief description of the group. This field is optional.

Security Id:
If multiple location security is implemented on your centralized database system, this informational field displays the Security Id of the local user that created this record. If multiple location security is not implemented on your system, or if this record was created by a global user, this field will be blank.

Password Days:
If this field is set to the default (0), a user's password will be good indefinitely. If this field is set to a number greater than zero, it defines the number of days a password is good for before it must be reset by the user. When this number has been reached, a member of this group will be required to enter and confirm a new password as part of the login process. When a member of this group enters a new password, the Password Renewal field in the User/User Functions record will be reset to the current date plus the number of Password Days.

Minimum Password Length:
The minimum number of alphanumeric characters required for a group member's password. Longer passwords (6-10 characters) are generally more secure, but also more difficult to remember.

Max Failed Logins Allowed:
If this field is set to the default (0), a user may attempt to login indefinitely without being locked out. If this field is set to a number greater than zero, this field defines maximum number of logins a member of this group is allowed before he or she is locked out of the system. This feature reduces the chances of unauthorized users logging in, but increases the chances of an authorized user being "locked out" due to typos in the login. If an authorized user accidentally locks him or herself out, the Failed Login Attempts counter in the user's record must be reset to 0 by a user with access to the User or User Desktop viewer.

 Login Processing in 9.1.611.051 and Above:

WHEN the Id is valid AND the Password is valid AND the user's Failed Login Limit is exceeded

THEN the 'User Account has too many Failed Logins' error is issued instead of the 'Invalid User Id and/or Password' error.

WHEN the Id is valid AND but Password is invalid AND the user's Failed Login Limit is exceeded

THEN also, the 'User Account has too many Failed Logins' error is issued instead of the 'Invalid User Id and/or Password' error.

WHEN the Id is valid AND the Password is valid AND the Logon process has been successfully completed

THEN the user's Failed Logon Count is reset to zero. Otherwise, the Failed Logon Count will remain in effect for all future logon attempts.

Password Requires Both Alpha and Numeric
If you flag this field, users belonging to this Group must include at least one numeric and one alphabetic character (a letter and a number) in their passwords.

Important Fields: Lower Section

Viewers Tab

Function Name:
The name of the viewer; this display-only field is maintained by the system.

Access?
If this field is checked, members of this group will have access to this function and can view all the information available on the record.  If this box is blank, members of this group will not have access to the records in the function.  However, users will be granted access to the LIST views. 

Add
If the Access and the Add boxes are checked, members of this user group will be able to add these records.

Change
If the Access and the Change boxes are checked, members of this user group will be able to change these records.

Delete
If the Access and the Delete boxes are checked, members of this user group will be able to delete these records providing a) the record has no associated references and b) programming logic does not disallow it.  

Export
To disable printing and saving of a viewer list (as a report or csv file), clear the Export box. The csv and table report buttons will be grayed out in a record viewer it the Export box has been cleared. Note that in User Group All, the Export Function is supported even if the box is not flagged.

 

Reports Tab:

This tab allows reports to be enabled / disabled for this User Group.

ALL:
If this field is set to Yes, members of this user group will be able to run this report. Clicking a line toggles the setting between Yes and No.  The Check All button sets all Report lines to Yes; the Clear All button sets all Report lines to No.

Function Name:  Function Id:
The Name and Id of the SAMPro Report, or the Function associated with the User Defined report.

Type:  
This tab lists both system reports (Type SAMPro Report) and the user-defined functions associated with reports that are imported into your database (Type UDR Class: User Report).  

 

Processes Tab:

This tab allows processes (e.g., SAMPro Starter functions)  to be enabled / disabled for this User Group.

Allow Access:
If this field is set to Yes, members of this user group will be able to run this function. Clicking a line toggles the setting between Yes and No.  The Check All button sets all Process lines to Yes; the Clear All button sets all Process lines to No.

Function Name: Function Id:
The Name and Id of the Function.

 

Analytics Tab:

This tab allows Analytics (KPI's) to be enabled / disabled for this User Group.

Allow Access:
This field is set to Yes, members of this user group will be able to run this Analytic. Clicking a line toggles the setting between Yes and No.  The Check All button sets all Analytic lines to Yes; the Clear All button sets all Analytic lines to No.

KPI Category: KPI Name:  KPI Type: Description:
Displays information about the KPI.

Function Type:  Function Id:
The Type and Id of the User-Defined Function associated with the Analytic (KPI).
