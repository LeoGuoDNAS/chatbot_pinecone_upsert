1297 Add Time Hours Validation Against Timesheet - ASI

1. Add a new subtotal line(grid) at the bottom of the Time Sheet grid's existing TOTALS line.  This grid should have a single column for each day shown in the Time Sheet grid.  The value to show in the cell is the accumlated Hours/Minutes from the Time Log grid for each specific day.

2. Implement validation to prevent a user from submitting a Time Sheet where the Time Log entries don't match the Time Sheet hours.  Note that the Time Log Entry doesn't attempt to classify hours as Reg/Ovt/Dbl so the comparison we're adding is for the sum of the R/O/D.  In other words, the sum of the Time Log hours/minutes must equal the sum of the R/O/D hours for each date.

3. The validation described in #2 doesn't apply to all employees.  Currently, SAMPRO offers no way for ASI to identify which employees are subject to the validation.  That being the case, they have opted to define the validation rule as follows:  when an employee has NO Time Log entries for the period, then the validation is disabled.  Basically, it means that an employee can ignore the Time Log entry form and skip that part of the entry.  Employees that should have entered Time Log entries and didn't will be identified on a Time Sheet Proofing Report.  They think they can manage this situation effectively on the server-side so we'll give them a shot doing it that way first.

Important Note: an RAQ is required to implement this mod. See case for details.
