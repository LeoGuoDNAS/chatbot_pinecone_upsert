Inventory Restock Pick List Entry

Purpose:

This function enables the user to enter the restock quantity actually picked. This function automatically makes inventory history entries adjusting quantity on order, quantity allocated, and quantities on hand.

Usage:

In previous versions, the system deleted Inventory Restock and Inventory Restock Pick Ticket Entry records when the Pick Ticket was closed. In this release, these records are only deleted if the DeleteClosedTransactions is set to true in the sys-viewer-vrstck entry.

A Status filter range has been added to the Inventory Restock List and Inventory Restock Pick List Entry viewers. By default, this range is set to Open. To view both Open and Closed entries, click the All filter button. To view only Closed entries, click the Closed filter button.  

Registry:

Registry Inventory Restock

Screen Organization:

The upper section of this screen enables you to specify the Restock To and From Locations. Once these fields have been entered, the system will populate the lower section with a list of the items that were picked.

Important Fields: Upper Section

Transaction Id:
To access an existing list, you may enter its Transaction Id.

Restock To Location:
The Location Id that you will restock to (e.g., a truck 1).

Restock From Location:
The Location Id from which you just picked the inventory (e.g., a regional warehouse).

S tatus: Open Closed
Closed By, Date Closed, and Time Closed

These display-only fields are maintained by the system. A Restock entry will be closed by the system when the Close Pick Ticket button is clicked. The Closed By, Date Closed, and Time Closed fields will also be set at this time.

Click this button to reprint the Restocking Pick Ticket. On the reprint, the quantity to pick will be computed as the original quantity to pick less the quantity already picked.

Click this button to auto fill the quantity picked fields to the value of the quantity to pick fields. At this point, the screen will be as it should be if the user physically picked the quantity appearing on the Restocking Pick Ticket. The user will be free to enter any exceptions in the lower section of this screen, and then Save this record.

After you have entered the Quantities Picked and saved this record, click the Close Pick Ticket button to close the restock list.

Select All: This allows users to select all items on the pick list at one time. 

 Field: push-autoSetTransferDone

At this point, no inventory history records will need to be automatically generated for items fully picked. For the remaining records, the following entries will be generated:

 The quantity on order for the restock-to location will be reduced by the quantity not picked.

 The quantity allocated for the restock-from location will be reduced by the quantity not-picked.

In previous versions, the system deleted Inventory Restock and Inventory Restock Pick Ticket Entry records when the Pick Ticket was closed. In this release, these records are only deleted if the DeleteClosedTransactions is set to true in the sys-viewer-vrstck entry.

Important Fields: Lower Section

Inventory Item Id: Qty Needed: Qty to Pick: Inventory Item Name:
These display-only fields default from the Restore List for this location pair.

Qty Picked:
Enter the quantity actually picked. The Set Qtys Picked button may be clicked to auto-fill this field.

Important buttons:

Click the Save button to record your picks.

When the restocking pick ticket record is saved, the following inventory history postings will be generated:

The quantity on order for the restock-to location will be reduced by the quantity picked.

The quantity on hand for the restock-to location will be increased by the quantity picked.

The quantity allocated for the restock-from location will be reduced by the quantity picked.

The quantity on hand for the restock-from location will be reduced by quantity picked.

When you have finished saving your picks, click the Close Pick Ticket button.
