Registry Start Compute Overtime

Purpose:

This registry specifies which overhead Job / Cost Code / Cost Category get the Overtime Premium charge based on the Employee's Branch.  A registry entry must be created for each Branch.

In Version 9.0 and above, Branches are not required for this function. However, if Branches are enabled and you wish to specify the Overhead Job by Branch, the corresponding sys-start-scmpteovrtme-branchid registry entries must be generated in order for this function to be run. The employee's branch is used to determine which job / costcode / costcategory gets the OT Premium charge, regardless of which direct jobs were charged originally.  

NOTE: the Earnings ID in the Employee's Earnings Tab must have appropriate wage lines flagged as "Add Pay Rate Factor?" in order for this functionality to work.

Additional Registries:

See Compute Overtime - No Checks Printed or Posted for details/additional registries for computing overtime when checks will not be generated.

Registry to Override RegPayTypeId and OTPayTypeId

Registry Id:   sys-start-scmpteovrtme

Setting Key: UseEmployeePayTypes  Setting Value: true  (default false)

With the above registry in place, the Employee Record provides overrides for the RegPayTypeId and OTPayTypeId settings. The decision to the Employee's Direct Pay or Indirect Pay defaults is based on the Job. If the Job is tagged as Overhead, the Indirect defaults are used. The Employee Direct Pay default are used if the Job is not Overhead.

Compute Overtime (MarketEquipment Method)

This 9.0 modification computes Overtime using a Weighted Average of Regular Pay Rates. AS with other overtime processing, the basic premise of this enhancement is that hourly time is entered as regular time, and Compute Overtime is run to generate an Overtime Premium for 'excess' hours over a regular work week BEFORE running Compute Payroll.  The Overtime Premium is assessed to an overhead Job and computed at the Average hourly paid for each employee for the payroll period.

The Compute Overtime checklist lists the labor eligible for adjustment for review; checked lines will be processed and the appropriate Labor Journal will be created to make Overtime adjustments. These Labor Journal entries remove hours at Regular time and insert hours at Overtime.

For those employees with Earnings IDs marked to Job Cost, you will see the cost incurred by the adjustments on the Overtime Job’s Revenues tab.

Registry Option to Calculate Overtime by Day / Payroll Period

Id:  sys-start-scmpteovrtme

Setting Key: method   Setting Value: Haller2

When the method is set to Haller2, the software will first compute by hours in day (>8 is ot), then by days in the pay period (>40 is OT).  Whichever gives the employee a better deal is used.  If there is a 'tie' the daily OT is used. The following registry is also required when method=Haller2.  

Setting Key: ServiceEmployeeUserField    Setting Value: emplye_usr6

Only Service Employees are eligible for Compute OT. When setting the flag on the employee, set this value to N if you want to Compute OT using ONLY the "By Pay Period" method.  Set this value to Y if you want to Compute OT using the "By Day" or "By Pay Period" methods (which ever is best).     

Setting Key: IncludeHolidayHours   Setting Value: Y (default is N)

Setting Key: IncludeSickHours   Setting Value: Y (default is N)

Setting Key: IncludeVacationHours   Setting Value: Y (default is N)

Branch Registry to Calculate Overtime by Day / Payroll Period

If Branch processing is desired, the Overhead Job information may be specified for each Branch as follows.

Id:  sys-start-scmpteovrtme-branchid

A registry setting must be made for each applicable Branch.

Setting Key: JobId   Setting Value: ValidOHJobId

ValidOHJobId must reference a valid Job with its Overhead flag set to be used when Overtime entries are generated for Jobs in this Branch.

Setting Key: CostCodeId  Setting Value:  ValidOHCostCodeId

ValidOHCostCodeId must reference the Job Cost Code Id (belonging to the Overhead Job referenced in the previous entry) to be used when Overtime entries are generated this function.

Note that if the Job Cost Code Id is the same for all overhead jobs, it may be specified in the base sys-start-scmpteovrtme.
