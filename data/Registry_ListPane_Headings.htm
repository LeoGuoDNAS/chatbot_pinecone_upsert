Registry To Set ListPane Headings

Purpose:

Some SAM Pro record viewers include list tabs with data headers above the column titles. This registry entry enables you to control the font used for the headings used above the column titles of these parent or child tab lists.

Testing Your Registry Entries
The system does not verify the validity of registry entries. Be sure to test your registry entries after completing them to verify that they are functioning correctly.

Setting Values are Uppercase-Lowercase Sensitive:
The Setting Value is case-sensitive: be sure that you enter the value exactly as specified in the documentation. If the documentation specifies a value of true, do NOT enter True or TRUE.

Important Fields:

Id

 sys-font-ListPaneHeading

See the Standard Registry section in Registry Viewer for additional Registry Id formats to allow entries for specific Users, Branches, Security, or User Groups to override the main entry.

Contents:

Setting Key: Facename Setting Value: aFaceName

where aFaceName is replaced by a valid font facename (e.g., Arial, Tahoma, etc.). The font must be installed on users’ computers in order for it to be used.

Setting Key: bold  Setting Value: yes or no

specify bold=yes to use the bold version of the font, or bold=no to use the normal version.

Setting Key: points  Setting Value: points

Where points is replaced with an appropriate point size (point sizes from 8-10 are recommended).

Example:

Id: sys-font-ListPaneHeading

Setting Key: Facename Setting Value: Arial

Setting Key: Bold  Setting Value: Yes

Setting Key: Points  Setting Value: 9
