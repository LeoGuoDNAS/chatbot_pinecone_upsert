Registry Start Client Site Import

Id: sys-start-Sclntsteimprt

Registry Option to Specify Directory for CSV Site Import Files

When this function is run, SAMPro will open a browser window so that you may select the appropriate .cvs file for importing your client site data. This registry allows you to specify the location of the files.

Setting Key: CSVDirectory   Setting Value: path (specify directory to place CSV files)

Registry to Allow an Existing Site to be Used

Id: sys-start-Sclntsteimprt 

Setting Key: UseExistingSite  Setting Value: true (default false)

This setting eliminates the requirement that the New Site Id must not exist and allow additional jobs to be imported for an existing Site. Note that any existing Sites will NOT be updated with contents from CSV.

Registry To Use the Zip Code Table to set the Service Zone / Branch  

Registry Id: sys-start-sclntsteimprt

Setting Key: UseZipCodeTable  Setting Value: true (default false)

When enabled, this option uses the Service Zone from the CSV on the import if specified and if not it retrieves it from the Zip Code table.

In addition, when this registry is enabled, the Branch from the Zip Code will be used for the Site Branch and Job Branch.

Registry to Set Tax Group from Avalara

Setting Key: SetTaxGroupFromAvalara    Setting Value: true (default false)

Enabling this registry uses Avalara's Avatax Web functionality to set the Job’s Sales Tax Group when appropriate. If this is used the following must be setup must be observed:

The Sales Tax Group in the Job Template(s) must be blank

The Job Templates must have the Job Tax Status set to Sales

The Sales Tax Group must be blank in the Zip Code table.

When not using Avalara, the spreadsheet needs to be populated with each Sites Sales Tax Group Id (Column BU).  Contact support to have the new Jobs populated with this value – the Job’s Date Open will need to be provided along with a list of new Job Id’s to be updated.
